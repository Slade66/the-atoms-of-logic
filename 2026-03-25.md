## df -h 命令详解与磁盘分区查看

`df` 的全称是 "disk free"，用于查看已挂载磁盘的使用率。该命令会列出所有已经挂载的文件系统、总容量、已用空间、可用空间以及具体的挂载点。

在实际运维中，可以通过 `df -h` 快速查看系统的 `/`（根目录）等各大分区加起来实际用掉了多少。

其中，`-h` 参数表示 `human-readable`，它会以我们习惯的 KB、MB、GB、TB 等单位直观地显示大小，而不是一堆难以阅读的字节数。

## 经典软件通用性与技术复利效应

在玩 VMware ESXi 时，发现它创建虚拟机和 VMware Workstation 几乎是一样的。

这体现出“只要会用经典软件，其它同类工具也都是一样用法，经典永存”的规律。技术知识是具有**复利效应**的，学习一次，以后很多地方都能用。例如学完了 GitHub 的部署密钥（Deploy Keys），在部署私有 Gitea 时也是完全一样的套路。

## 现场离线部署的 Docker 镜像打包与交付策略

对于需要交付到客户现场进行本地/离线部署的项目，应当遵循以下策略：

1. **镜像交付优先**：优先准备 Docker 镜像，配合外部配置文件模板，而非携带源码到现场构建。
2. **规避无网风险**：客户现场通常没有外网，而构建项目需要拉取大量外部依赖。如果直接携带源码，万一在现场拉不到 Go 依赖包或工具链，部署将会直接卡死。
3. **避免环境编译差异**：源代码在不同的系统/内核环境下编译出的结果可能有偏差，且现场编译极其容易遇到缺少 C 库（如果项目使用了 CGO）等编译报错。
4. **镜像打包流程示例**：
   * 将镜像导出为 tar 包：
     ```bash
     docker save -o network-config-generator.tar network-config-generator:latest
     ```
   * 对导出的 tar 包进行压缩以减小体积：
     ```bash
     gzip network-config-generator.tar
     ```

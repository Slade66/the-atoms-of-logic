###

Docker Desktop 有 MCP Toolkit，这是一个很好的运行各种 MCP 服务器的平台。

### 工具

AI 能调用的工具，可执行的命令或函数，执行一个操作。AI 通过 MCP Client 调用工具，MCP Server 执行。
工具最后要调用相应的函数，工具需要的参数，工具的介绍。

在服务器上定义工具和资源，AI 会自动识别并使用它们。

### 资源
服务器提供的只读的数据项（比如文档），客户端随后可以按需读取这些信息。
指的是服务器上希望提供给客户端的数据。
uri
也可以是执行一个函数，获取所有数据。

### 采样（Samplings）
反过来，服务器需要获取客户端的信息。服务器在执行任务时，反过来向宿主应用说一句：
「我这边需要大模型帮我想一想 / 写一段东西，你帮我问一下模型，把结果给我。」

采样是一种工具的类型

采样流程：
服务器正在执行某个工具，比如 summarizeDocument。

它读到一个超大的文档，觉得「这个让我自己用 if/for 去写规则太蠢了，不如交给 LLM」。

服务器发起一个「采样请求」，里面带上：

要给模型看的提示词（system / user 内容）

相关的上下文（比如文档内容或片段）

限制（最大 token、要的风格等）

客户端收到这个请求，会先审核 / 修改：

看看是不是安全的

要不要裁剪内容

用哪个具体模型来跑

客户端真正去调用大模型，拿到补全结果。

客户端把这段结果返回给服务器。

服务器用这段结果作为「中间决策」或「中间文本」，继续完成自己的工具逻辑，最后再把最终结果返回给模型。

### 测试 MCP Server

`npx @modelcontextprotocol/inspector` 启动 MCP Server 检查器
`set DANGEROUSLY_OMIT_AUTH=true` 忽略token

### MCP 开发

创建服务器
给服务器注册工具，工具的名字、工具的AI注解（只读？破坏？运行多次有副作用？会与外部世界交互吗？），工具的说明，工具的参数和输出结构，工具调用哪个函数
服务器绑定连接层启动

返回错误时，要非常具体。
返回的结果是结构化的。有类型和数值。

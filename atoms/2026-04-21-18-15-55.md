# Kratos 的错误处理逻辑

## 一、核心主张：业务只返 error，编码由框架兜底

- **业务层只负责返 error** 从"error → HTTP 响应"的整条链路由 Kratos 接管，业务代码里不用写任何编码逻辑。

## 二、核心数据结构：`*errors.Error`

- **四个字段各司其职** `Code` 是 HTTP 状态，`Reason` 是机器可读枚举，`Message` 是人可读原因，`Metadata` 是附加键值。

    ```go
    type Error struct {
        Code     int32              // HTTP 状态码，如 404 / 500
        Reason   string             // 机器可读的枚举字符串，如 "ESXI_HOST_NOT_FOUND"
        Message  string             // 人可读的原因，如 "id=abc not found"
        Metadata map[string]string  // 附加键值，比如 {"host": "10.0.0.1"}
    }
    ```

## 三、业务层怎么抛错

### 用内置构造器：一类 HTTP 状态一个

- **直接 return 构造结果** 每个构造器返回的就是 `*Error`，业务层一行搞定。

    ```go
    import "github.com/go-kratos/kratos/v2/errors"

    // HTTP 400，参数错误
    return errors.BadRequest("INVALID_NAME", "name must not be empty")

    // HTTP 404，资源不存在
    return errors.NotFound("ESXI_HOST_NOT_FOUND", fmt.Sprintf("id=%s", id))

    // HTTP 409，唯一键冲突
    return errors.Conflict("ESXI_HOST_NAME_TAKEN", fmt.Sprintf("name=%s", name))

    // HTTP 500，服务端内部错误
    return errors.InternalServer("DB_WRITE_FAILED", err.Error())

    // 自定义
    return errors.New(418, "I_AM_A_TEAPOT", "...")
    ```

### 透传普通 error 的兜底行为

- **非 `*Error` 会被包装** Kratos 的 `FromError` 把普通 error 包成 `*Error{Code:500, Reason:"", Message: err.Error()}`，例如 `gorm.ErrRecordNotFound` / `sql.ErrNoRows` / `io.EOF` 等。

    ```go
    // 仓库层
    item, err := repo.GetByID(ctx, id)
    if err != nil {
        return nil, err   // err 可能是 gorm.ErrRecordNotFound、sql.ErrNoRows...
    }
    ```

- **代价** 这样写能跑，但所有错误 HTTP 状态都统一成 500、`Reason` 为空——前端判错和运维排障都丢信息，只作最后一道兜底用。

## 四、Metadata：给错误挂附加上下文

- **用 `WithMetadata` 追加键值** 前端和运维可以基于 metadata 做更精细的提示（比如按 `attempt` 决定要不要重试）。

    ```go
    return errors.NotFound("ESXI_HOST_NOT_FOUND", "...").
        WithMetadata(map[string]string{"id": id, "attempt": "3"})
    ```

## 五、`protoc-gen-go-errors`：可选增强

### 在 proto 里集中登记错误码

- **用 `ErrorReason` enum 定义** 每个枚举值绑一个 HTTP 状态码，`make errors` 自动生成 Go 辅助函数。

    ```proto
    import "errors/errors.proto";

    enum ErrorReason {
      option (errors.default_code) = 500;
      ESXI_HOST_NOT_FOUND     = 0 [(errors.code) = 404];
      ESXI_HOST_NAME_CONFLICT = 1 [(errors.code) = 409];
    }
    ```

### 生成后的用法

- **抛错一行完成** 一次调用同时搞定 code / reason / message。

    ```go
    return v1.ErrorEsxiHostNotFound("id=%s", id)
    ```

- **判错也有专用函数** 调用方不必手写 `Code` / `Reason` 比较。

    ```go
    if v1.IsEsxiHostNotFound(err) {
        // do something
    }
    ```

### 收益

- **错误码有全局清单** 所有错误码在 proto 里集中登记，前端和客户端 SDK 生成代码里能看到完整枚举，避免口头约定。

# Appsmith 如何连接 MySQL 数据库

### 一、问题背景

- **使用场景** 在 Appsmith 中创建应用时，需要先连接 MySQL 数据库，后续页面才能通过 Query 读取或写入业务数据。

- **核心问题** Appsmith 能否连上 MySQL，主要取决于网络是否互通，以及 `Host address`、`Port`、数据库账号密码是否填写正确。

### 二、先确认 Appsmith 和 MySQL 的网络关系

- **容器网络** 如果 `MySQL` 容器和 `Appsmith` 容器在同一个 `Docker` 网络内，两个容器可以通过容器网络互相访问。

- **映射访问** 如果 `MySQL` 是通过宿主机端口映射暴露出来的，`Host address` 填服务器地址，`Port` 填映射后的端口。

- **配置重点** 不管使用哪种访问方式，Appsmith 里填写的地址和端口都必须是从 Appsmith 容器视角能够访问到的地址和端口。

### 三、在 Appsmith 中创建 MySQL 数据源

1. 登录 `Appsmith`。

2. 在 `Appsmith` 首页新建一个 App。

3. 在左侧菜单进入 `Datasources`。

4. 在数据源列表里选择 `MySQL`。

### 四、填写 MySQL 连接参数

1. `Host address` 填写 MySQL 的访问地址。

2. `Port` 填写数据库端口。

3. `Database name` 填写数据库名。

4. `Username` 填写数据库用户名。

5. `Password` 填写数据库密码。

6. `SSL` 可以先关闭。

- **地址判断** 如果走端口映射，`Host address` 通常填 `Docker` 服务器 IP；如果走容器内网访问，则应填写 Appsmith 容器能解析到的 MySQL 地址。

- **排错方向** 连接失败时，优先检查网络是否互通、端口是否填对、数据库账号是否允许从 Appsmith 所在网络访问。

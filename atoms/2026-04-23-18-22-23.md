# MoRef：虚拟化资源的唯一标识

### MoRef 与 MOID

- **MoRef** Managed Object Reference，VMware vSphere 对“可被 API 管理的对象”分配的内部唯一句柄，包含类型信息。

- **MOID** Managed Object ID，MoRef 的 Value 部分，即具体的 ID 值（如 `"vm-4012"`）。有些文档里 MoRef 和 MOID 被混用，大部分时候可以认为是同一个东西。

### 为什么需要 MoRef

- **名称冲突** 同一个数据中心可能有几千台虚拟机，名称可能重复（如很多虚拟机都叫 "webserver-01"），系统无法定位具体是哪一个。

- **类比** 用名字找人相当于在社区喊“找李明”——有 100 个李明，无法定位；用唯一 ID 找人相当于用“京A·123456”车牌号找人，立刻定位。

### 实际格式与生命周期

- **组成** MoRef 由对象类型 + MOID 构成。

    | 对象 | Type | Value（MOID） |
    | --- | --- | --- |
    | 一台 VM（ESXi 直连） | `VirtualMachine` | `"110"` |
    | 一台 VM（vCenter 下） | `VirtualMachine` | `"vm-4012"` |
    | ESXi 主机 | `HostSystem` | `"ha-host"`（直连）/ `"host-15"`（vCenter） |
    | Datastore | `Datastore` | `"datastore-10"` |
    | 端口组 | `Network` | `"network-20"` |

- **生命周期** VM 存在期间，MOID 绝对不变。只有删除重建才会分配新 MOID。

# Go 的 time.Time

讲清 Go 的 `time.Time` 本质、常见操作（获取时间、时间运算、比较、格式化与解析）与易错点。

### `time.Time` 是“一个时间点”

不是“日期字符串”，也不是“时间戳整数”，而是 Go 里的时间对象。

## 常见操作

你最常做的事只有 4 类：拿当前时间、做加减、比较先后、格式化/解析。

### 获取时间

- **拿当前时间：** `now := time.Now()`

- **解析时间字符串：** `t, err := time.Parse("2006-01-02 15:04:05", "2026-03-17 14:30:00")`

    - **⚠️注意：** Go 的格式化模板不是 `%Y-%m-%d`（`YYYY-MM-DD`），而是一串神奇数字（`2006-01-02 15:04:05`）。

### 时间运算

- 加一段时间： `expireAt := now.Add(30 * time.Minute)`

- 求两个时间差：

  - `d := end.Sub(start)`

  - `time.Since(start)`
  
    ```go
    start := time.Now()
    // do something
    cost := time.Since(start) // 本质上等于：time.Now().Sub(start)
    ```

### 比较时间

- 不要自己拆年月日比较。

- 
    ```go
    if t1.Before(t2) { ... }
    if t1.After(t2) { ... }
    if t1.Equal(t2) { ... }
    ```

### 格式化输出

- `s := now.Format("2006-01-02 15:04:05")`

### 时区

- “时间”不只是一个数字，它还带着“这个数字属于哪个时区”的语义。

- 为什么存 UTC 更稳定
  - 同一时刻只有一种表示。
  - 举个最直接的例子：
    - 服务器 A 在上海，写入：2026-03-18 10:00:00
    - 服务器 B 在纽约，写入：2026-03-17 22:00:00
    - 这两个其实可能是同一时刻，但如果你没统一成 UTC，数据库里看起来像两个不同时间

- 存储时间统一使用 UTC，展示给用户时再转本地时区。

    ```go
    t := time.Now()
    fmt.Println(t)       // 2026-03-18 16:16:47.3167735 +0800 CST m=+0.000614601
    fmt.Println(t.UTC()) // 2026-03-18 08:16:47.3167735 +0000 UTC
    ```

- **转 UTC：** `utc := time.Now().UTC()`
  - time.Now().UTC() 的作用是把当前时间统一转换成 UTC 表示。
  - time.Now()：拿到本机当前时间，带本地时区

- **指定时区：**
  
    ```go
    loc, err := time.LoadLocation("Asia/Shanghai")
    t := time.Now().In(loc)
    ```

- 如果不转 UTC
  - 同一项目在不同机器上写出的时间不一致
  - 你本机是 Asia/Shanghai，服务器可能是 UTC，容器可能又是别的时区。
  - 同样的创建动作，存进去的 CreatedAt 会带不同 Location，后面日志和排查会很乱。
  - 举例：
  - "created_at": "2026-03-18T10:00:00+08:00"
  - "created_at": "2026-03-18T02:00:00Z"
  - 它们表示的是同一时刻，但字符串不一样，如果你不统一成 UTC，数据库里看起来像两个不同时间。

# Kratos 项目从创建到运行

## 一、前置要求

- **安装基础工具：** 装好下面这几样。

    - **`go`：** Go 语言工具链，编译和运行的基础。
    - **`protoc`：** Protocol Buffers 编译器，把 `.proto` 文件编译成代码。
    - **`protoc-gen-go`：** protoc 插件，生成 Go 结构体。
    - **`protoc-gen-go-grpc`：** protoc 插件，生成 gRPC 服务代码。
    - **`protoc-gen-go-http`：** protoc 插件，生成 Kratos 的 HTTP 服务代码。

- **安装 wire 工具：** `go install github.com/google/wire/cmd/wire@latest`。

- **设置 GOPROXY：** 加速依赖下载，`go env -w GOPROXY=https://goproxy.cn,direct`。

## 二、创建并运行 kratos 项目

1. **安装命令行工具：** `go install github.com/go-kratos/kratos/cmd/kratos/v2@latest`

2. **创建项目：** `kratos new 项目名`（项目名不能叫 `server`，否则会报错）

3. **选择项目模板：** 新版 kratos 会让你从几套预设模板里挑一个来生成项目骨架

    ```text
    kratos new server
    ┃ Select a template
    ┃ > Service
    ┃   Admin
    ┃   Custom (enter repo URL)
    ```

    - **`Service`：** 标准后端服务模板，即 `api/`、`cmd/`、`internal/biz`、`internal/data`、`internal/service`、`internal/server` 的分层结构。
    - **`Admin`：** 面向“后台管理系统”的更重的脚手架，自带更多管理系统相关的样板代码。
    - **`Custom`（enter repo URL）：** 填一个自定义模板仓库地址，相当于交互式版本的 `-r` 参数。

4. **进入项目目录：** `cd helloworld`

5. **下载依赖：** `go mod tidy`

6. **生成代码：** `go generate ./...`（可以跳过）

    - **作用：** 主要是跑 wire 生成依赖注入代码（`cmd/server/wire.go` 里有 `//go:generate wire` 指令）。
    - **首次可跳过：** layout 本身已带一份生成好的 `wire_gen.go`，严格说首次运行可以跳过它直接 `kratos run`。
    - **何时必须重跑：** 只要改了 provider 就必须重新执行。

7. **运行项目：** `kratos run`

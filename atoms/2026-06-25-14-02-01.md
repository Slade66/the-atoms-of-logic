# Kratos 项目从创建到运行

## 一、前置要求

- **安装基础工具：** 装好下面这几样。

    - **`go`：** Go 语言工具链，编译和运行的基础。

    - **`buf`：** Protobuf 构建工具，负责读取 `.proto`、解析依赖并调用插件生成代码。`go install github.com/bufbuild/buf/cmd/buf@latest`（buf 已内置 protoc 的编译能力，无需单独安装 `protoc` 二进制）。
    
    - **`protoc-gen-go`：** 生成 Go 结构体的插件，由 buf 调用。
    
    - **`protoc-gen-go-grpc`：** 生成 gRPC 服务代码的插件，由 buf 调用。
    
    - **`protoc-gen-go-http`：** 生成 Kratos HTTP 服务代码的插件，由 buf 调用。

- **安装 wire 工具：** `go install github.com/google/wire/cmd/wire@latest`。

- **设置 GOPROXY：** 加速依赖下载，`go env -w GOPROXY=https://goproxy.cn,direct`。

> 提示：`buf` 是“调度者”，`protoc-gen-*` 是真正干活的“插件”。

## 二、创建并运行 kratos 项目

1. **安装命令行工具：** `go install github.com/go-kratos/kratos/cmd/kratos/v3@latest`

2. **创建项目：** `kratos new 项目名`（项目名不能叫 `server`，否则会报错）

3. **选择项目模板：** 新版 kratos 会让你从几套预设模板里挑一个来生成项目骨架

    ```text
    kratos new server
    ┃ Select a template
    ┃ > Service
    ┃   Admin
    ┃   Custom (enter repo URL)
    ```

    - **`Service`：** 标准后端服务模板，即 `api/`、`cmd/`、`internal/biz`、`internal/data`、`internal/service`、`internal/server` 的分层结构。
    
    - **`Admin`：** 面向“后台管理系统”的更重的脚手架，自带更多管理系统相关的样板代码。
    
    - **`Custom`（enter repo URL）：** 填一个自定义模板仓库地址，相当于交互式版本的 `-r` 参数。

4. **进入项目目录：** `cd helloworld`

5. **生成代码：** 先用 buf 生成 Protobuf 代码，再跑 wire（`make all` 末尾会自动 `go mod tidy`，无需单独下载依赖）。

    - **最省事的方式是一条命令：**

    ```bash
    make all        # = make api + make config + make generate
    ```

    - **等价于手动分步：**

    ```bash
    make api        # buf 生成 api/**/*.pb.go
    make config     # buf 生成 internal/conf/conf.pb.go
    go generate ./... && go mod tidy   # 跑 wire 生成依赖注入代码
    ```
    
    - **何时必须重跑：** 改了 `.proto` 后重跑 `make api` / `make config`；
      改了 provider 后重跑 `go generate`。

6. **运行项目：** `kratos run`（启动成功会看到 HTTP `:8000` 与 gRPC `:9000` 监听日志。）

# 虚拟机整机迁移：Agent 方案与 CBT 方案深度解析

### 🧠 在线迁移的核心逻辑：全量 + 持续增量 + 最终收口

- **在线迁移三步走** 无论是 Agent 还是 CBT 方案，实现“在线整机迁移”的核心逻辑是高度一致的：通过牺牲一定的背景性能，换取最小化的业务停机。

    1. **全量基线同步**：首先将源虚机的磁盘数据完整地同步到目标端。

    2. **持续增量捕捉**：在全量复制期间，业务仍在运行并产生新数据，迁移引擎需通过某种机制捕捉这些“脏块”并同步。

    3. **最终收口切换**：当差额极小时停止业务，完成最后一批脏块同步，随后在中间机环境进行驱动注入（如 FusionCompute 驱动）与引导修复，最后在目标平台拉起虚机。

### 🧠 CBT 方案：虚拟化层的无侵入块记录

- **逻辑位置** CBT（Changed Block Tracking）将增量捕获逻辑下沉到虚拟化层（如 VMware 的 VMkernel）。由它记录哪些虚拟机的哪些磁盘块发生了变动，迁移程序通过 API 直接向平台查询并拉取变化块。

- **核心价值：零侵入性** 方案最大的优势在于不需要进入客户机操作系统内部。这避免了处理复杂的内核问题，也不会引发驱动冲突或杀毒软件的误拦截，是虚拟化环境下的首选方案。

### 🧠 Agent 方案：客户机内的实时写入监控

- **逻辑视角** Agent 方案的核心是在虚拟机的内部捕获增量。通过在源 OS 内安装代理程序，利用操作系统的 Write Filter（写过滤驱动）拦截实时写请求，并由 Agent 自行将变更块推送到目标端。

- **核心优势：平台适配面广** 方案最大的价值在于脱离了对 VMware CBT 的依赖。在无法操作 vCenter/ESXi、CBT 故障或源端为非 VMware 平台时，Agent 方案只要能安装代理，即可支持物理机、Hyper-V、KVM 乃至公有云实例的迁移。

- **优势：缩短停机窗口** Agent 采用持续“盯写入”的模式。不同于部分 CBT 实现中“打快照 -> 查变化 -> 拉变化”的周期性拉取，Agent 在高频变更场景下能实时推送脏块数据，使停机同步的时间窗缩减至最小。

- **优势：天然绕过磁盘加密障碍** 由于 Agent 运行在操作系统内部，其读取的数据已由系统驱动层自动解密。这使其能够天然绕过 BitLocker 或 LUKS 等磁盘加密技术带来的读取难题，无需处理复杂的离线解密问题。

### 🧠 Agent 方案的局限性与实施成本

- **高度侵入性与运维考量** 方案必须登录每台源机进行安装、权限分配、驱动注入、杀毒软件兼容性处理以及最终的卸载清理。对于大规模迁移项目，这种逐台触达的模式会产生巨大的运维压力。

- **面对 OS 底层内核** CBT 方案在虚拟化层看到的是“虚拟机磁盘（VMDK）”这一层抽象，不关心客户机内核。而 Agent 必须直面 Windows 和 Linux 的版本与内核差异，适配成本极高，且存在驱动冲突导致蓝屏的风险。

### 🧠 Agent 方案的技术实现路径

- **代理程序安装** 在每一台待迁移的 Windows/Linux 虚拟机内部部署 Agent 代理程序，作为数据捕获的端点。

- **全量基线复制** Agent 首先会执行磁盘全盘扫描，将现有数据完整同步至目标端，建立首次全量基线。

- **持续增量捕获** 全量完成后，Agent 转入增量监控模式。它不再反复扫描全盘，而是利用操作系统层的 Write Filter（写过滤驱动）拦截实时写请求，持续记录自全量完成后发生变化的磁盘块。

- **数据推送同步** Agent 实时或定时将捕获到的变化块推送到中转端或目标存储，保持两端数据的动态趋同。

- **停机收敛与驱动重构** 当数据趋于一致时，执行最终的切换流程：

    1. **业务停机**：停止业务并关闭源虚机，刷写物理内存中最后一批未完成的脏块。

    2. **离线重构（V2V 转换）**：在中转机或目标端完成系统盘修复，包括注入 FusionCompute 所需驱动、修复系统引导（Bootloader）以及调整网卡配置。

    3. **业务上云**：在目标虚拟化平台上通过重构后的磁盘拉起新虚机，完成整机迁移。

### 🧠 迁移平台的演进策略建议

- **混合双引擎架构** 如果目标是打造一个通用的 VMware → FusionCompute 在线迁移平台，最稳妥的架构路线是：

    1. **主路线：CBT 增量复制** 用以覆盖 90% 以上的标准 VMware 场景，追求无侵入、高效率的迁移体验。

    2. **备路线：Agent 增量复制** 作为保底方案，用于处理 CBT 被禁用、源非 VMware 环境或无法获取虚拟化管理权限的特殊场景。

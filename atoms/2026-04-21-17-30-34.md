# 不要叫 uuid，直接叫 id

## 一、结论

- **接口字段统一用 `id`** 写成 `string id = 1;`，值到底是 UUID / ULID / Snowflake / 短哈希由内部决定，不要叫 `uuid`。

## 二、为什么不叫 `uuid`

- **字段名不该泄露实现** 叫 `uuid` 等于把"这个 id 必须是 UUID 格式"写进对外接口。换一种生成策略（ULID、Snowflake、短哈希）时，字段名就尴尬。

- **前端认知会乱** 前端会问"这个 `uuid` 跟 `id` 是一个东西吗"，无谓的解释。

## 三、对比表

| 维度 | `string id = 1;`（值为 UUID） | `string uuid = 1;` |
| --- | --- | --- |
| 与生成策略解耦 | ✅ 换 ULID / Snowflake / 短哈希都不用动字段名 | ❌ 字段名绑死"必须是 UUID"，换 id 实现会乱 |
| 前端认知成本 | 全链路一致，零解释 | 需要解释 “`uuid` 和 `id` 是不是一回事” |

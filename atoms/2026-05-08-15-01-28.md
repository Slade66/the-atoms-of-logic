# Docker 同网络容器之间互相访问用哪个端口

### 一、核心结论

- **访问端口** 当容器 A 和容器 B 在同一个 Docker 网络内时，B 应该通过容器名访问 A 的“容器内部端口”，不是宿主机映射端口，`-p` 映射对它们没有意义。

### 二、端口映射的作用

- **`-p` 含义** `-p 宿主机端口:容器端口` 用来把容器端口暴露给宿主机或外部网络，它是给“外部访问者”用的。

### 三、访问示例

- **容器启动** 容器 A 内部服务监听 `8080`，并映射到宿主机 `9090`。

    ```bash
    docker run --name A --network my-net -p 9090:8080 my-image
    docker run --name B --network my-net another-image
    ```

- **正确访问** 在容器 B 内访问容器 A 时，应该使用容器名和容器内部端口。

    ```bash
    curl http://A:8080
    ```

### 四、端口选择规则

- **同一 Docker 网络内的容器** 使用容器内部端口，例如 `8080`。

- **宿主机上的进程** 使用宿主机映射端口，例如 `9090`。

- **外部机器访问宿主机** 使用宿主机 IP 加宿主机映射端口，例如 `宿主机IP:9090`，前提是网络和防火墙允许访问。

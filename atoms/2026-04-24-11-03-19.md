# 为什么同一台 VM 在 ESXi 和 vCenter 下 MOID 不同

### 现象

- **直连 ESXi** MOID 是纯数字，如 `"110"`（由 ESXi 自己分配）。

- **vCenter 管理** MOID 是 `"vm-4012"` 这种带前缀格式（由 vCenter 重新分配）。

- **切换风险** 同一台物理 VM 从 ESXi 直连切到 vCenter 接入，MOID 会变。

### 本质：MOID 是管理实例分配的

- **不是 VM 自身属性** MOID 不是物理 VM 本身的属性，而是管理系统为了方便自己管理而分配的“内部编号”。

- **类比** 同一个人在不同医院的“患者编号”不同——北京医院编 `"110"`，上海医院编 `"vm-4012"`，人的 DNA 没变，但编号不同。

### ESXi 直连的编号系统

- **范围** 只管理本主机上的虚拟机，最多几百个 VM。

- **格式** 纯数字，从 1, 2, 3… 依次分配。

- **示意**

    ```
    ┌─────────────────────┐
    │  单个 ESXi 主机      │
    ├─────────────────────┤
    │ 虚拟机 MOID="110"   │
    │ 虚拟机 MOID="111"   │
    │ 虚拟机 MOID="112"   │
    └─────────────────────┘
    ```

### vCenter 的编号系统

- **范围** 跨所有 ESXi 主机统一编号，管理成千上万的 VM。

- **格式** 带前缀（如 `"vm-"`），便于识别对象类型和来源，编号空间可支持数万个 VM。

- **示意**

    ```
    ┌──────────────────────────────────────┐
    │  vCenter 服务器（统一管理中心）         │
    ├──────────────────────────────────────┤
    │ ┌──────────────┐  ┌──────────────┐   │
    │ │ ESXi Host-1  │  │ ESXi Host-2  │   │
    │ ├──────────────┤  ├──────────────┤   │
    │ │ vm-4012      │  │ vm-4013      │   │
    │ │ vm-4014      │  │ vm-4015      │   │
    │ └──────────────┘  └──────────────┘   │
    └──────────────────────────────────────┘
    ```

### 切换时 MOID 变化的过程

1. 直连 ESXi 时，ESXi 给 VM 分配 MOID = `"110"`，API 调用使用此编号。
2. 加入 vCenter 后，vCenter 接入该 ESXi 主机，扫描发现虚拟机。
3. vCenter 在自己的数据库里重新编号，分配 MOID = `"vm-4012"`。

- **重新编号的原因**

    | 角度 | 说明 |
    | --- | --- |
    | 编号体系不同 | ESXi 用简单数字，vCenter 用带前缀格式 |
    | 作用域不同 | ESXi 编号只在本主机有效，vCenter 需要全局唯一 |
    | 数据库不同 | vCenter 有自己的中央数据库，需按自己的规则重新编号 |

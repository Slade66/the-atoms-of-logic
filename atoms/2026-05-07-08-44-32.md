# `.cat` 文件是什么

## 一、基本定义

- **文件含义** `.cat` 是 Security Catalog，也就是安全目录文件，本质上是一个数字签名容器。

- **核心作用** `.cat` 文件是驱动包的防篡改封条和身份证，Windows 靠它确认驱动来自可信来源，并且文件没有被人改过。

## 二、解决的问题

- **签名成本** 驱动包里通常包含 `.inf`、`.sys`、`.dll` 等多个文件，如果微软或厂商对每个文件单独签名，流程会很繁琐，也难管理。

- **目录签名** `.cat` 的思路是先计算每个文件的哈希值，把这些哈希值收集到一份清单里，再对整份清单做数字签名。

- **防伪标签** 可以把 `.cat` 理解成驱动包上的防伪标签：它记录包里每个文件的“指纹”，这张标签本身再由微软或可信机构盖章。

- **清单内容** 用工具打开 `.cat` 文件，可以看到它记录了驱动包中各文件的哈希值，并对整份清单做签名。

    ```text
    driver.inf    -> SHA256: a1b2c3d4...
    driver.sys    -> SHA256: e5f6a7b8...
    driver.dll    -> SHA256: 12345678...
    [以上整体]   -> 由 Microsoft 签名
    ```

## 三、验证流程

- **判断逻辑** Windows 安装驱动时，实际要判断两个问题：这个驱动是不是正规厂商发布的，驱动包里的文件有没有被偷偷改过。

- **安装检查** Windows 会先验证 `.cat` 文件本身的数字签名，再用 `.cat` 里的哈希值校验驱动包中的实际文件。

    1. Windows 拿到 `.cat` 文件。
    2. 验证 `.cat` 上的数字签名是否可信。
    3. 如果签名不可信，拒绝安装或弹出警告。
    4. 如果签名可信，取出 `.cat` 里记录的各文件哈希值。
    5. 逐一计算驱动包中实际文件的哈希值。
    6. 如果哈希不一致，说明文件被篡改，拒绝安装。
    7. 如果哈希一致，说明驱动包完整可信，允许继续安装。

## 四、安全要求

- **强制签名** 从 Windows 10 1607 开始，64 位系统要求内核态驱动 `.sys` 必须经过微软签名，否则默认无法加载。

- **防护目标** 强制校验 `.cat` 是为了减少内核级风险。

    - 防止恶意软件伪装成驱动进入内核。
    - 防止驱动被中间人篡改后仍能安装。
    - 防止内核级 rootkit 植入。

- **缺失后果** 如果没有 `.cat`，Windows 无法判断驱动是否可信，通常会弹出“无法验证此驱动程序软件的发布者”，64 位系统也可能直接拒绝加载。

- **测试模式** 可以通过 `bcdedit /set testsigning on` 开启测试模式绕过签名验证，但这会降低系统安全性，不适合正式环境。

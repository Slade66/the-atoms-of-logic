# 华为 Eth-Trunk 链路聚合

## 一、是什么

- **技术称呼：** Eth-Trunk 是华为设备上对链路聚合（Link Aggregation）技术的称呼。

- **核心思想：** 把多条物理以太网链路捆绑成一条逻辑链路来使用，即“把几条线绑成一条用”。

- ![[eth_trunk_多条物理链路捆绑为一条逻辑链路.svg]]

## 二、为什么要用它

- **增加带宽：** 把多条链路捆成一个 Eth-Trunk，逻辑带宽近似各成员链路之和，省去直接上更贵高速接口的成本。

    - 例如 4 条千兆链路捆成一个 Eth-Trunk，逻辑上就有了接近 4Gbps 的带宽，而不用直接上万兆接口。

- **提高可靠性：** 捆绑后只要还有一条成员链路是好的，整个 Eth-Trunk 就不会断。

    - 流量按负载分担算法分散到各条成员链路上。单条链路故障时流量自动切到其他链路，对上层业务几乎无感知。

## 三、配置 Eth-Trunk

- **配置流程（手工模式，以 SwitchA 为例）：**

    1. 创建 Eth-Trunk 接口，默认即手工模式，无需额外指定。

    2. 把各物理口加入 Eth-Trunk 1。

    3. 配置负载分担方式（按源目的 IP 哈希）。

    ```text
    # ---------- SwitchA ----------
    [SwitchA] interface Eth-Trunk 1
    [SwitchA-Eth-Trunk1] quit                       # 默认就是手工模式，无需指定
    [SwitchA] interface GigabitEthernet 0/0/1
    [SwitchA-GigabitEthernet0/0/1] eth-trunk 1      # 把物理口加入 Eth-Trunk 1
    [SwitchA-GigabitEthernet0/0/1] quit
    [SwitchA] interface GigabitEthernet 0/0/2
    [SwitchA-GigabitEthernet0/0/2] eth-trunk 1
    [SwitchA-GigabitEthernet0/0/2] quit
    # 配置负载分担方式（按源目的 IP 哈希）
    [SwitchA] interface Eth-Trunk 1
    [SwitchA-Eth-Trunk1] load-balance src-dst-ip
    ```

- **对端配置：** SwitchB 端做完全对应的配置即可。

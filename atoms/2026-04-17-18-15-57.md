# SQL 删除表里的全部数据

- **DELETE：** 逐行删除。

    - **示例：** `DELETE FROM 表名;`

    - **特点：** 一行一行删除，可以回滚，事务安全；会触发 DELETE 触发器；删除速度较慢（大表不推荐）；不会重置自增 ID。

- **TRUNCATE：** 推荐清空整表使用，本质是重建表。

    - **示例：** `TRUNCATE TABLE 表名;`

    - **特点：** 直接清空整张表，速度非常快；不触发 DELETE 触发器；会重置自增 ID；一般不能回滚（DDL 操作）。

- **DROP：** 直接删表，表结构 + 数据全部删除。

    - **示例：** `DROP TABLE 表名;`

- **三者对比：**

    | 方法       | 删除数据 | 删除表结构 | 是否可回滚 | 速度 | 自增ID |
    | -------- | ---- | ----- | ----- | -- | ---- |
    | DELETE   | ✅    | ❌     | ✅     | 慢  | 不变   |
    | TRUNCATE | ✅    | ❌     | ❌（一般） | 快  | 重置   |
    | DROP     | ✅    | ✅     | ❌     | 很快 | —    |

- **怎么选？**

    - **想清空表数据（常用）：** 用 `TRUNCATE`。

    - **需要事务 / 条件删除：** 用 `DELETE`。

    - **表都不要了：** 用 `DROP`。

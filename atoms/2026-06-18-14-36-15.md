# 交换机堆叠（iStack）

## 一、什么是 iStack

- **定义：** iStack（Intelligent Stack，智能堆叠）是把多台盒式交换机用线缆连起来，逻辑上合并成一台设备。

- **物理连接两种方式：** 堆叠卡堆叠用专用插卡加专用线缆；业务口堆叠直接拿普通业务口（如 `10GE`/`40GE` 上行口）当堆叠口用。现在用得最多的是业务口堆叠，不用额外买卡。

- **逻辑上要规划四样东西：** 成员 ID（堆叠 ID）、优先级、域编号、堆叠端口。

## 二、成员 ID 与优先级

- **成员 ID 是什么：** 堆叠里每台成员的唯一“身份证号”，`renumber 1` 的字面意思就是“把当前的 0 号改编成 1 号”。

- **为什么要从 0 改成 1：** 一个堆叠里成员 ID 必须唯一，而出厂默认值相同（很多 S 系列默认都是 0），两台都是 0 号凑一起就撞车，堆叠组不起来，所以要手动编成 1、2、3……

- **它解决两个问题：** 一是让主控能区分这是堆叠里的第几台设备；二是成员 ID 就是端口名的第一段，编号唯一了端口名才不会重复。

- **优先级管的是另一件事：** ID 只管“身份编号”，优先级只管“谁当主”，两者无关。`priority 200 > 默认 100`，所以优先级高的那台争当 Master。

## 三、堆叠域（domain）

- **是什么：** 给堆叠贴的一个“组标签”，同一个堆叠里所有成员域号必须一致，不同堆叠用不同域号隔开。

- **解决什么问题：** 防止两套本不相干的堆叠，因线缆误连意外合并成一个。

- **多套堆叠共存的例子：**

    ```text
    堆叠A：SW1 + SW2   都设 domain 10
    堆叠B：SW3 + SW4   都设 domain 20

    域号不同     → 即便误连 A、B 的线，也不会合体
    域号相同+连通 → 才允许组成同一个堆叠
    ```

    - 若 SW2 和 SW3 被误连一根线，因 A 是域 10、B 是域 20，“暗号”对不上，设备直接拒绝把对方当成员，两套堆叠各过各的。

    - 反之若图省事两套都设域 10，这根误连的线就会让四台设备“认亲”合并成一个大堆叠，业务全乱套——这正是域要防的事。

- **本质是一道准入校验：** 设备之间先确认“我们是同一个域”，才肯认对方是自己人。只有一套堆叠时它没存在感，多套共存时它是防误并的关键。

## 四、逻辑堆叠口（stack-port）

- **口袋比喻：** 逻辑堆叠口本身不是物理口，而是一个“口袋 / 容器”，用来装真正连线的物理口，告诉设备“这几个口以后专职干堆叠，不再当普通业务口”。

    - `interface stack-port 1/1` 是进入这个口袋，`port interface 10GE 1/0/1 enable` 是把物理口塞进去。

    ```text
    逻辑堆叠口（口袋）          装进去的物理口（真正连线的口）
    ─────────────────────────────────────────────────
    成员1
      ├ stack-port 1/1  ◄──装──  10GE1/0/1  （还能再装 1/0/2 做冗余/扩带宽）
      └ stack-port 1/2  ◄──装──  10GE1/0/48
    ```

- **为什么是“逻辑”而非直接用物理口：** 一个逻辑堆叠口里可以捆多个物理口，多根线既叠加带宽、又互为备份（断一根不影响），思路和 Eth-Trunk 把多条链路捆成一条一样。

- **它身兼两职：** 一是当“控制通道”，堆叠建立、主备选举、拓扑发现、域和成员 ID 协商、配置与转发表同步、心跳保活都走它；二是当“数据通道”，跨成员转发的业务流量也经它传过去。它就是把多台物理设备粘成“一台”的纽带。

- **命名规则是“成员 ID / 堆叠口序号”：** 第一段是成员 ID 表示属于哪台设备（成员 1 都以 1 开头），第二段是堆叠口序号，每台成员固定只有 `1` 和 `2` 两个。

- **为什么每台固定两个口：** 要把多台设备串成环，每台中间成员都得连“左右两个邻居”，一个方向用 `x/1`、另一个用 `x/2`。

- **连线规则与拓扑：** 一台的 `stack-port x/1` 接相邻设备的 `stack-port x/2`，首尾相接就成了环。

    ```text
    成员1 [1/1]──[1/2] ── [2/1]成员2[2/2] ── [3/1]成员3[3/2]
      └───────────────────────────────────────────────┘
            成员3的 3/2 绕回接成员1的 1/1 → 闭环
    ```

    - 环形比链形可靠：断掉任意一段，流量还能反方向绕一圈到达，不中断。

    - 所以只拿出极少数口（常是高速上行口）当堆叠线，剩下几十个业务口照常接服务器、上联交换机。

## 五、两台设备合并成一个堆叠的条件

- **条件一（堆叠口）：** 两端口都配成堆叠物理成员口、并绑进逻辑堆叠口；这是前提，没有它后面都免谈。

- **条件二（拓扑）：** 用这些堆叠口互联、拓扑接对，一台的 `x/1` 接对端 `x/2`，串成链形或环形。

- **条件三（域相同）：** 域不同会被直接拒绝。

- **条件四（ID 不冲突）：** 两台都默认 0 号没改会撞 ID，导致组建失败或一方异常，必须先 `renumber` 成唯一号。

- **域只是准入校验里的一关，门本身是“这个口是不是堆叠口”：** 插在业务口上，域再一样也只是条普通网线；插在堆叠口上且其余条件都满足，才会真正合并。

## 六、配置业务口堆叠

- **配置命令（SW1 为主、拉高优先级，SW2 默认）：**

    ```text
    # ===== SW1（主，成员ID设为1，优先级拉高）=====
    system-view
    sysname SW1
    stack                                    # 进入堆叠配置视图
     stack member 0 renumber 1               # 给这台设备分配成员ID = 1
     stack member 1 priority 200             # 设成员1的优先级 = 200（争主）
     stack domain 10                         # 加入堆叠域 10
    quit
    interface stack-port 1/1                 # 进入"1号成员的1号逻辑堆叠口"
     port interface 10GE 1/0/1 enable        # 把物理口10GE1/0/1塞进这个堆叠口
    quit
    save

    # ===== SW2（成员ID设为2，优先级默认）=====
    system-view
    sysname SW2
    stack
     stack member 0 renumber 2
     stack member 2 priority 100
     stack domain 10
    quit
    interface stack-port 2/1
     port interface 10GE 2/0/1 enable
    quit
    save
    ```

- **三条关键命令的作用：** `renumber` 给每台设备一个唯一身份证号（顺带决定端口名第一段），`domain` 给这套堆叠贴组标签防误并，`stack-port` 是装互联线的口袋、只装那几根互联线对应的物理口。

## 七、堆叠后重命名

- **端口名自动重命名：** 堆叠后第一段从 0 变成成员 ID 是自动的，不用手动逐个改——只要给每台设备设一次成员 ID，系统会把它所有端口的第一段都改成该成员 ID。

- **重命名前后对照：**

    ```text
    堆叠前（两台都是独立设备，成员ID默认都是0）
       SW1: 10GE0/0/1、10GE0/0/2 …
       SW2: 10GE0/0/1、10GE0/0/2 …   ← 名字一模一样，会冲突

           ↓ 配好成员ID（SW1=1, SW2=2），重启堆叠

    堆叠后（一台逻辑设备，端口名自动加上成员ID前缀）
       原SW1的口 → 10GE1/0/1、10GE1/0/2 …
       原SW2的口 → 10GE2/0/1、10GE2/0/2 …
    ```

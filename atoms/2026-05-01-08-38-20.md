# Appsmith 如何把字段显示成下拉选择

### 一、问题背景

- **使用场景** 在 Appsmith 中展示借阅状态时，不希望用户直接输入 `PENDING`、`BORROWED` 这类状态值，而是希望通过下拉选择来展示和选择状态。

- **核心做法** 将字段的 `Field Type` 设置为 `Select`，再在 `Options` 中配置每个状态对应的显示文本和实际值。

### 二、配置字段类型

1. 找到需要展示状态的字段配置。

2. 将 `Field Type` 设置为 `Select`。

3. 进入 `Options` 配置项，填入状态选项数组。

### 三、配置 Select 的 Options

- **配置格式** `Options` 需要使用数组，每一项都包含 `label` 和 `value`。

- **显示文本** `label` 是页面上展示给用户看的中文状态。

- **实际值** `value` 是真正提交或保存到数据库里的状态枚举值。

    ```json
    {{
      [
        { "label": "待审核", "value": "PENDING" },
        { "label": "借阅中", "value": "BORROWED" },
        { "label": "已归还", "value": "RETURNED" },
        { "label": "已拒绝", "value": "REJECTED" },
        { "label": "已取消", "value": "CANCELLED" }
      ]
    }}
    ```

### 四、理解 label 和 value 的区别

- **label 面向用户** `label` 负责让页面更易读，例如显示“待审核”“借阅中”。

- **value 面向系统** `value` 负责和后端或数据库保持一致，例如保存 `PENDING`、`BORROWED`。

- **通用原则** 页面展示可以用中文，但数据传递尽量使用系统稳定识别的枚举值。

# Linux 系统是如何启动的？

1. 你按下开机键，主板通电。

2. 引导程序（Bootloader/GRUB）启动，它把操作系统的内核（Kernel）加载到内存里唤醒。

3. 内核醒了，它准备去读取硬盘里的文件，把整个系统跑起来。

4. 内核要读取硬盘，必须先有“硬盘驱动”；而“硬盘驱动”作为一个文件，偏偏平时又是存放在硬盘里的！

5. 大脑打不开保险箱（拿不到驱动），因为开保险箱的钥匙（驱动）就锁在保险箱里面。

6. 为了解开上面那个死结，Linux 设计了一个叫 initramfs（Initial RAM File System）的微型内存文件系统（你可以把它当成一个“随身急救包”，这个急救包里面装了最底层的磁盘控制器驱动）。
    
    1. 每次当你装完系统或者升级内核时，系统会把你当前电脑硬件最核心的几个“硬盘驱动”，打包成一个独立的压缩包（就是 /boot 目录下的 initramfs-xxx.img 文件）。
    
    2. 开机时，GRUB 引导程序会把大脑（内核）和这个小压缩包（initramfs）同时塞进内存。
    
    3. 内核醒来后，不需要访问真实的硬盘，直接先把这个放在内存里的微型压缩包解开，从里面拿出了当前硬盘的驱动
    
    4. 用这把钥匙，内核成功认出了外面的大硬盘，并挂载了真实的根目录。危机解除，开机成功！

- 在 RedHat/CentOS 体系，打包这个急救包的工具叫 dracut；在 Ubuntu/Debian 体系，打包工具叫 update-initramfs。

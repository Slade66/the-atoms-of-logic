# Go 的模板渲染

## 一、两个模板渲染包

- **text/template：** 渲染纯文本，只负责文本替换和逻辑控制，不做任何转义；配置文件、代码生成等不需要转义的场景用它。

- **html/template：** 渲染 HTML，接口和 `text/template` 几乎一样，但会自动做转义，可防止 XSS 注入；输出给浏览器的 HTML 应该用它。

## 二、模板渲染的核心流程

- **三步：** 创建 → 解析 → 执行。

    1. 创建：`New`

    2. 解析：`Parse`

    3. 执行：`Execute`

- **示例：**

    ```go
    package main

    import (
    	"os"
    	"text/template"
    )

    func main() {
    	// 1. 创建一个名叫 "demo" 的模板
    	t := template.New("demo")
    	// 2. 解析模板字符串，{{.}} 代表传进来的数据
    	t, _ = t.Parse("Hello, {{.}}!\n")
    	// 3. 执行，把数据填进去，输出到 os.Stdout
    	t.Execute(os.Stdout, "World")
    }
    ```

- **`{{.}}` 里的点是“当前数据”：** 执行时传进来的是字符串 `"World"`，所以这个点就是它。

## 三、用 Must 包裹解析

- **Must 是什么：** 一个辅助函数，包裹任何返回 `(*template.Template, error)` 的调用——解析报错就直接 panic，否则返回模板本身。

- **作用是省去每次的 err 判断：** 不用每次解析都写 `if err != nil`。

    ```go
    // 不用 Must：要接收并判断 error
    t, err := template.New("demo").Parse("Hello, {{.}}!\n")
    if err != nil {
    	panic(err)
    }

    // 用 Must：一行搞定
    t := template.Must(template.New("demo").Parse("Hello, {{.}}!\n"))
    ```

- **为什么这样惯用：** 模板字符串通常是写死在代码里的常量，解析失败属于程序员的错误而非运行时的外部输入，所以用 Must 在程序启动时就暴露问题，是惯用做法。

## 四、传 struct 或 map，用 `.字段名` 取字段

- **取值方式：** 不管传的是结构体还是 map，都在模板里用 `.字段名` 取值。

- **结构体字段必须导出（首字母大写）：** 否则模板访问不到。

    ```go
    type User struct {
    	Name string
    	Age  int
    }

    func main() {
    	t := template.Must(template.New("u").Parse("{{.Name}} is {{.Age}} years old.\n"))
    	t.Execute(os.Stdout, User{Name: "Tom", Age: 18})
    }
    ```

- **map 的 key 不要求大写：**

    ```go
    t := template.Must(template.New("m").Parse("{{.name}}\n"))
    t.Execute(os.Stdout, map[string]string{"name": "Jerry"})
    ```

## 五、条件判断 if

- **语法：** `{{if 条件}} ... {{else}} ... {{end}}`。

    ```go
    const tpl = `{{if .VIP}}尊贵的会员{{else}}普通用户{{end}}：{{.Name}}
    `
    t := template.Must(template.New("if").Parse(tpl))
    t.Execute(os.Stdout, struct {
    	Name string
    	VIP  bool
    }{Name: "Tom", VIP: true})
    ```

- **空值即为假：** `false`、`0`、空字符串、空切片、空 map、`nil` 都算假，其余为真。

## 六、循环 range

- **语法：** `{{range .Items}} ... {{end}}`，循环体里的 `{{.}}` 是当前项。

    ```go
    const tpl = `{{range .Items}}- {{.}}
    {{end}}`
    t := template.Must(template.New("r").Parse(tpl))
    t.Execute(os.Stdout, struct{ Items []string }{
    	Items: []string{"苹果", "香蕉", "橙子"},
    })
    // {{.}} 打印的是每个水果
    ```

- **range 会改变点的指向：** 进入 range 后，循环体内的点 `.` 不再是顶层数据，而是“当前这一项”。

- **同时拿索引和值，写 `{{range $i, $v := .Items}}`：**

    ```go
    const tpl = `{{range $i, $v := .Items}}{{$i}}: {{$v}}
    {{end}}`
    ```

- **循环内访问外层字段靠 `$`：** 点已被当前项占用，`$` 始终指向顶层数据。

    ```go
    const tpl = `{{range .Items}}{{$.Title}} - {{.}}
    {{end}}`
    // $.Title 拿的是顶层的 Title，. 拿的是当前项
    ```

## 七、with 切换作用域

- **作用：** 如果值非空，就把点 `.` 切换成这个值，省得在块内反复写长路径。

    ```go
    type Address struct {
    	City   string
    	Street string
    }
    type User struct {
    	Name    string
    	Address Address
    }

    const tpl = `{{.Name}} 住在 {{with .Address}}{{.City}}{{.Street}}{{end}}`
    t := template.Must(template.New("w").Parse(tpl))
    t.Execute(os.Stdout, User{Name: "Tom", Address: Address{City: "北京", Street: "长安街"}})
    // with 块内点已切换成 .Address，所以直接写 .City、.Street，不必写 .Address.City
    ```

- **值为空时跳过块：** 和 if 一样，`.Address` 为空时整个 `with` 块不渲染，也可带 `{{else}}` 处理空的情况。

## 八、管道与函数

- **函数调用写法：** `{{函数 参数1 参数2}}`，空格分隔，不用括号。

    ```go
    const tpl = `{{printf "%s-%d" .Name .Age}}`
    t := template.Must(template.New("fn").Parse(tpl))
    t.Execute(os.Stdout, struct {
    	Name string
    	Age  int
    }{Name: "Tom", Age: 18})
    // 输出：Tom-18
    ```

- **管道 `|`：** 把前一个的结果作为后一个函数的最后一个参数传进去。

    ```go
    const tpl = `{{.Name | printf "%s!"}}`
    t := template.Must(template.New("pipe").Parse(tpl))
    t.Execute(os.Stdout, struct{ Name string }{Name: "Tom"})
    // 等价于 printf "%s!" .Name，输出：Tom!
    ```

- **嵌套调用要用括号：** 想把一个函数调用的结果作为另一个函数的参数，必须用括号把内层括起来，否则参数会被摊平传给外层。

    ```go
    {{printf "%d" (len .Items)}}   // 正确：先算 len .Items，再把结果传给 printf
    {{printf "%d" len .Items}}     // 错误：printf 会收到 len、.Items 三个参数
    ```

- **常见内置函数：** 下面示例假设传入数据为 `.List = []int{10, 20, 30}`、`.A = 1`、`.B = 2`。

    - `len`（长度）：`{{len .List}}` → `3`

    - `index`（按下标取值）：`{{index .List 0}}` → `10`

    - `printf`（同 `fmt.Sprintf`）：`{{printf "%d/%d" .A .B}}` → `1/2`

    - `print` / `println`（同 `fmt.Sprint` / `Sprintln`）：`{{println .A .B}}` → `1 2`（末尾带换行）

    - `eq`（等于）：`{{eq .A .B}}` → `false`

    - `ne`（不等）：`{{ne .A .B}}` → `true`

    - `lt`（小于）：`{{lt .A .B}}` → `true`

    - `le`（小于等于）：`{{le .A .B}}` → `true`

    - `gt`（大于）：`{{gt .A .B}}` → `false`

    - `ge`（大于等于）：`{{ge .A .B}}` → `false`

## 九、自定义函数

- **本质：** 模板里的函数就是普通的 Go 函数，模板引擎用一个名字把它登记下来，让你在 `{{ }}` 里能调用。

- **用 FuncMap 注册：** 自定义函数靠 `FuncMap`（本质是 `map[string]interface{}`）注册。

- **`Funcs` 必须在 `Parse` 之前：** 解析阶段就要识别函数名，注册晚了模板会报 `function not defined`。

    ```go
    func main() {
    	funcs := template.FuncMap{
    		"upper": strings.ToUpper,
    		"add":   func(a, b int) int { return a + b },
    	}
    	t := template.Must(
    		template.New("f").Funcs(funcs).Parse("{{upper .Name}} {{add .Age 1}}\n"),
    	)
    	t.Execute(os.Stdout, User{Name: "tom", Age: 18})
    }
    ```

- **返回值只能是 1 个或 2 个：** 返回 1 个就是结果；返回 2 个时第二个必须是 `error`，返回非 nil 的 error 时 `Execute` 会中断并把这个错误返回给你。其它形式（比如返回三个值）注册时不报错，但执行调用时会 panic 或报错。

- **也可直接调数据自身的方法：** 传进去的结构体若有方法，模板里直接写 `{{.Greet}}` 就能调，无需注册。

    ```go
    type User struct{ Name string }

    func (u User) Greet() string { return "Hi, " + u.Name }

    // 模板里直接写 {{.Greet}}，不需要注册
    ```

    - 区别在于：方法跟着数据走，只能在“点指向这个类型”时调用且无需注册；函数是全局注册的，整个模板任何地方都能用。

## 十、模板复用：define / template / block

- **场景：** 当模板需要拆分复用。

- **函数复用“逻辑”，子模板复用“结构”：** 函数把重复的计算（格式化、判断、转换）收进 `FuncMap`，子模板把重复的版式（页头页尾、卡片、列表项）收进 `define` / `block`。

### define：定义一个具名子模板

- **作用：** `{{define "名字"}} ... {{end}}` 把一段模板登记成一个有名字的块，但它本身不会输出，要等被调用才执行。

    ```go
    const tpl = `
    {{define "greeting"}}你好，{{.}}！{{end}}
    正文开始
    `
    // 光定义不调用，greeting 这段不会出现在输出里
    ```

### template：调用子模板并传数据

- **作用：** `{{template "名字" 数据}}` 去执行那个具名块；第二个参数是传给子模板的数据，子模板内部的点就是它。

    ```go
    const tpl = `
    {{define "greeting"}}你好，{{.}}！{{end}}
    {{template "greeting" .Name}}
    今天的任务：{{.Task}}
    `
    t := template.Must(template.New("main").Parse(tpl))
    t.Execute(os.Stdout, struct {
    	Name, Task string
    }{Name: "Tom", Task: "写接口"})
    ```

- **不传数据时子模板的点是 nil：** 第二个参数省略，子模板内部的 `.` 就是 nil。

- **想把外层全部数据传给子模板：** 常用写法是 `{{template "greeting" .}}`，把当前的点整个传进去。

### block：定义 + 立即执行 + 可覆盖

- **作用：** block 等于“定义一个子模板，并马上在原地执行它”，同时它提供的内容是“默认实现”，可被同名的 define 覆盖。

    ```go
    {{block "content" .}}默认内容{{end}}
    // 等价于：先 define "content" 为“默认内容”，再 template "content" .
    ```

- **示例（layout 里的 body 被同名 define 覆盖）：**

    ```go
    const tpl = `
    {{define "layout"}}
    =========== 页头 ===========
    {{block "body" .}}这里是默认正文{{end}}
    =========== 页尾 ===========
    {{end}}

    {{define "body"}}用户 {{.Name}} 的专属正文{{end}}

    {{template "layout" .}}
    `
    t := template.Must(template.New("page").Parse(tpl))
    t.Execute(os.Stdout, struct{ Name string }{Name: "Tom"})
    ```

- **渲染流程（数据 `.` 如何一路透传到 block）：**

    1. 执行根模板 `page`，传入 `{Name: "Tom"}`，此时顶层的点 `.` 就是这份数据。

    2. 根模板里的 `{{template "layout" .}}`：把顶层的 `.` 整个传给子模板 `layout`，于是 `layout` 内部的 `.` 也是 `{Name: "Tom"}`。

    3. `layout` 里的 `{{block "body" .}}`：再把 `layout` 当前的 `.` 往下传给名为 `body` 的块。又因为顶层存在同名的 `{{define "body"}}`，它覆盖掉 block 自带的默认正文“这里是默认正文”，所以真正执行的是这个 define，传进去的点仍是 `{Name: "Tom"}`。

    4. `define "body"` 里的 `{{.Name}}`：点是 `{Name: "Tom"}`，取出 `Name`，渲染成“用户 Tom 的专属正文”。

    5. 最终输出：页头 +“用户 Tom 的专属正文”+ 页尾。

    > 关键：每跨一层（`template` → `block` → 被覆盖的 `define`）都靠显式写的那个 `.` 把数据往下递；哪一步漏写 `.`，下层的点就变 nil。

## 十一、空白控制

- **问题：** 模板里的换行和缩进会原样输出，导致结果里多出很多空行。

- **用 `{{-` 和 `-}}` 吃掉左右空白（包括换行）：**

    ```go
    const tpl = `{{range .Items -}}
    {{.}}
    {{- end}}`
    ```

    - `-}}` 会删掉它右边到下一个非空白字符之间的空白。

    - `{{-` 会删掉它左边的空白。

- **注释写成 `{{/* 这是注释 */}}`：** 不会出现在输出里。

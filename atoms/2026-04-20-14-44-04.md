# Bootloader（引导程序）：决定启动哪个系统

- **定义：** Bootloader 是一段装在硬盘里、体积几百 KB 到几 MB 的代码（不是硬件、不是芯片），开机时被 BIOS 叫醒，负责把操作系统内核从硬盘搬到内存、配好运行环境、跳过去让内核接管，然后退场。俗称"开机时的搬家工人"。

### 为什么需要 Bootloader？

- **核心矛盾：** CPU 只会执行内存里某个固定地址的指令，但操作系统是硬盘上几 GB 的文件，而刚通电时内存是空的——必须有人先把操作系统从硬盘搬进内存，CPU 才有东西可执行。

- **鸡生蛋：** 这件事 Windows 自己干不了（它还没启动呢），CPU 也干不了（它不认识文件系统）——所以必须有个"中间人"来搬，这就是 Bootloader。

### Bootloader 和 MBR 是啥关系？

- **MBR 里的 446 字节就是 bootloader 的"第一小段"。** 但 446 字节实在太小，装不下完整的搬家逻辑，所以 bootloader 被拆成几段接力：

    ```
    第一小段（住在 MBR 里，446 字节）
        ↓ 我太小了，啥也干不了，我去叫第二段
    第二小段（住在硬盘某个角落）
        ↓ 我稍微大点了，但还是有限，我去叫主程序
    主程序（住在硬盘文件系统里，好几 MB）
        ↓ 我才是真正的 bootloader！我来搬操作系统！
    操作系统内核
    ```

- **"GRUB" / "Windows Boot Manager" 指的是最后那个主程序，** 前面两小段只是跳板。所以 "bootloader" 这个词其实指的是一整套接力过程，不是单个东西。

### Bootloader 的核心工作

- **显示启动菜单，让用户做选择：** 就是那个黑底白字的菜单（`Ubuntu / Windows / 高级选项...`）。

    - **选哪个系统：** Ubuntu / Windows。
    - **选哪个内核版本：** 升级后能回滚旧版本。

- **加载内核到内存：** 你选一项后，它去硬盘上把 Linux 内核（`vmlinuz`）读进内存。

- **准备内核运行环境：** 加载 `initramfs`（一个临时文件系统，里面装着挂载根分区必需的驱动）。

- **把控制权交给内核：** 跳到内核入口，bootloader 退场。

### 常见的 Bootloader 有哪些？

| Bootloader | 平台 | 特点 |
| --- | --- | --- |
| GRUB | Linux | 支持几乎所有文件系统、带菜单、能跨盘、脚本化 |
| Windows Boot Manager（`bootmgr`） | Windows | Win Vista 以后的默认 |

### Bootloader 住在哪？（以 GRUB 为例）

```
┌──────────────────── 一块硬盘 ────────────────────┐
│                                                  │
│  LBA 0       ┌─ MBR (512B) ────────────────┐    │
│              │  446B: GRUB stage 1          │    │
│              │   64B: 分区表                │    │
│              │    2B: 55 AA                 │    │
│              └──────────────────────────────┘    │
│                                                  │
│  LBA 1~2047  ┌─ 保留区 (约 1MB) ────────────┐    │
│              │  GRUB stage 1.5              │    │
│              │  （"MBR 后隙 / MBR gap"）     │    │
│              └──────────────────────────────┘    │
│                                                  │
│  LBA 2048+   ┌─ 分区 1 (/boot 或 /) ────────┐    │
│              │  /boot/grub/grub.cfg         │    │
│              │  /boot/grub/i386-pc/*.mod    │    │
│              │  /boot/vmlinuz-6.x.x         │    │
│              │  /boot/initrd.img-6.x.x      │    │
│              └──────────────────────────────┘    │
└──────────────────────────────────────────────────┘
```

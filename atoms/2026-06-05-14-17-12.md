# Go 的 fmt.Errorf

fmt.Errorf = 造一个带格式化消息的 error。关键在两个占位符:

┌────────┬────────────┬─────────────────────────────────────────────────────────────────────────────────────────────────────┐
  │ 占位符 │    作用    │                                               大白话                                                │
  ├────────┼────────────┼─────────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ %w     │ wrap(包裹) │ 把 ErrPlanExportFailed 这个哨兵错误「链」进去,让上层能用 errors.Is(err, ErrPlanExportFailed) 认出来 │
  ├────────┼────────────┼─────────────────────────────────────────────────────────────────────────────────────────────────────┤
  │ %v     │ 普通格式化 │ 只是把 err 的文字拼进消息,不建立可识别的链                                                          │
  └────────┴────────────┴─────────────────────────────────────────────────────────────────────────────────────────────────────┘

  所以这一行做了两件事:
  1. 消息 = "规划导出失败: <底层错误的文字>"(给人看)。
  2. 可识别 = 包裹了 ErrPlanExportFailed,上层 errors.Is(返回值, ErrPlanExportFailed) 为 true(给程序判断用)。

为什么一个用 %w 一个用 %v:
  - 你想被识别的那个错误(哨兵 ErrPlanExportFailed)用 %w。
  - 底层细节 err 只想把文字带上、不需要被 errors.Is 匹配,用 %v 就够。

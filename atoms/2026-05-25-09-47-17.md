# Git Tag

## 一、核心作用

- **版本标记** `tag` 可以给某个 `commit` 打固定标签，通常用于标记版发布版本；以后只需要用 `v1.0.0` 这类标签，就能定位到对应的 `commit`。CI/CD 看到 commit 打了 tag 后，可能会自动构建。

## 二、示例理解

- **提交记录** 在 Git 里，`commit` 是一条提交记录。

    ```text
    a1b2c3d 修复登录 bug
    d4e5f6g 完成订单接口
    h7i8j9k 发布 v1.0.0
    ```

- **版本对应** 如果 `h7i8j9k` 对应正式版本 `v1.0.0`，就可以给它打上 `v1.0.0` 这个 `tag`。

- **定位方式** 以后不需要记住 `h7i8j9k` 这个 `commit hash`，只需要使用 `v1.0.0` 就能定位到对应的 `commit`。

## 三、Tag 与 Branch 的区别

- **分支移动** `branch` 是一个会随着新提交不断前进的指针，通常指向某条开发线的最新位置。

    ```text
    A --- B --- C --- D  main
    ```

    当继续提交后，`main` 会从 `D` 移动到 `E`。

    ```text
    A --- B --- C --- D --- E  main
    ```

- **标签固定** `tag` 通常是不动的，适合指向某个固定版本点。

    ```text
    A --- B --- C --- D --- E  main
                |
             v1.0.0
    ```

    即使后面继续提交到 `D`、`E`，`v1.0.0` 仍然指向 `C`。

- **核心区别** `branch` 指向开发线的最新位置，通常会移动；`tag` 指向固定版本点，通常不应该移动。

## 四、Git Tag 命令

- **查看标签** 使用 `git tag` 查看本地所有 `tag`，使用 `git show` 查看某个 `tag` 的详细信息。

    ```bash
    git tag
    git show v1.0.0
    ```

- **创建轻量标签** 使用 `git tag <tag>` 给当前 `HEAD` 打一个轻量 `tag`；它本质上是一个简单指针，指向某个 `commit`。

    ```bash
    git tag v1.0.0
    ```

- **创建附注标签** 使用 `git tag -a` 创建附注 `tag`，并用 `-m` 写入标签说明。

    ```bash
    git tag -a v1.0.0 -m "Release v1.0.0"
    ```

    - `-a` 表示 `annotated`
    - `-m` 表示 `tag message`

- **标记历史提交** 给历史 `commit` 打 `tag` 时，先用 `git log --oneline` 找到目标 `commit hash`。

    ```bash
    git log --oneline
    ```

    ```text
    e111111 add payment api
    d222222 fix login bug
    c333333 release version 1.0.0
    b444444 add user api
    a555555 init project
    ```

    然后给 `c333333` 打上附注 `tag`。

    ```bash
    git tag -a v1.0.0 c333333 -m "Release v1.0.0"
    ```

- **删除标签** 使用 `git tag -d` 删除本地标签，使用 `git push origin :refs/tags/<tag>` 或 `git push origin --delete refs/tags/<tag>` 删除远程标签。

    ```bash
    git tag -d v0.1.0
    git push origin :refs/tags/v0.1.0
    git push origin --delete refs/tags/v1.0.0
    ```

- **推送标签** `git push` 默认不会自动推送 `tag`；即使执行 `git push origin main`，也只会推送 `main` 分支，不会推送本地新建的 `tag`。

    ```bash
    git push origin main
    git push origin v1.0.0
    git push origin --tags
    ```

    - `git push origin v1.0.0` 推送某一个 `tag`
    - `git push origin --tags` 推送所有本地 `tag`

- **切换标签** 使用 `git checkout <tag>` 可以查看某个版本的代码。

    ```bash
    git checkout v1.0.0
    ```

## 五、Tag 命名规范

- **语义化版本** 最常见的 `tag` 命名方式是语义化版本，格式是 `v主版本号.次版本号.修订号`。

    ```text
    v1.0.0
    v1.1.0
    v1.1.1
    v2.0.0
    ```

- **版本含义** `major` 表示破坏性变更，`minor` 表示兼容旧版本的新增功能，`patch` 表示兼容旧版本的错误修复。

    ```text
    v1.0.0
    │ │ │
    │ │ └── patch：修复 bug，兼容旧版本
    │ └──── minor：新增功能，兼容旧版本
    └────── major：破坏性变更，不兼容旧版本
    ```

- **命名示例** 不同版本号对应不同发布含义。

    ```text
    v1.0.0：第一个稳定版本
    v1.1.0：新增功能，但兼容 v1.0.0
    v1.1.1：修复 bug
    v2.0.0：接口发生破坏性变化
    ```

## 六、使用注意

- **避免直接开发** 不要直接在 `tag` 上开发；如果要基于某个 `tag` 修复问题，应该先从 `tag` 创建一个分支。

    ```bash
    git switch -c hotfix-v1.0.1 v1.0.0
    ```

# Appsmith Table 组件如何给每一行添加操作按钮

### 一、问题背景

- **使用场景** 在 Appsmith 的 Table 组件里，需要给每一行数据添加一个“同意”按钮，让用户可以直接在表格行内审批借阅申请。

- **核心问题** 行内按钮不能写死某一个 `id`，必须在点击时拿到当前这一行的数据，再把当前行的 `id` 传给后端 SQL。

- **关键变量** Appsmith Table 的行内按钮可以通过 `data_table.triggeredRow.id` 读取当前被点击行的 `id`。

### 二、创建审批借阅的 SQL 查询

- **查询目标** `approve_loan` 负责把当前行的借阅申请从 `PENDING` 改成 `BORROWED`，同时写入借阅时间、到期时间，并扣减对应图书的可借库存。

- **当前行定位** 这个 SQL 的关键是 `data_table.triggeredRow.id`，它表示用户点击“同意”按钮时所在行的借阅记录 `id`。

1. 新建一个 SQL 查询，命名为 `approve_loan`。

2. 填入审批借阅的更新 SQL。

    ```sql
    UPDATE loans l
    JOIN books b ON b.id = l.book_id
    SET
      l.status = 'BORROWED',
      l.borrow_time = NOW(),
      l.due_time = DATE_ADD(NOW(), INTERVAL 30 DAY),
      l.return_time = NULL,
      b.available_quantity = b.available_quantity - 1
    WHERE l.id = {{data_table.triggeredRow.id}}
      AND l.status = 'PENDING'
      AND b.available_quantity > 0;
    ```

- **条件保护** `WHERE` 部分同时做了 3 层限制，避免按钮误操作或重复操作。

    - `l.id = {{data_table.triggeredRow.id}}` 只处理当前点击按钮的这一行借阅记录

    - `l.status = 'PENDING'` 只允许审批待处理的借阅申请，避免重复审批

    - `b.available_quantity > 0` 只有图书还有可借库存时，才允许审批通过

### 三、在 Table 中添加 Approve 按钮列

1. 进入 `data_table` 的列配置。

2. 新增一个 `Approve` 列。

3. 将 `Column type` 设置为 `Button`。

4. 将按钮 `Label` 设置为“同意”。

5. 在按钮的 `onClick` 中选择执行 SQL 查询。

### 四、配置按钮点击后执行 SQL 查询

1. 在 `onClick` 中选择 `Action`。

2. 选择 `Execute a query`。

3. 选择执行 `approve_loan.run`。

4. 在 `On success` 中配置成功提示。

    ```text
    Show Alert -> Message: XXX -> Type: Success
    ```

5. 在 `On failure` 中配置失败提示。

    ```text
    Show Alert -> Message: XXX -> Type: Error
    ```

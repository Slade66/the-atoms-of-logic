# FusionCompute 三大组件：CNA、VRM、Hypervisor

### CNA：计算节点代理

- **全称** Computing Node Agent，部署在需要虚拟化的物理服务器上。

- **职责** 安装在每台物理服务器上的虚拟化软件，负责运行虚拟机，做所有的虚拟化工作（计算、存储、网络）。

### VRM：虚拟资源管理

- **全称** Virtual Resource Management，可部署在 VM 或物理服务器上。

- **职责** 向下管理 CNA 节点形成的虚拟化资源，向上提供 Web 图形界面（FusionCompute 界面）。

- **容灾** VRM 挂掉只会使管理界面不可用，底层仍可通过命令行管理调度虚拟机。

### Hypervisor：虚拟化引擎

- **定位** 在底层硬件之上加一层虚拟化层。

- **演进** FusionCompute 6.1 之前使用 Xen，6.1 之后改为 KVM。

### 类比

- **CNA = 生产车间** 每个车间有生产设备（CPU、内存），工人可同时做多个任务（虚拟机）。

- **VRM = 工厂办公室** 管理人员在此分配任务给各车间，监控各车间工作情况。

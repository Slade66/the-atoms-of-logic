# 在 macOS 上用 Homebrew 安装与配置 MySQL

## 一、安装

- **首选 Homebrew：** 在 Mac 上安装 MySQL 最常用的方式是用 Homebrew，对开发来说也最方便管理。

- **两种版本：** 装最新版还是装 LTS，按需选择。

    ```bash
    # 安装最新版（目前是 9.x 系列）
    brew install mysql

    # 安装 8.4 LTS（长期支持版，生产环境更稳妥）
    brew install mysql@8.4
    ```

## 二、服务管理

- **用 `brew services` 管理：** 注册为后台服务，可开机自启，改配置后能一键生效。

    ```bash
    # 启动并设置开机自启
    brew services start mysql@8.4

    # 停止
    brew services stop mysql@8.4

    # 重启（改了 my.cnf 配置后用它让配置生效）
    brew services restart mysql@8.4

    # 查看所有 brew 服务的运行状态
    brew services list
    ```

- **临时启动：** 只启动一次、不开机自启，用 `mysql.server start`。

## 三、初始化与连接验证

- **安全初始化：** 安装后运行 `mysql_secure_installation` 做加固，包括设置 root 密码、移除匿名用户等。

- **连接验证：** 用 `mysql -u root -p` 连库验证。

## 四、默认账号配置

- **整体很保守：** `brew install mysql@8.4` 装完后默认配置很安全，核心就两点——root 没密码、只能本机连。

- **root 默认无密码：** Homebrew 装的 MySQL 默认没有 root 密码，会提示运行 `mysql_secure_installation` 加固。第一次连库直接 `mysql -u root` 就能进，不需要密码。

- **账号 = 用户名 + 来源主机：** 这个无密码 root 的完整身份是 `'root'@'localhost'`。MySQL 的账号由「用户名 + 来源主机」共同构成，`'root'@'localhost'` 和 `'root'@'%'` 是两个不同的账号。默认只存在 localhost 这一个，意味着 root 天生就只能从本机连。

- **走 Unix socket 而非 TCP：** 默认连接走 Unix socket（`/tmp/mysql.sock`）而不是 TCP，这也是本机不输密码也能进的原因——它信任的是「你能访问这台机器的这个 socket 文件」这件事。

## 五、默认无法远程连接

- **默认拒绝远程：** Homebrew 的默认配置不接受任何远程连接，原因有两层。

    1. **网络监听层：** 默认的 `my.cnf`（在 `/opt/homebrew/etc/my.cnf`）通常把 `bind-address` 设成 `127.0.0.1`，即 mysqld 只监听本地回环网卡，外部请求根本到不了端口 `3306`。
    2. **账号来源层：** 就算网络通了，默认也只有 `'root'@'localhost'` 这个账号，没有任何允许从其它主机连入的账号，远程请求会被「没有这个用户」直接挡掉。

- **结论：** 默认状态下，MySQL 是一个只服务本机的数据库。

## 六、开启远程连接

- **第一步改监听地址：** 编辑 `/opt/homebrew/etc/my.cnf`，在 `[mysqld]` 段把 `bind-address` 改成 `0.0.0.0`（监听所有网卡），然后 `brew services restart mysql@8.4` 重启。

- **第二步创建专用远程账号：** 别直接放开 root，单独建一个远程账号。

    ```sql
    -- 创建一个允许从任意主机连入的账号（% 表示任意主机）
    CREATE USER 'dev'@'%' IDENTIFIED BY 'YourStrongPassword';
    GRANT ALL PRIVILEGES ON your_db.* TO 'dev'@'%';
    FLUSH PRIVILEGES;
    ```

- **收窄来源更安全：** `'dev'@'%'` 的 `%` 是通配符，表示允许从任何主机连。更安全的做法是把 `%` 换成具体的来源 IP。

## 七、认证插件说明

- **默认用 `caching_sha2_password`：** 8.4 默认认证插件是 `caching_sha2_password`，新建的账号也默认用它。

- **Go 驱动已原生支持：** `go-sql-driver/mysql` 较新版本已原生支持 `caching_sha2_password`，不需要像老教程那样把账号改回 `mysql_native_password`——而且在 8.4 上也改不回去，因为它默认被禁用了。把驱动升到较新版本、DSN 配好就能正常连。

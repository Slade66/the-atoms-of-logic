# 外观模式（Facade）

## 一、是什么

- **定义：** 结构型设计模式，在复杂子系统前加一层高层接口，把编排复杂度从调用方收拢到外观类内部——只简化接口、不增加新功能。

- **一句话：** 给复杂系统加的一层"傻瓜式遥控器"，对内封装复杂编排，对外简化使用入口。

## 二、解决什么问题

- **背景：** 子系统由很多类组成，调用关系复杂，客户端直接打交道需了解大量内部细节，耦合高、容易出错。

- **做法：** 外观类封装全部调用编排，对外只暴露几个简单方法，客户端不用关心内部步骤。

## 三、优点

- **简化接口：** 降低客户端与子系统之间的耦合。

- **隔离变化：** 子系统内部如何改动，只要外观接口不变，客户端不受影响。

- **分层清晰：** 外观层成为子系统的统一入口。

## 四、类比

- **场景：** 以"下单购物"为例，流程背后涉及库存、支付、物流、通知四个子系统（两种方式共用）。

```go
package main

import "fmt"

type Inventory struct{}

func (i *Inventory) CheckStock(item string) bool { fmt.Println("检查库存:", item); return true }
func (i *Inventory) ReduceStock(item string)     { fmt.Println("扣减库存:", item) }

type Payment struct{}

func (p *Payment) Pay(account string, amount float64) bool {
	fmt.Printf("%s 支付: %.0f 元\n", account, amount)
	return true
}

type Shipping struct{}

func (s *Shipping) Ship(item, address string) { fmt.Printf("发货: %s 到 %s\n", item, address) }

type Notification struct{}

func (n *Notification) Send(msg string) { fmt.Println("通知用户:", msg) }
```

### 没有外观模式

- **问题：** 客户端被迫了解四个子系统的所有方法、调用顺序和判断逻辑，一旦流程改动，所有调用处都得修改。

```go
func main() {
	inventory := &Inventory{}
	payment := &Payment{}
	shipping := &Shipping{}
	notification := &Notification{}

	item, account, address := "iPhone", "用户A", "北京"
	price := 6999.0

	if inventory.CheckStock(item) {
		if payment.Pay(account, price) {
			inventory.ReduceStock(item)
			shipping.Ship(item, address)
			notification.Send("您的订单已发货")
		} else {
			notification.Send("支付失败")
		}
	} else {
		notification.Send("库存不足")
	}
}
```

### 有了外观模式

- **效果：** 把整套流程封装进外观类，客户端从十几行编排缩减成两行，不再需要了解任何子系统细节。

```go
type OrderFacade struct {
	inventory    *Inventory
	payment      *Payment
	shipping     *Shipping
	notification *Notification
}

func NewOrderFacade() *OrderFacade {
	return &OrderFacade{
		inventory:    &Inventory{},
		payment:      &Payment{},
		shipping:     &Shipping{},
		notification: &Notification{},
	}
}

func (o *OrderFacade) PlaceOrder(item, account, address string, price float64) {
	if !o.inventory.CheckStock(item) {
		o.notification.Send("库存不足")
		return
	}
	if !o.payment.Pay(account, price) {
		o.notification.Send("支付失败")
		return
	}
	o.inventory.ReduceStock(item)
	o.shipping.Ship(item, address)
	o.notification.Send("您的订单已发货")
}

func main() {
	facade := NewOrderFacade()
	facade.PlaceOrder("iPhone", "用户A", "北京", 6999)
}
```

# HTTP 请求的 Envelope 设计

Envelope（信封） 是指在实际业务数据之外，包裹一层统一的结构，用于携带元信息

核心结构（最小集）
```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```
这三个字段是硬核心，任何响应都必须有，前端可以无脑解包。

常见响应 Envelope

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 123,
    "name": "Alice"
  },
  "requestId": "a1b2c3d4",
}
```

分页场景
{
  "code": 0,
  "data": {
    "items": [...],
    "pagination": {
      "page": 1,
      "pageSize": 20,
      "total": 200,
      "hasMore": true
    }
  }
}

错误 Envelope
{
  "code": 40001,
  "message": "参数校验失败",
  "errors": [
    { "field": "email", "reason": "格式不正确" },
    { "field": "age",   "reason": "不能为负数" }
  ],
  "requestId": "a1b2c3d4"
}
错误时 data 字段可省略，改用 errors 数组携带字段级错误。

字段说明code业务状态码（0 = 成功，非 0 = 各种错误）message可读描述，调试用data实际业务载荷requestId链路追踪 ID，方便排查问题

成功：code: 0, message: "ok"（或 "success"）是固定常量。成功时真正有意义的信息是 data，外层 message 只是给人看一眼"是成功了"的占位。

失败：code / message 表达错误原因

code 分层 — HTTP 状态码（4xx/5xx）表达传输层错误，业务 code 表达逻辑错误，两者不要混用。


requestId 必带 — 微服务环境下，没有 requestId 排查问题会非常痛苦。

# FusionCompute URN

## 一、基本定义

- **资源标识** URN 是 FusionCompute 中用于唯一标识资源对象的字符串。

- **核心作用** 无论是虚拟机、主机、集群还是存储，系统内的每个资源都可以通过全局唯一的 URN 来定位。

- **直观理解** URN 就是 FusionCompute 资源的“身份证号”，API 调用时通过它定位和操作目标资源。

## 二、格式规则

- **通用格式** FusionCompute URN 通常由站点 ID、资源类型和资源 ID 组成。

    ```text
    urn:sites:{siteID}:{resourceType}:{resourceID}
    ```

- **站点标识** `siteID` 是 VRM 站点的唯一标识，调用登录接口后可以获取。

- **示例格式** 例如 `urn:sites:06630EFD:datastores:1` 表示站点 `06630EFD` 下 ID 为 `1` 的 datastore 资源。

## 三、常见示例

- **资源示例** 常见 FusionCompute 资源的 URN 格式如下。

    | 资源类型 | URN 示例 |
    | --- | --- |
    | 站点 Site | `urn:sites:3C4B5D6E` |
    | 虚拟机 VM | `urn:sites:3C4B5D6E:vms:vm-00000001` |
    | 主机 Host | `urn:sites:3C4B5D6E:hosts:1` |
    | 集群 Cluster | `urn:sites:3C4B5D6E:clusters:1` |

## 四、为什么需要

- **名称不可靠** `name` 是给人看的，可能被修改，也可能重复，例如两个站点里都存在名为 `datastore-01` 的存储资源。

- **URN 可靠** URN 是给程序看的，具有唯一性，可以精准定位到某个具体资源。

- **API 操作** 如果没有 URN 这类唯一标识，只靠名称调用 API，遇到重名资源时系统无法稳定判断要操作哪一个对象。

# ESXi “Sidecar Proxy” 挂载截取

## 一、核心价值

- **前提解释** 传统方式通常通过 `vCenter` 的 `HTTPS API` 从管理网络下载虚拟机磁盘数据，而管理网卡往往不是为大规模数据搬迁设计的，因此容易成为迁移速度瓶颈。

- **方案价值** 当 `vCenter` 的 `HTTPS API` 下载速度太慢时，可以通过底层挂载方式绕开管理网卡限速，获得极致的读写速度。

- **整体收益** 相比传统 API 提取方式，速度可以提升 `3-5` 倍；数据读取压力也不再集中到 `vCenter` 管理节点上，因此对管理节点的压力几乎为零。

## 二、实现流程

- **代理机创建** 在源端 `VMware` 集群里启动一个极小的 `Alpine Linux` 虚拟机，作为 `Proxy` 代理机使用。

- **磁盘热挂载** 调用 `VMware Hot-Add` 热拔插 API，将客户要迁移的虚拟机 `VMDK` 磁盘直接挂载到 `Proxy` 代理机上，挂载后该磁盘会变成 `/dev/sdb`。

- **底层读取** `Proxy` 机和目标磁盘位于同一个底层物理存储上，例如 `SAN` 或 `vSAN`，因此可以直接以光纤级别的存储侧速度读取磁盘，通常可达 `2GB/s` 以上，并完全绕过网络限速。

- **内网推送** 在 `Proxy` 内部读取磁盘后，直接通过内网高速流推送到华为 `FusionCompute`。

# fstab

- File System Table（文件系统表）。

- **它有什么用？**

    - 在 Windows 里，您插上一块新硬盘，系统会自动给它分配一个盘符，比如 D 盘、E 盘；而在 Linux 世界里，没有盘符的概念，所有的设备都必须“挂靠（Mount）”到一个具体的文件夹下才能使用。
    
    - 如果您没有 fstab，那么每次 Linux 电脑开机启动后，电脑插着的好几块硬盘就只是干巴巴地躺在那里，您每次重启服务器后，都得手动敲一大堆命令（比如 mount /dev/sdb1 /data）来激活它们。这显然不现实。
    
    - 所以 Linux 引入了 /etc/fstab 这个配置文件： 系统每次开机时，都会像看说明书一样，逐行阅读这个文件，然后全自动地帮您把所有硬盘都挂载好。它让系统启动后自动化挂载磁盘，使存储资源直接就绪。

- **基本格式：**

    - 它里面的每一行像是一句有 6 个词的固定语法句子： “把【哪一块具体的硬盘】，挂载到【哪一个具体的文件夹】，这块盘是【什么格式化类型】，以及使用【什么样的读写权限】。”
    
    - 比如： `UUID=xxxx   /data   xfs   defaults   0 0` 意思就是：把身份证号是 xxxx 的那块硬盘，每次开机都自动帮我绑在该系统主文件夹底下的 /data 目录下，这盘是 xfs 格式，给普通权限即可。

- **问题：**

    - 如果您不小心在这个文件里打错了一个字（比如把硬盘名拼错了，或者指定的文件夹不存在），Linux 开机时一看对不上，就会立马翻脸，直接拒绝启动系统（或者跌落到紧急命令行模式，就像您之前遇到的 Dracut 界面一样）。

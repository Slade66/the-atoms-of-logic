# VMware Guest Operations

## 一、基本定义

- **功能定位** Guest Operations 是 VMware vSphere 提供的一套 API，允许外部程序在虚拟机内部执行操作。

- **核心特点** GuestOps 不依赖虚拟机自身的 SSH、RDP 或业务网络，只要 Guest OS 内的 VMware Tools 正常运行，并且调用方具备必要账号权限，就可以通过虚拟化通道在虚拟机内部执行命令或传输文件。

## 二、通信链路

- **链路结构** GuestOps 的通信路径是从 vCenter / ESXi 进入 VMware Tools，再由 VMware Tools 在 Guest OS 内执行具体动作。

    ```text
    宿主机 ESXi / vCenter
        |
        | 不需要 SSH / RDP / Guest 网络
        | 通过 VMware Tools 通道通信
        v
    虚拟机 Guest OS
    ```

- **执行流程** 以 `bifrost-migrate` 调用 GuestOps 在 Windows 内执行 PowerShell 为例，整体流程如下。

    1. `bifrost-migrate` 调用 VMware API，请求在指定虚拟机中运行 `powershell.exe`。
    2. 请求通过网络发送到 vCenter / ESXi。
    3. ESXi 通过虚拟化通道把命令传给 Guest OS 内的 VMware Tools 或 open-vm-tools。
    4. VMware Tools 收到命令后，调用 Windows 底层 `CreateProcess` API。
    5. PowerShell 进程以配置的 Guest 用户身份启动，例如 `Administrator`。

## 三、能力范围

- **操作能力** GuestOps 可以在 Guest OS 内完成常见的系统操作。

    | 能力 | 说明 |
    | --- | --- |
    | 执行程序 | 在 Guest 内运行可执行文件或脚本 |
    | 文件传输 | 上传或下载 Guest 内的文件 |
    | 目录操作 | 创建、删除、列出目录 |
    | 环境变量 | 读取 Guest 内的环境变量 |
    | 进程管理 | 列出或终止 Guest 内的进程 |

## 四、执行条件

- **Tools 要求** VMware Tools 必须已安装并运行在 Guest OS 内；如果 Tools 停止，GuestOps 会完全失效。

- **双重权限** GuestOps 不是免认证执行，调用方既需要访问 vCenter / ESXi 的账号权限，也需要 Guest 系统内的账号和密码。

## 五、执行特点

- **后台执行** GuestOps 最终确实是在虚拟机内部运行命令，但它是通过 VMware Tools 启动的非交互式后台进程，不是用户在桌面上手动打开 CMD 后执行，因此没有桌面窗口，也没有真正的控制台屏幕。

- **无法交互** 如果脚本中途弹出确认框，或需要输入 `Y/N`，进程可能会一直卡住，因为没有用户能点击或输入。

# 3X-UI 实战：配置静态住宅 IP 链式代理，绕过 AI 平台风控

- 在某些高风控场景（如 Claude、ChatGPT、Gemini）下，直接使用机房 IP 的服务器往往会被拦截、降智。此时，通过 3X-UI 配置“链式代理”，让特定流量经过一层静态住宅 IP 出站，是极佳的解决方案。

### 1. 准备工作：获取静态住宅 IP

1. **购买资源** 前往 IPRoyal 或同类服务商购买“静态住宅 IP（Static Residential Proxy）”。
    - 由于对 IP 纯净度有极高要求，在下单时务必勾选“补充额外需求”选项（注：该项为付费增值服务），并明确备注所需的特定运营商，以获取高质量的 IP。

2. **提取凭据** 在服务商后台查看并记录 SOCKS5 代理的 “IP 地址、端口、用户名和密码”。

### 2. 配置出站规则 (Outbound)

1. 在 3X-UI 面板中，进入 Xray 设置 -> 出站规则 -> 添加出站，按以下 JSON 进行配置：

    ```json
    {
    "protocol": "socks",
    "settings": {
        "servers": [
        {
            "address": "65.195.35.158",
            "port": 12324,
            "users": [
            {
                "user": "14a733923892f",
                "pass": "cedca20c12"
            }
            ]
        }
        ]
    },
    "tag": "iproyal"
    }
    ```

> [!IMPORTANT]
> **规则顺序**：在出站规则列表中，请确保这条新增规则排在 `direct`（直连）规则之后。如果将其置于首位，服务器的全部流量都会走代理。

### 3. 配置路由规则 (Routing)

1. 为了实现精细化分流（即：仅让 AI 相关流量走住宅 IP，其余流量走原机房 IP），在 “路由规则 -> 添加规则” 中进行如下设置：

    - **域名列表 (Domain)**：添加以下关键域名（涵盖 Google 基础服务、OpenAI、Claude 及 IP 检测工具）：
     `domain:gstatic.com,domain:googleusercontent.com,domain:google.com,domain:googleapis.com,domain:chatgpt.com,domain:openai.com,domain:claude.com,domain:anthropic.com,domain:claude.ai,domain:ipinfo.io,domain:x.ai,domain:grok.com,domain:withgoogle.com`

    - **出站标签 (Outbound Tag)**：填写你刚才在 “出站规则” 中设置的标签，例如 `iproyal`。

- 完成上述配置后，你的服务器便具备了“随时变身为美国家庭用户”的能力。可通过访问 `ipinfo.io` 验证出站 IP 是否已成功切换为 IPRoyal 的住宅段。

> [!IMPORTANT]
> **记得保存 & 重启**：在完成配置后，点击上面的 “Save” 和 “Restart Xray”，使配置生效。

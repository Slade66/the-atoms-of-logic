# VDDK（Virtual Disk Development Kit）

## 一、基本定义

- **工具定位** VDDK 全称是 Virtual Disk Development Kit，是 VMware 官方提供的虚拟磁盘开发工具包，主要给备份、迁移、灾备这类软件使用，让程序可以用官方方式读取或写入 VMware 虚拟机磁盘，也就是 VMDK。

## 二、解决的问题

- **格式复杂** VMDK 是 VMware 的虚拟磁盘格式，内部结构复杂，手动解析不可靠。

- **快照链复杂** 虚拟机存在快照时，磁盘数据可能分散在基础盘和多个 delta 文件中，需要合并快照链才能得到完整数据。

- **官方通道** VMware 通过 VDDK 提供一个官方、受控的访问通道，让第三方程序不用自己解析 VMDK 和快照链。

- **增量能力** 配合 Changed Block Tracking，也就是 CBT，VDDK 可以只读取发生变化的数据块，而不是每次都全量读取磁盘。

## 三、协作分工

- **CBT 与 VDDK** CBT 负责告诉程序哪些块发生变化，VDDK 负责把这些变化块的真实字节读取出来。

    | 组件 | 角色 |
    | --- | --- |
    | CBT | 变化块追踪，告诉程序哪些块变了 |
    | VDDK | 磁盘数据读取，把变化块的真实字节读出来 |

- **govmomi 与 VDDK** `govmomi` 负责 vCenter / ESXi 管理面操作，VDDK 负责磁盘数据面读写。

    | 组件 | 分工 |
    | --- | --- |
    | `govmomi` | 管理面 API，负责查 VM、建快照、查 `changeId` |
    | VDDK | 数据面 SDK，负责打开 VMDK、读取磁盘块 |

## 四、传输模式

- **模式差异** VDDK 提供多种传输模式，不同模式的性能和部署要求差异很大。

    | 模式 | 说明 | 特点 |
    | --- | --- | --- |
    | `nbd` | 通过明文 TCP 网络传输数据 | 部署简单，但速度慢 |
    | `nbdssl` | `nbd` 加 SSL 加密 | 更安全，但通常更慢 |
    | `san` | 备份机直连 SAN 存储网络，例如 FC / iSCSI | 性能高，但物理拓扑要求高 |
    | `hotadd` | 将 VM 快照磁盘热挂载到代理 VM 上读取 | 速度快，适合代理 VM 与源 VM 在同一 ESXi 的场景 |

- **HotAdd 优势** `hotadd` 不走普通网络传输，读取速度更接近本地磁盘，通常是高性能的优先选择。

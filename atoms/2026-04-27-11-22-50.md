# virtio

## 一、virtio 是什么

- **基本定义** virtio 是一种虚拟化 I/O 的标准化接口规范，属于半虚拟化 paravirtualization 方案，用于让虚拟机 Guest 通过 virtio 驱动与宿主机 hypervisor 高效通信。

- **核心思想** virtio 不在 hypervisor 中完整模拟真实硬件，而是定义虚拟化场景专用协议，让 Guest 直接提交 I/O 请求，从而绕开低效模拟层，获得接近裸机的性能。

## 二、为什么需要 virtio

- **接口碎片化** virtio 出现之前，不同虚拟化方案各自维护自己的半虚拟化 I/O 接口，Guest 操作系统难以用同一套驱动适配多个 hypervisor。

    - Xen 有自己的 `xenbus`。
    - VMware 有自己的 `VMXNET`、`PVSCSI`。
    - KVM/QEMU 仍大量依赖低效的全模拟设备。

- **模拟开销** 在全模拟 Full Emulation 中，虚拟机以为自己正在操作一块真实硬件，例如 Intel e1000 网卡；QEMU 需要完整模拟这块网卡的寄存器、总线行为和设备状态机，Guest 每次 I/O 操作都可能触发大量 trap。

## 三、virtio 如何解决问题

- **旧路径** virtio 之前，Guest 驱动面向“模拟出来的具体硬件”，再由 hypervisor 模拟层转发到后端。

    ```text
    Guest 驱动  ->  模拟的具体硬件  ->  hypervisor 后端
    e1000.ko       假装是真网卡        QEMU 模拟层
        |                                 |
        |                                 维护完整硬件状态机
        不知道自己在虚拟机里
    ```

- **新路径** virtio 之后，Guest 驱动面向 virtqueue，把请求写入共享内存环，再由 hypervisor 后端直接消费，从而减少硬件模拟、降低数据拷贝，并让一套 Guest 驱动可以服务多个 hypervisor。

    ```text
    Guest 驱动  ->  virtqueue 共享内存环  ->  hypervisor 后端
    virtio-net      知道自己在虚拟机里        直接处理队列
        |                                      |
        |                                      不用模拟寄存器
        一套驱动跑多个 hypervisor              零拷贝或少拷贝
    ```

- **交互过程** Guest OS 与 hypervisor 通过 virtio 前端驱动、virtqueue 和后端设备完成 I/O 请求传递。

## 四、发展与设备

- **早期时间线** virtio 由 Linux 内核开发者 Rusty Russell 提出并实现，当时他在 IBM 工作；2007 年，virtio 以论文 “virtio: Towards a De-Facto Standard For Virtual I/O Devices” 的形式发表在 EuroSys 2008，同年代码合并进 Linux 内核 2.6.24。

- **事实标准** KVM 在 Linux 2.6.20 合并进内核后，迅速采用 virtio 作为默认 I/O 方案；QEMU 随后实现 virtio 后端，使 KVM + virtio 成为高性能虚拟化 I/O 的事实标准组合。

- **设备分类** virtio 不是单个设备，而是一组虚拟 I/O 设备规范。

    | 设备 | 用途 |
    | --- | --- |
    | `virtio-net` | 网络接口 |
    | `virtio-blk` | 块设备，常用于磁盘 |
    | `virtio-scsi` | SCSI 存储 |
    | `virtio-balloon` | 内存气球，用于动态调整内存 |
    | `virtio-gpu` | 图形输出 |
    | `virtio-fs` | 宿主机目录共享 |

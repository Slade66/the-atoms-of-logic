# Linux 内核的启动流程

### Step 1：自解压

- **CPU 跳到 `vmlinuz` 的入口后，执行的第一段代码不是内核本身，而是解压器。** `vmlinuz` 的结构其实是：

    ```
    ┌────────────────────────────────┐
    │ 一小段解压代码（未压缩）       │  ← CPU 先执行这里
    ├────────────────────────────────┤
    │ 真正的内核（压缩）             │
    │   (vmlinux，几十 MB)           │
    └────────────────────────────────┘
    ```

- **解压器自己解压自己的"身体"，** 把真正的内核释放到内存另一块位置，然后跳过去执行。

### Step 2：内核早期初始化（start_kernel）

- **内核进入 `start_kernel()` 函数**（Linux 源码里真实的函数名），初始化内存管理、调度器、中断、驱动框架等内核子系统。

### Step 3：解压 initramfs，挂载为临时根文件系统

- **内核的第一个"文件系统"不是真硬盘，而是 initramfs：**

    ```
    内核把 initramfs（还在内存里，压缩的）
        │
        ▼
    解压到内存里
        │
        ▼
    把这块内存"当成硬盘"来用
        │
        ▼
    挂载为 /
    ```

- **此刻的内存布局：**

    ```
    ┌────────────────────────┐
    │ 物理内存               │
    │   ├─ 内核自己          │
    │   ├─ initrd 解压后     │ ← 被当成 / 来用
    │   │    /init           │
    │   │    /bin/           │
    │   │    /lib/modules/   │
    │   │    ...             │
    │   └─ 其他空闲空间      │
    └────────────────────────┘
    ```

### Step 4：启动 PID 1 = /init

- **内核创建第一个用户态进程 PID 1，执行的程序是 initrd 里的 `/init`**（一个 shell 脚本或小程序）。

- **initrd 里的 `/init` 完整流程：**

    ```
    ┌──────────────────────────────────────┐
    │ ① 加载真实硬件驱动                   │
    │   insmod /lib/modules/sd_mod.ko      │  SATA 硬盘驱动
    │   insmod /lib/modules/nvme.ko        │  NVMe 驱动
    │   insmod /lib/modules/ext4.ko        │  ext4 驱动
    │   ...                                │
    ├──────────────────────────────────────┤
    │ ② 探测硬盘                           │
    │   看到 /dev/sda, /dev/nvme0n1 了     │
    ├──────────────────────────────────────┤
    │ ③ 处理特殊情况                       │
    │   - 如果根分区加密 → 问你密码        │
    │   - 如果是 LVM → 激活逻辑卷          │
    │   - 如果是 RAID → 组装阵列           │
    ├──────────────────────────────────────┤
    │ ④ 找到真正的根分区                   │
    │   内核参数里有 "root=UUID=abc-123"   │
    │   扫所有硬盘找这个 UUID              │
    │   找到了：/dev/sda2                  │
    ├──────────────────────────────────────┤
    │ ⑤ 把真正的根分区挂到 /new_root       │
    │   mount /dev/sda2 /new_root          │
    ├──────────────────────────────────────┤
    │ ⑥ 切换根目录（关键一步！）           │
    │   switch_root /new_root /sbin/init   │
    │   这会：                             │
    │     - 把 / 从 initrd 切到 /dev/sda2  │
    │     - 丢弃 initrd（回收内存）        │
    │     - 执行新的 /sbin/init（=systemd）│
    └──────────────────────────────────────┘
    ```

- **initrd 就是"临时脚手架"，用完就拆：** 它只在开机前几秒存在，一旦 `switch_root` 完成，就被从内存里扔掉。

### Step 5：真正的 PID 1 = systemd 接管

- **PID 1 的两次接手：** `switch_root` 之后，PID 1 变成硬盘上真正的 `/sbin/init`（Ubuntu 上是 systemd 的软链接）。

    - **第一个 PID 1：** initrd 里的临时 `/init`（开机前几秒存在）。
    - **第二个 PID 1：** systemd（现代 Linux 几乎都是它）。
    - PID 号不变，都是 1，但执行的程序换了。

- **systemd 接手后做的事（用户空间初始化）：** 读 `/etc/systemd/system/` 下的 unit 文件，按依赖并行拉起所有服务。

    ```
    systemd 启动
        │
        ▼
    读 /etc/systemd/ 下的一大堆配置
        │
        ▼
    并行拉起服务：
        - journald          (日志)
        - NetworkManager    (网络)
        - sshd              (远程登录)
        - cron              (定时任务)
        - gdm               (图形登录)
        - ......
        │
        ▼
    gdm 画出登录界面
        │
        ▼
    你输密码 → 进桌面
    ```

### 全景时序图

```
时间
  │
  │   [你按了回车，选 Ubuntu]
  │
T=0   GRUB stage2 读 vmlinuz → 内存
  │   GRUB stage2 读 initrd  → 内存
  │   GRUB stage2 设置内核参数
  │   GRUB stage2 切 CPU 模式（实模式 → 64 位保护模式）
  │   jmp 到内核入口
  │             ▼
T=1   内核解压自己
  │             ▼
T=1.5 start_kernel()：初始化内存、调度器、中断、驱动框架...
  │             ▼
T=2   内核解压 initrd，挂为 /
  │   内核启动 PID 1 = initrd 的 /init
  │             ▼
T=2.5 /init 加载真硬盘/文件系统驱动模块
  │   /init 找根分区 UUID
  │   /init 挂真根分区到 /new_root
  │             ▼
T=3   /init 执行 switch_root → 丢掉 initrd
  │   PID 1 替换为 /sbin/init（systemd）
  │             ▼
T=3.5 systemd 并行拉起所有服务
  │             ▼
T=8   gdm 画登录界面
  │             ▼
T=12  你输密码 → GNOME 桌面出现
  │             ▼
  [开机完成]
```

### 一句话总结

- **GRUB = 搬砖的：** 把压缩的内核、压缩的 initrd、启动参数一股脑扔进内存，切好 CPU 模式，跳过去。

- **内核 = 真正干活的：**

    1. 自解压（`vmlinuz` 是压缩包，解压器自己解自己）。
    2. 初始化：内存管理、调度器、中断、驱动框架。
    3. 解压 initrd 挂为 `/`（临时根，在内存里）。
    4. 启动 PID 1 = initrd 里的 `/init`（临时的）。

- **`/init` 干的事：** 加载真硬盘/文件系统驱动（ext4、nvme...） → 按 UUID 找到真根分区 → 挂到 `/new_root` → `switch_root`：根切到真硬盘，丢掉 initrd，PID 1 换成 `/sbin/init`（= systemd）。

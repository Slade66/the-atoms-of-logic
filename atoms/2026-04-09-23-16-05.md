# DHCP 协议：自动化配置网络

- DHCP（Dynamic Host Configuration Protocol）是一种局域网协议，核心作用是自动为终端分配 IP 地址及相关的网络信息（掩码、网关、DNS 等），从而省去手动配置的繁琐。

### 1. DHCP 交互四步流程（DORA）

当一台新设备接入网络时，会经历以下四个核心阶段：

1.  **DHCP DISCOVER（发现阶段）**：
    - **动作**：客户端向全网段发送 UDP 广播包。
    - **语义**：“局域网里有没有 DHCP 服务器？我想领个 IP 地址。”
    - **技术点**：此时客户端并不知道服务器的 IP，必须通过广播寻找。

2.  **DHCP OFFER（提供阶段）**：
    - **动作**：服务器作出回应。
    - **语义**：“我在这里！我可以借给你这个 IP：`10.0.0.10`。”
    - **下发内容**：不仅包含 IP，还包括子网掩码、网关、DNS 以及关键的租期（Lease Time）。

3.  **DHCP REQUEST（请求阶段）**：
    - **动作**：客户端确认选择。
    - **语义**：“太好了，我决定租用你提供的那个 IP 地址了！”

4.  **DHCP ACK（确认阶段）**：
    - **动作**：服务器最终拍板。
    - **语义**：“OK，协议达成，这个 IP 正式分配给你了。”
    - **结果**：客户端正式获得合法身份，可以开始上网。

---

### 2. 租约与续租机制

在 DHCP 的世界里，IP 不是永久拥有的，而是“租来”的。

- **租期概念**：系统会约定一个有效期（如 24 小时）。

- **续租触发（Renewal）**：
    - **T1 时刻（租期消耗 50%）**：客户端会发送单播包直接联系原服务器申请续约。
    - **T2 时刻（租期消耗 87.5%）**：若 T1 续约失败，客户端会改发广播包，寻找局域网内任何可用的服务器进行“抢救式”续约。

---

### 3. 技术进阶：为什么基于 UDP？

- **端口分配**：服务器监听 67 端口，客户端监听 68 端口。

- **设计初衷（为什么不用 TCP？）**：
    
    - **逻辑悖论**：TCP 建立连接需要进行“三次握手”，而握手的前提是双方都有合法的 IP 地址。
    
    - **UDP 优势**：在设备连 IP 都没有的“原始阶段”，UDP 这种无连接协议可以直接通过广播进行“盲发”通信，是建立网络连接最简单、高效的路径。

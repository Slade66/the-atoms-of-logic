# golang.org/x/sync/errgroup：并发任务的错误传播与取消

- **定位：** `errgroup` 是用来管理一组并发 goroutine 的工具，在 `sync.WaitGroup` 的基础上增加了错误传播和 Context 取消两个能力，让一组 goroutine 共同完成一个大任务时，任何一个失败，整组都能感知并提前退出。

## 一、解决的问题

- **`WaitGroup` 的局限：** `sync.WaitGroup` 只能解决“等待一组 goroutine 全部跑完”这一件事，只有 `Add` / `Done` / `Wait` 三个方法。

- **错误无处接住：** goroutine 里发生的 error 没有任何地方可以承接，你得自己用加锁的共享变量去收集。

- **取消要手动做：** 一个任务失败后，通常希望其他还在跑的任务赶紧停下来别浪费资源，但 `WaitGroup` 不管这个，你得自己开 context、在出错时手动 `cancel()`。

- **样板示例：** 用 `WaitGroup` 实现“并发抓取一组 URL，任意一个失败就整体退出”，需要手动维护一个 mutex、一个 `firstErr` 变量、一个 `cancel` 函数，还要小心别忘了 `defer wg.Done()`（忘了就死锁）。

    ```go
    func fetchAll(ctx context.Context, urls []string) error {
        var wg sync.WaitGroup
        var mu sync.Mutex          // 保护下面的 firstErr
        var firstErr error
        ctx, cancel := context.WithCancel(ctx)
        defer cancel()

        for _, url := range urls {
            url := url
            wg.Add(1)
            go func() {
                defer wg.Done()
                if err := fetch(ctx, url); err != nil {
                    mu.Lock()
                    if firstErr == nil {
                        firstErr = err
                        cancel()    // 手动取消，通知其他 goroutine 退出
                    }
                    mu.Unlock()
                }
            }()
        }

        wg.Wait()
        return firstErr
    }
    ```

- **本质作用：** `errgroup` 就是把上面这一坨样板封装起来，内置错误收集、只保留第一个 error、出错自动 cancel context，一并解决 goroutine 无法返回错误、错误需要手动同步收集、失败后需要手动通知取消这三个痛点。

## 二、用法

### 最基本的用法

- **两个核心方法：** `g.Go()` 把函数丢到新 goroutine 里执行，`g.Wait()` 阻塞直到全部完成，并返回第一个 error，其余 error 会被丢弃。

- **零值可用：** `errgroup.Group` 不用初始化，声明即可使用。

    ```go
    import "golang.org/x/sync/errgroup"

    func main() {
        var g errgroup.Group  // 零值即可用,不用初始化

        urls := []string{"https://a.com", "https://b.com", "https://c.com"}

        for _, url := range urls {
            url := url // Go 1.22 之前需要这行,避免闭包捕获问题
            g.Go(func() error {
                resp, err := http.Get(url)
                if err != nil {
                    return err
                }
                defer resp.Body.Close()
                return nil
            })
        }

        // 等待所有 goroutine 结束,返回第一个非 nil 的 error
        if err := g.Wait(); err != nil {
            log.Fatal(err)
        }
    }
    ```

### 配合 Context 实现“一个失败，全体取消”

- **最有价值的用法：** 用 `WithContext` 创建的 group，只要任意一个 goroutine 返回 error，派生出的 `ctx` 就会被自动 cancel，其他还在跑的 goroutine 可以借此提前退出。

- **关键前提：** 你必须真的把这个 `ctx` 用在每个 goroutine 内部（传给 HTTP 请求、数据库查询，或在循环里 `select { case <-ctx.Done(): }`）。

- **不会强制 kill：** `errgroup` 不会强行终止你的 goroutine，它只是 cancel 掉 context，能否及时退出取决于你有没有监听它。

    ```go
    func fetchAll(ctx context.Context, urls []string) error {
        g, ctx := errgroup.WithContext(ctx)

        for _, url := range urls {
            url := url
            g.Go(func() error {
                // 把 ctx 传进去，这样别人失败时本任务能被打断
                req, _ := http.NewRequestWithContext(ctx, "GET", url, nil)
                resp, err := http.DefaultClient.Do(req)
                if err != nil {
                    return err // 这里一旦返回错误,ctx 立刻被 cancel
                }
                defer resp.Body.Close()
                return nil
            })
        }

        return g.Wait()
    }
    ```

### 限制并发数量

- **`SetLimit` 控制上限：** 任务很多时（比如几千个 URL），不想同时开几千个 goroutine，可以用 `SetLimit(n)`，达到上限后 `g.Go()` 会阻塞，直到有空位。

- **设置时机：** `SetLimit` 要在任何 `Go` 调用之前设置好。

    ```go
    g, ctx := errgroup.WithContext(ctx)
    g.SetLimit(10) // 最多同时跑 10 个

    for _, item := range items {
        item := item
        g.Go(func() error {
            return process(ctx, item)
        })
    }
    return g.Wait()
    ```

- **`TryGo` 非阻塞版：** 不想阻塞时用 `g.TryGo()`，它在满额时直接返回 `false`，由你自己决定怎么处理。

    ```go
    if !g.TryGo(func() error { return work() }) {
        // 当前没有空闲额度,自己决定怎么处理
    }
    ```

### 获取 errgroup 的返回值

- **天然限制：** `g.Go` 只接受 `func() error`，既不能返回值，`Wait` 也只给你第一个错误。所以要自己在 goroutine 外面准备好容器，让每个 goroutine 把结果写进去。

- **切片按下标写：** 预先分配一个切片，每个 goroutine 按下标写自己的位置。结果天然保持和输入相同的顺序，且各写各的下标、内存不重叠，所以不需要加锁。

    ```go
    func fetchAll(ctx context.Context, urls []string) ([]Result, error) {
        g, ctx := errgroup.WithContext(ctx)
        results := make([]Result, len(urls)) // 提前按数量分配好

        for i, url := range urls {
            i, url := i, url
            g.Go(func() error {
                r, err := fetch(ctx, url)
                if err != nil {
                    return err
                }
                results[i] = r // 各写各的下标,无需锁
                return nil
            })
        }

        if err := g.Wait(); err != nil {
            return nil, err
        }
        return results, nil
    }
    ```

- **map 必须加锁：** 如果结果按 key 组织（比如 map），就必须加锁，因为 map 不支持并发写。

    ```go
    var mu sync.Mutex
    results := make(map[string]Result)
    // ...
    g.Go(func() error {
        r, err := fetch(ctx, url)
        if err != nil {
            return err
        }
        mu.Lock()
        results[url] = r
        mu.Unlock()
        return nil
    })
    ```

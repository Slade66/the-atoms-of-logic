# golang.org/x/sync/errgroup

- errgroup 是用来管理一组并发 goroutine 的工具，它在 sync.WaitGroup 的基础上增加了错误传播和 Context 取消两个能力：一组 goroutine 共同完成一个大任务,任何一个失败,整组都能感知并提前退出。

## 解决的问题

在 errgroup 出现之前,标准做法是 sync.WaitGroup。它能解决"等待一组 goroutine 全部跑完"这一件事，WaitGroup 只有 Add / Done / Wait 三个方法,goroutine 里发生的 error 没有任何地方可以"接住",你得自己用加锁的共享变量去收集。

更麻烦的是取消。如果其中一个任务失败了,你通常希望其他还在跑的任务赶紧停下来,别浪费资源。但 WaitGroup 不管这个,你得自己手动 context,在出错时手动 cancel()。

"并发抓取一组 URL,任意一个失败就整体退出"的需求：

```go
func fetchAll(ctx context.Context, urls []string) error {
    var wg sync.WaitGroup
    var mu sync.Mutex          // 保护下面的 firstErr
    var firstErr error
    ctx, cancel := context.WithCancel(ctx)
    defer cancel()

    for _, url := range urls {
        url := url
        wg.Add(1)
        go func() {
            defer wg.Done()
            if err := fetch(ctx, url); err != nil {
                mu.Lock()
                if firstErr == nil {
                    firstErr = err
                    cancel()    // 手动取消,通知其他 goroutine 退出
                }
                mu.Unlock()
            }
        }()
    }

    wg.Wait()
    return firstErr
}
```

要手动管理一个 mutex、一个 firstErr 变量、一个 cancel 函数,还要小心别忘了 defer wg.Done()(忘了就死锁)。每写一个并发任务就要重抄一遍这些样板,errgroup 做的事情,本质上就是把上面这一坨样板代码封装起来:错误收集、只保留第一个 error、出错自动 cancel context,全都内置了。所以它解决的是三个问题:goroutine 无法返回错误、错误需要手动同步收集、失败后需要手动通知取消。

## 用法

### 最基本的用法

g.Go() 把函数丢到新 goroutine 里执行,g.Wait() 阻塞直到全部完成,并返回第一个 error(其他 error 会被丢弃)。

```go
import "golang.org/x/sync/errgroup"

func main() {
    var g errgroup.Group  // 零值即可用,不用初始化

    urls := []string{"https://a.com", "https://b.com", "https://c.com"}

    for _, url := range urls {
        url := url // Go 1.22 之前需要这行,避免闭包捕获问题
        g.Go(func() error {
            resp, err := http.Get(url)
            if err != nil {
                return err
            }
            defer resp.Body.Close()
            return nil
        })
    }

    // 等待所有 goroutine 结束,返回第一个非 nil 的 error
    if err := g.Wait(); err != nil {
        log.Fatal(err)
    }
}
```

### 配合 Context 实现"一个失败,全体取消"

这是 errgroup 最有价值的用法。用 WithContext 创建的 group,只要有任意一个 goroutine 返回 error,派生出的 ctx 就会被自动 cancel,其他还在跑的 goroutine 可以借此提前退出:

要点:你必须真的把这个 ctx 用在每个 goroutine 内部(比如传给 HTTP 请求、数据库查询、或者在循环里 select { case <-ctx.Done(): })。errgroup 不会强行 kill 你的 goroutine,它只是 cancel 掉 context,能不能及时退出取决于你有没有监听它。

```go
func fetchAll(ctx context.Context, urls []string) error {
    g, ctx := errgroup.WithContext(ctx)

    for _, url := range urls {
        url := url
        g.Go(func() error {
            // 把 ctx 传进去，这样别人失败时本任务能被打断
            req, _ := http.NewRequestWithContext(ctx, "GET", url, nil)
            resp, err := http.DefaultClient.Do(req)
            if err != nil {
                return err // 这里一旦返回错误,ctx 立刻被 cancel
            }
            defer resp.Body.Close()
            return nil
        })
    }

    return g.Wait()
}
```

### 限制并发数量

如果任务很多(比如几千个 URL),你不想同时开几千个 goroutine,可以用 SetLimit，SetLimit(n) 之后,g.Go() 在达到上限时会阻塞,直到有空位。SetLimit 要在任何 Go 调用之前设置好。

```go
g, ctx := errgroup.WithContext(ctx)
g.SetLimit(10) // 最多同时跑 10 个

for _, item := range items {
    item := item
    g.Go(func() error {
        return process(ctx, item)
    })
}
return g.Wait()
```

如果你不想阻塞,可以用 g.TryGo(),它在满了的时候直接返回 false:

```go
if !g.TryGo(func() error { return work() }) {
    // 当前没有空闲额度,自己决定怎么处理
}
```

### 获取 errgroup 的返回值

errgroup 的 Go 只接受 func() error，既不能返回值，Wait 也只给你第一个错误。得自己在 goroutine 外面准备好容器,让每个 goroutine 把结果/错误写进去。

预先分配一个切片,每个 goroutine 按下标写自己的位置。结果天然保持和输入相同的顺序。因为各写各的下标、内存位置不重叠,所以不需要加锁：

```go
func fetchAll(ctx context.Context, urls []string) ([]Result, error) {
    g, ctx := errgroup.WithContext(ctx)
    results := make([]Result, len(urls)) // 提前按数量分配好

    for i, url := range urls {
        i, url := i, url
        g.Go(func() error {
            r, err := fetch(ctx, url)
            if err != nil {
                return err
            }
            results[i] = r // 各写各的下标,无需锁
            return nil
        })
    }

    if err := g.Wait(); err != nil {
        return nil, err
    }
    return results, nil
}
```

如果结果是按 key 组织的(比如 map),那就必须加锁了,因为 map 不支持并发写:

```go
var mu sync.Mutex
results := make(map[string]Result)
// ...
g.Go(func() error {
    r, err := fetch(ctx, url)
    if err != nil {
        return err
    }
    mu.Lock()
    results[url] = r
    mu.Unlock()
    return nil
})
```

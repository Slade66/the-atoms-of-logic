# GRUB 怎么加载内核

### GRUB 要准备的 3 样东西

- **加载内核前，GRUB 需要从 `grub.cfg` 里拼凑出三样东西：**

    | 东西 | 是什么 | 干嘛用 |
    | --- | --- | --- |
    | 内核文件路径 | `/boot/vmlinuz-6.5.0-21-generic` | 真正的操作系统内核 |
    | initrd 文件路径 | `/boot/initrd.img-6.5.0-21-generic` | 临时文件系统，辅助内核启动 |
    | 内核参数 | `root=UUID=abc-123 ro quiet splash` | 要告诉内核的启动选项 |

### Step 1：读内核文件到内存

- **GRUB 打开文件 `/boot/vmlinuz-xxx`**（用上一阶段学会的文件系统能力），把它完整读到内存某个位置。

- **`vmlinuz` 是压缩的内核镜像。** GRUB 不解压，它把压缩包原样扔进内存——解压是内核自己启动后的第一件事。

### Step 2：读 initramfs 到内存

- **initramfs (Initial RAM File System) 是什么：** 一个打包压缩好的临时文件系统（几十 MB），里面装着一个"迷你 Linux 环境"。

- **它为啥存在？为了破解一个经典的鸡生蛋问题：**

    - **问题：** 内核要挂载根分区 `/` 才能加载硬盘/文件系统驱动，但又要先有这些驱动才能读根分区。

    - **解法：** 在 GRUB 阶段就把必需的驱动提前塞到内存，让内核"先有工具再干活"——这些工具就打包在 initramfs 里（它是"提前塞进内存的驱动工具箱"）。

### Step 3：准备内核参数

- **GRUB 把一串 cmdline 字符串放到内存某个约定位置，作为交给内核的"交接清单"：**

    ```
    "root=UUID=abc-123 ro quiet splash"
    ```

    - `root=UUID=abc-123` → 真正的根分区是哪个
    - `ro` → 先以只读挂载
    - `quiet` → 启动日志少一点
    - `splash` → 显示开机画面

### Step 4：跳到内核入口

```
jmp <vmlinuz 入口地址>        ← GRUB 的最后一条指令
```

- **GRUB 光荣退休。** 从这一刻起，内核接管一切，GRUB 占的那几 MB 内存等下会被内核回收。

### GRUB 交接清单（内核接手时拿到的东西）

```
内存里已经有的：
┌───────────────────────────────────────┐
│ ① 压缩的内核镜像 vmlinuz               │  ← CPU 正要执行这里
│ ② 压缩的 initramfs                    │  ← 内核一会儿要用
│ ③ 启动参数字符串                       │  ← 告诉内核怎么跑
└───────────────────────────────────────┘
```

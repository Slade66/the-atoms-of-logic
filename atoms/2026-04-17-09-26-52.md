# VMware ESXi HotAdd

- **作用：**  HotAdd 是 VMware 提供的一种虚拟磁盘数据传输方式，允许在虚拟机运行时动态挂载 SCSI 磁盘。

- **一句话：** 把别人的磁盘临时 “热插” 到自己身上。

- **场景：**
    - 在迁移场景中，备份代理 VM（Backup Proxy VM）运行在与目标 VM 相同的 ESXi 宿主机上，通过 HotAdd 将目标 VM 的虚拟磁盘（VMDK）热挂载到自身。这样，代理 VM 可以像访问本地磁盘一样直接从数据存储读取数据，完全绕过网络传输，从而实现高效的数据迁移。

- **限制：**
    1. 目标 VM 磁盘类型必须是 SCSI，不支持 IDE
    2. Proxy VM 所在 ESXi host 必须能访问源 vmdk 所在的 datastore。
        - **原因：** Proxy VM 所在的 ESXi host 必须挂载了源 vmdk 所在的 datastore。因为 HotAdd 只是把一个 .vmdk 文件路径（如 `[datastore1] sourceVM/sourceVM-000001.vmdk`）注册到 Proxy VM 的配置里，Proxy VM 的 VMkernel 要能打开这个文件。
            - 如果是共享存储（VMFS on SAN、NFS、vSAN），集群里所有 host 都能看到 → Proxy 可以在同集群任意 host 上
            - 如果是本地存储（host 的本地 SSD/HDD），只有那台 host 能看到 → Proxy 必须跑在同一台 host 上
        - **最佳实践：** Proxy 与源 VM 同一 ESXi host（同 datastore 是底线，同 host 性能最优）。
        - **违反后果：** VDDK 自动 fallback 到 nbd，带宽从 200+ MB/s 降到 20-40 MB/s。

    3. **SCSI 控制器限制**
        - 要求：Proxy VM 要有空闲 SCSI 槽位给热插磁盘用（要有地方插这块盘）。
        - 每 VM 最多 4 个控制器 × 每控制器 15 块盘 = 60 块盘上限。
        - **违反后果：** 槽位满 → HotAdd 失败 → fallback 到 nbd。

- **在 ESXi 网页控制台用 HotAdd 挂载磁盘的步骤：**
    1. 找到 Proxy VM → 编辑设置
    2. 添加硬盘 → 现有硬盘，选一个现有 VMDK
    3. 保存后进入 Proxy VM 检查是否有新磁盘出现（`lsblk`）
    4. 如果成功出现 → 说明 ESXi 环境支持 HotAdd
    5. 在 ESXi 网页控制台 → 编辑 Proxy VM 设置 → 找到刚添加的硬盘 → 移除（不要勾选删除文件），否则下次源 VM 启动有冲突。

- **原理图示：**

    ┌─────────────────── ESXi Host ───────────────────┐
    │                                                  │
    │  ┌──────────────┐         ┌──────────────────┐  │
    │  │ Source VM    │         │ Proxy VM         │  │
    │  │ (被迁移)     │         │ (跑 bifrost/VDDK)│  │
    │  │              │         │                  │  │
    │  │  disk-0.vmdk │◄────────│ 挂载为 /dev/sdX  │  │
    │  │  (快照)      │  热插   │                  │  │
    │  └──────┬───────┘         └────────┬─────────┘  │
    │         │                          │            │
    │         └──────► Datastore ◄───────┘            │
    │              (VMFS/NFS/vSAN)                    │
    └──────────────────────────────────────────────────┘

- **迁移场景 HotAdd 关键流程：**
    1. 创建快照：对源 VM 打快照，冻结磁盘状态（vmdk 变为只读基础链 + redo log）
    2. Hot-Add：vCenter/VDDK 调用 ReconfigVM_Task，把快照 vmdk 作为新磁盘挂到 Proxy VM 的虚拟 SCSI 控制器上
    3. 数据读写：Proxy VM 里的 VDDK 通过本地块设备读写（实际走 VMkernel → datastore）
    4. Hot-Remove：传输完成后把磁盘从 Proxy VM 上拔下来
    5. 删除快照：完成后清理源 VM 快照（合并 redo log）

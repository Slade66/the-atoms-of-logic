# CMDB 面临的挑战

- 维护地狱：需要在资产的整个生命周期中持续收集最新的变更数据，否则数据会过时。
    - 如果缺乏自动化的发现工具（Discovery），CMDB 的数据必须靠人工录入。人工维护的 CMDB “永远是过时的”，并且很快就会变得像以前的独立文件一样毫无用处。这是一项永远做不完的工作。
    - 自动化发现消除数据滞后：保持信息的绝对准确。现代 CMDB 能够通过自动化扫描和 API 集成，自动发现网络中的新设备并捕获现有配置项的更改，确保这张“地图”始终反映 IT 环境的最新真实状态。如果没有这个功能，员工需要手动录入记录，极易出错，导致不一致。

# Git Autostash

## 一、问题与解决

- **原始问题** 本地有未提交改动，但又想先同步远程代码时，直接执行 `git pull --rebase` 可能会因为工作区不干净而失败。

    ```bash
    git pull --rebase
    ```

- **解决方式** `autostash` 表示“自动临时 stash”，它不是一个独立命令，而是 `git rebase`、`git merge`、`git pull` 等命令的一个选项或配置。

- **执行逻辑** 加上 `--autostash` 后，Git 会在操作开始前把当前工作区里的未提交改动临时 `stash` 起来；操作完成后，再自动把这些改动应用回来。

    ```bash
    git pull --rebase --autostash
    ```

## 二、等价流程

- **流程展开** `git pull --rebase --autostash` 大致等价于先 `stash`，再 `pull --rebase`，最后 `stash pop`。

    ```bash
    git stash
    git pull --rebase
    git stash pop
    ```

## 三、注意事项

- **未跟踪文件** 普通 `git stash` 默认只保存已跟踪文件的修改和暂存改动；未跟踪文件需要使用 `git stash -u` 才会包含。

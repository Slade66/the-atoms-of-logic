# 1


MySQL 默认模式下，如果前端 PUT 一个完全相同的值，RowsAffected = 0，会被误判成"记录不存在"。

GORM 的 Save / Updates 会自动维护 UpdatedAt 字段。也就是说：

即使你业务字段全部等于旧值，updated_at 一定在变
MySQL 看到行"实际变更"了，RowsAffected = 1
只有真的没匹配到，才 = 0

只要 PO 里有 UpdatedAt 字段，RowsAffected 就是可靠的。

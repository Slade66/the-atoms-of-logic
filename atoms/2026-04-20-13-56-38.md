# 硬盘开头那段引导代码：MBR

- **MBR 是什么？** 硬盘的第一个扇区（LBA 0），共 512 字节，全称 Master Boot Record（主引导记录）。MBR 里那段代码能做的事非常少——它不认识文件系统，不认识 Linux/Windows，唯一的本事就是"再去读下一段代码"，必须靠它过渡到真正的 bootloader（GRUB / Windows Boot Manager）。

- **开机其实是一场多级接力：**

    ```
    BIOS（只会读第 0 扇区）
      └─► MBR 里的 446 字节（只会读"活动分区"的第 0 扇区）
            └─► 分区里的引导程序 VBR（能读懂文件系统了）
                  └─► GRUB / Windows Boot Manager（能列菜单、能加载内核了）
                        └─► Linux 内核 / Windows 内核（真·操作系统启动）
    ```

- **BIOS 做的事极其机械：**

    - **第 1 步：** 把硬盘第一页（512 字节）整个复制到内存。

    - **第 2 步：** 检查这一页最后两个字节是不是 `55 AA`（相当于暗号："我是合法的接棒人"）。

    - **第 3 步：** 暗号对上了，就跳过去执行这一页开头的代码。

    - **第 4 步：** 然后 BIOS 就退休了，再也不管了。

- **两种引导体系的差异：**

    - **传统 BIOS + MBR 分区表：** 就是上面这套，最多 4 个主分区，单盘最大 2 TB。

    - **UEFI + GPT：** MBR 这 512 字节被降级成 "保护性 MBR"（Protective MBR），只是占位防止老工具误伤；真正的引导逻辑换成了 UEFI 直接读 ESP 分区里的 `.efi` 文件（比如 `\EFI\BOOT\BOOTX64.EFI`），不再走那 446 字节。

### 512 字节怎么分配

- **三段式总体布局：**

    ```
    ┌─────────────────────────────┬──────────────┬──────┐
    │  446 字节：一小段程序代码   │ 64 字节：分区表 │ 55AA │
    └─────────────────────────────┴──────────────┴──────┘
    ```

- **三段各自的作用：**

    - **前 446 字节 — 引导代码（Boot Code / Bootstrap）：** 固件（BIOS）真正"跳过去执行"的就是这一段机器码。它的任务很简单：在分区表里找到被标记为"活动（active）"的分区，把那个分区的第一个扇区（叫 VBR，Volume Boot Record）再读进内存，然后再跳过去。用大白话说就是——"我瞄一眼旁边那 64 字节的分区表，找到被打了★标记的那个分区，再把那个分区的第一页读进内存，再跳过去。"

    - **中间 64 字节 — 分区表：** 4 个表项 × 每项 16 字节，记录硬盘被切成哪几个分区（起始位置、大小、类型、是否活动）。

    - **最后 2 字节 — 魔数 `0x55 0xAA`：** 固件读完这 512 字节后会校验末尾两字节，对得上才承认"这是个合法的引导扇区"，否则就去试下一个引导设备。

- **加载到内存后的详细布局：** BIOS 把硬盘第 0 扇区读到内存地址 `0x7C00` 处，布局如下：

    ```
    内存地址         偏移      大小        内容
                ┌──────────────────────────────────────────────────────┐
    0x7C00      │ 偏移 0x000                                           │
                │                                                      │
                │          引导代码（Boot Code）                       │
                │          446 字节 = 0x1BE                            │
                │                                                      │
                │   作用：读分区表 → 找活动分区 → 加载 VBR → 跳过去    │
                │                                                      │
    0x7DBD      │ 偏移 0x1BD                                           │
                ├──────────────────────────────────────────────────────┤
    0x7DBE      │ 偏移 0x1BE    分区表项 #1     16 字节                │
                ├──────────────────────────────────────────────────────┤
    0x7DCE      │ 偏移 0x1CE    分区表项 #2     16 字节                │
                ├──────────────────────────────────────────────────────┤
    0x7DDE      │ 偏移 0x1DE    分区表项 #3     16 字节                │
                ├──────────────────────────────────────────────────────┤
    0x7DEE      │ 偏移 0x1EE    分区表项 #4     16 字节                │
                ├──────────────────────────────────────────────────────┤
    0x7DFE      │ 偏移 0x1FE    0x55                                   │
    0x7DFF      │ 偏移 0x1FF    0xAA          ← 魔数（结束标志）       │
                └──────────────────────────────────────────────────────┘
                                      共 512 字节 = 0x200
    ```

- **每个分区表项（16 字节）里装的是什么：**

    ```
    偏移   大小   含义
    ┌────┬─────┬────────────────────────────────────────────┐
    │ 0  │ 1 B │ 活动标志  0x80 = 活动(★)   0x00 = 非活动    │
    ├────┼─────┼────────────────────────────────────────────┤
    │ 1  │ 3 B │ 起始 CHS 地址（老式寻址，基本弃用）         │
    ├────┼─────┼────────────────────────────────────────────┤
    │ 4  │ 1 B │ 分区类型  0x07=NTFS  0x83=Linux  0xEE=GPT   │
    ├────┼─────┼────────────────────────────────────────────┤
    │ 5  │ 3 B │ 结束 CHS 地址                               │
    ├────┼─────┼────────────────────────────────────────────┤
    │ 8  │ 4 B │ 起始 LBA（从第几个扇区开始）← 真正用这个    │
    ├────┼─────┼────────────────────────────────────────────┤
    │ 12 │ 4 B │ 分区总扇区数                                │
    └────┴─────┴────────────────────────────────────────────┘
    ```

- **一个真实的例子：**

    ```
    地址      00 01 02 03 04 05 06 07  08 09 0A 0B 0C 0D 0E 0F
    ─────────────────────────────────────────────────────────────
    0x7C00    EB 63 90 ...                                        ┐
      ...     (446 字节的 x86 机器码，BIOS 会跳到这里执行)        │ 引导代码
    0x7DBD    ... 00                                              ┘
    ─────────────────────────────────────────────────────────────
    0x7DBE    80 20 21 00 83 FE FF FF  00 08 00 00 00 F8 9F 07   ← 分区 #1
              ↑           ↑            ↑
              活动(★)    Linux  起始LBA=2048  大小≈128MB
    0x7DCE    00 FE FF FF 83 FE FF FF  00 00 A0 07 00 00 60 F8   ← 分区 #2
    0x7DDE    00 00 00 00 00 00 00 00  00 00 00 00 00 00 00 00   ← 空
    0x7DEE    00 00 00 00 00 00 00 00  00 00 00 00 00 00 00 00   ← 空
    ─────────────────────────────────────────────────────────────
    0x7DFE    55 AA                                              ← 魔数
    ```

- **BIOS 的完整动作：**

    1. 读硬盘 LBA 0 的 512 字节 ──► 写入内存 `0x7C00 ~ 0x7DFF`。

    2. 检查 `[0x7DFE] == 0x55 && [0x7DFF] == 0xAA`？
        - 否 → 放弃，换下一个引导设备。
        - 是 → 继续。

    3. CPU 跳转到 `0x7C00` 开始执行那 446 字节的代码。

    4. 那段代码自己去扫 `0x7DBE / 0x7DCE / 0x7DDE / 0x7DEE` 四个表项，找到首字节是 `0x80` 的那个 ──► 读它的"起始 LBA"扇区到 `0x7C00`（把自己覆盖掉！）──► 再次跳到 `0x7C00`。

    5. BIOS 完全退场，接力棒交给了 VBR。

### 关键数字速查表

| 项目 | 值 |
| --- | --- |
| MBR 所在扇区 | LBA 0（硬盘第一个扇区） |
| MBR 大小 | 512 字节 |
| 引导代码大小 | 446 字节（`0x000 ~ 0x1BD`） |
| 分区表大小 | 64 字节（`0x1BE ~ 0x1FD`），4 项 × 16B |
| 结束魔数 | 2 字节 `55 AA`（`0x1FE ~ 0x1FF`） |
| 加载内存地址 | `0x7C00` |
| MBR 搬家目的地 | `0x0600`（业界惯例） |
| CPU 模式 | 16 位实模式 |
| 活动分区标志 | 表项首字节 = `0x80` |

### 阶段 1：BIOS 上场（固件阶段）

- **BIOS 的完整流程：**

    ```
    按下电源
       │
       ▼
    CPU 复位，PC 寄存器指向 BIOS ROM 的入口（物理地址 0xFFFFFFF0）
       │
       ▼
    BIOS 执行 POST（自检 CPU/内存/外设）
       │
       ▼
    BIOS 按"启动顺序"依次尝试每个设备：硬盘1 → 硬盘2 → U盘 → 网卡 ...
       │
       ▼
    对当前设备：读 LBA 0 的 512 字节 → 加载到内存 0x0000:7C00
       │
       ▼
    检查末尾 2 字节是不是 55 AA ？
       ├─ 否 → 换下一个设备，回到上一步
       └─ 是 → 继续
       │
       ▼
    执行指令：jmp 0000:7C00     ← BIOS 的最后一个动作
    ```

### 阶段 2：MBR 的 446 字节开始干活

CPU 现在站在 `0x7C00`，执行那 446 字节。它要做的事按顺序是：

- **Step 1 — 把自己搬家：** 因为一会儿要把 VBR 也读到 `0x7C00`（约定俗成的位置），会把自己覆盖掉。所以先做一次 memcpy，再跳到新家继续执行（相当于"搬到新家接着干"）。

    ```
    0x7C00 ~ 0x7DFF  ──►  0x0600 ~ 0x07FF
    然后 jmp 到 0x0600 继续执行
    ```

- **Step 2 — 扫描分区表，找活动分区：** 检查 `0x07BE / 0x07CE / 0x07DE / 0x07EE` 这 4 个表项（搬家后的新地址）的第 0 字节。规则是必须且只能有 1 个 `0x80`，否则报错（`Invalid partition table` / `No active partition`）。

    ```
    ┌────────┬──────────┐
    │ 表项#1 │   0x00   │ 不是
    │ 表项#2 │   0x80   │ ★ 就是你！
    │ 表项#3 │   0x00   │
    │ 表项#4 │   0x00   │
    └────────┴──────────┘
    ```

- **Step 3 — 读出该分区的第一个扇区（VBR）：** 从表项里取出"起始 LBA"（偏移 `+8` 的 4 字节），例如 LBA = 2048。调用 BIOS 中断 `INT 13h / AH=42h`（扩展读）把这个扇区读进内存 `0x7C00`。这就是为什么前面要先搬家——现在 `0x7C00` 被新内容占了。

    ```
    读 1 个扇区 (LBA=2048)  ──►  内存 0x7C00
    ```

- **Step 4 — 再次校验 `55 AA`：** 看 `0x7DFE / 0x7DFF` 是不是 `55 AA`。

    - **否：** 打印错误，挂起。
    - **是：** 继续。

- **Step 5 — 把接力棒交出去：** 执行 MBR 的最后一条指令 `jmp 0000:7C00`。MBR 光荣退休，CPU 开始执行 VBR / GRUB stage1.5。

### BIOS 的启动顺序（Boot Order）

- **它长什么样：** 就是你在 BIOS 设置里看到的那个列表。

    ```
    Boot Priority:
      1. USB HDD
      2. SATA0: Samsung SSD 980        ← 你把系统装在这里
      3. SATA1: WD Blue 2TB
      4. Network (PXE)
    ```

- **BIOS 的算法极其死板：** 按顺序一块一块试，谁的 MBR 末尾是 `55 AA` 就执行谁，后面的一律不看。

    ```
    for 设备 in 启动顺序列表:
        读 该设备的 LBA 0 到 0x7C00
        if 末尾两字节 == 55 AA:
            jmp 0x7C00        ← 就它了，后面的硬盘看都不看
            return
        else:
            continue          ← 这块没装系统，换下一块
    报错: "No bootable device"
    ```

- **关键点：** 一旦某块硬盘的 MBR 被执行，BIOS 就彻底退场，剩下几块硬盘在启动阶段的"话语权"为零。

### 多硬盘启动的决策树

- **整体流程：** BIOS 一旦在某块盘上跳到 MBR，后面的盘在启动阶段就彻底出局，能否加载到别盘的内核完全取决于当前 bootloader 的本事。

    ```
    按下电源
       │
       ▼
    [BIOS] 按启动顺序遍历硬盘
       │
       ├── 第 1 块：读 LBA 0 → 有 55AA？
       │         ├─ 否 → 下一块
       │         └─ 是 → 执行其 MBR（就它了！）
       │                    │
       │                    ▼
       │              MBR 找本盘活动分区
       │                    │
       │                    ▼
       │              VBR / GRUB stage1
       │                    │
       │         ┌──────────┴──────────┐
       │         ▼                     ▼
       │   单系统 bootloader      GRUB（够聪明）
       │   只能启动本盘           可跨盘加载其他盘的内核
       │
       ├── 第 2 块：前面已经跳走了，根本轮不到
       └── 第 3 块：同上
    ```

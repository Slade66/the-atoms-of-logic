# Zig 的介绍

### 🧠 Zig 的基本定位

- **语言愿景** Zig 诞生于 2016 年，是一门旨在取代 C 语言的“现代系统级编程语言”。

### 🧠 Zig 命名的由来

- **命名原则** “Zig” 这个名字没有任何全称，创始人制定了明确的起名标准：长度为 3 个字母、必须以 'Z' 开头，且在搜索结果中具有极高的辨识度。

- **筛选过程** 创始人编写了一个 Python 脚本来筛选所有首字母为 Z 的三字母组合，并在其中寻找听起来顺耳的词。最终，“Zig” 在脚本筛选出的候选词中脱颖而出。

### 🧠 Zig 的核心特点

- **极致运行性能** Zig 无垃圾回收（GC）机制，手动管理内存。由于没有运行时开销，其运行速度达到了极致水平。

- **微型二进制体积** Zig 的编译产物极小，通常仅为几 KB。相比之下，Go 语言因必须内置垃圾回收器和运行时，二进制文件通常 5MB - 10MB 起步。

- **C 语言互操作性** Zig 是目前所有主流语言中集成 C SDK 最好的选择。其 C 互操作性被认为是现代语言中做得最好的，甚至超过了 Rust。

- **极简语法设计** Zig 的语法极度克制，全语言仅有 28 个关键字，甚至比以简洁著称的 Go 还要纯粹。开发者通常只需一个下午即可掌握全部语法规则。

    - **学习难度对比**：C < Zig < Go < Rust

### 🧠 Zig 的明星项目

- **Ghostty (终端)** 开发者 Mitchell Hashimoto（HashiCorp 创始人）在放弃了 Rust 和 C++ 后最终选择了 Zig，这为语言提供了极强的专业背书。

- **Bun (JavaScript 运行时)** 目前性能最强的 JS 运行时，执行速度号称比 Node.js 快出数倍。

- **TigerBeetle (数据库)** 开源的金融交易数据库。

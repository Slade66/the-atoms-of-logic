# VMware ESXi

- **VMware ESXi 是什么？**

    - 它是一个裸机虚拟化管理程序（Type-1 Hypervisor）。

    - 不同于我们在 Windows 或 macOS 上安装的 VMware Workstation 或 VirtualBox（这类叫 Type-2，需要寄生在操作系统之上），ESXi 直接安装在物理服务器的硬件上。

    - 它自己就是一套精简的操作系统，不需要底层的 Windows 或 Linux 支持。这种架构使得虚拟机的运行效率极高，资源损耗几乎为零。

- **VMware ESXi 的作用：**

    - ESXi 的存在让一台物理服务器能够“变身”为几十台甚至上百台独立的虚拟服务器，并且每台机器都像运行在真实硬件上一样稳定高效。

- **ESXi 磁盘文件结构**

    - **存储路径**：ESXi 所有的物理磁盘和 LUN 最终都挂载在 `/vmfs/volumes/` 目录下。

    - **.vmx (虚拟机配置文件)**：大小仅几 KB。它相当于虚拟机的“主板”或“蓝图”，以纯文本形式记录了 CPU 数量、内存大小、网卡 MAC 地址及引导方式（BIOS/UEFI）等核心配置。
    
    - **.vmdk (磁盘描述文件)**：大小约 1~2 KB。它是磁盘的“逻辑索引”，本身不含业务数据。通过其中的 `# Extent description` 行作为“指针”，告诉底层程序去关联并读取同目录下的实际数据文件。
    
    - ***-flat.vmdk (原始数据文件)**：这是虚拟机真正的“硬盘”，包含了操作系统、应用和所有用户数据。它采用 VMware 专有的扁平二进制格式，通常与 `.vmdk` 描述文件成对出现。

- **离线迁移至华为 FusionCompute 的核心流程：**
    
    1. **下载描述文件 (.vmdk)**：首先通过驱动代码下载仅几 KB 的描述文件。它是整个虚拟机的“磁盘索引”，包含关键的元数据引用。
    
    2. **解析配置架构 (.vmx 或通过 API 获取)**：
        
        - **传统方案**：解析 `.vmx` 纯文本，手动提取 CPU、内存、MAC 地址及引导模式（BIOS/UEFI）。
        
        - **进阶方案**：调用 VMware 原生 API（如 `govmomi` 对象模型）直接提取虚机的“骨架信息”。
    
    3. **下载数据盘 (-flat.vmdk)**：异步下载实际包含业务数据的二进制大文件，这是整个流程中最耗时的步骤。
    
    4. **格式转换 (qemu-img)**：使用 `qemu-img convert` 将原始镜像转换为华为所需的 `qcow2` 格式。**核心技巧**：命令必须指向几 KB 的 `.vmdk` 描述文件，而非大文件本身。`qemu-img` 会自动识别描述文件中的“指针”，顺藤摸瓜完成转换。
    
    5. **云端点亮**：当三份核心文件在跳板机凑齐并完成转换后，即可通过代码将 `qcow2` 镜像挂载至华为 FC 环境，正式点亮虚机。

- **SDK：**

    - /sdk 后缀：无论连接 vCenter 还是单机 ESXi，API 的端点永远是 /sdk，不要漏掉。

    - nsecure: true：ESXi 默认使用的是自己签发的 SSL 证书。在调用 govmomi.NewClient(ctx, u, true) 时，必须把第三个参数设为 true（相当于 curl -k），否则 Go 会报 x509 证书验证失败的错误。

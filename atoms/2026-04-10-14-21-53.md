# 如何配置 SSH 密钥登录

配置 SSH 密钥登录（免密码登录）通常分为四个标准步骤：**生成密钥** → **上传公钥** → **配置服务器** → **测试登录**。

## 1. 生成 SSH 密钥

在本地机器执行以下命令生成密钥对：

```bash
ssh-keygen -t ed25519 -C "your_email@example.com"
```

> [!TIP]
> 建议使用 `ed25519` 算法，它比传统的 `rsa` 更安全且速度更快。

**操作说明：**
- 出现提示时一路回车即可。
- **默认生成位置：**
    - `~/.ssh/id_ed25519`：**私钥**（绝对不要泄露，需妥善保存）。
    - `~/.ssh/id_ed25519.pub`：**公钥**（可以上传）。

---

## 2. 上传公钥到服务器

### 方法 A：使用自动化工具（推荐）

```bash
ssh-copy-id root@[服务器IP]
```
输入一次密码后，该工具会自动帮你完成配置。

### 方法 B：手动配置（如环境不支持 ssh-copy-id）

1. **本地：** 查看公钥内容：
   ```bash
   cat ~/.ssh/id_ed25519.pub
   ```

2. **服务器：** 登录并写入公钥：
   ```bash
   ssh user@服务器IP  # 输入密码登录
   
   # 创建目录并设置权限
   mkdir -p ~/.ssh
   chmod 700 ~/.ssh
   
   # 将公钥内容追加到 authorized_keys
   echo "你的公钥内容" >> ~/.ssh/authorized_keys
   chmod 600 ~/.ssh/authorized_keys
   ```

---

## 3. 服务器 SSH 配置

为确保密钥登录生效且安全，需要编辑 SSH 配置文件：

```bash
sudo vim /etc/ssh/sshd_config
```

确保以下配置项正确（若被注释请取消注释）：
- `PubkeyAuthentication yes`
- `AuthorizedKeysFile .ssh/authorized_keys`

**生效更改：**
```bash
sudo systemctl restart sshd
```

---

## 4. 测试与使用

### 验证登录
```bash
ssh root@服务器IP
```
如果直接登录成功且不需要密码，说明配置成功。

### 指定私钥登录
如果你的私钥不在默认搜索位置，可以使用 `-i` 参数明确指定：
```bash
ssh -i ~/.ssh/id_ed25519 user@服务器IP
```

---

## ⚠️ 注意事项

- **安全性：** 私钥文件（`id_ed25519`）是访问服务器的凭证，绝对不要泄露。

- **密码限制：** 原生 `ssh` 命令不支持直接通过命令行参数传递密码。

- **目录权限：** SSH 对权限非常敏感。如果无法登录，请确认服务器上 `~/.ssh` 是 `700`，`authorized_keys` 是 `600`。

- **SSH 的私钥权限要求：** 私钥必须绝对私密，只能自己读取，不能被其他用户访问。

    - **违规后果：** 否则会出现 `WARNING: UNPROTECTED PRIVATE KEY FILE!`。

    - **Windows 处理方法：** 右键私钥文件 → 属性 → 安全 → 高级 → 禁用继承 → 删除其他用户，只保留当前用户。

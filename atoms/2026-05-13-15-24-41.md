# 推送 github 如何触发构建 + 部署

背景，我的服务需要推送到github，然后触发构建，然后部署到 aws ec2

GitHub-hosted runner 负责：docker build
EC2 负责：运行 docker-fleet 服务

GitHub Actions 负责构建
默认情况下，构建跑在 GitHub 提供的 runner 服务器 上
你写 workflow，例如 runs-on: ubuntu-latest
GitHub Actions 会分配一台临时的 GitHub-hosted runner
这个 runner 是一台新的虚拟机
GitHub 官方文档说 Linux/Windows runner 是托管在 Microsoft Azure 上的虚拟机
job 跑完后，这台临时机器会被销毁

镜像发布
推荐把 Docker 镜像推到 GitHub 自己的镜像仓库 GHCR：
EC2 负责运行服务：
EC2 是你真正部署服务的机器。它不会自动拿到新代码，除非 GitHub Actions 在构建成功后通过 SSH 连上 EC2，执行：
docker pull ghcr.io/slade66/docker-fleet:latest
docker stop docker-fleet
docker rm docker-fleet
docker run -d \
  --name docker-fleet \
  -p 8000:8000 \
  -p 9000:9000 \
  -v /opt/docker-fleet/configs:/data/conf \
  ghcr.io/slade66/docker-fleet:latest

流水线建议是：

GitHub Actions 临时机器：测试 + 构建 + 推镜像
AWS EC2：运行 docker-fleet 服务

完整链路应该是这样：

本地代码
  ↓ git push
GitHub 仓库
  ↓ 触发 GitHub Actions
GitHub 托管 runner
  ↓ go test ./...
构建 Docker 镜像
  ↓ docker build
推送镜像到 GHCR
  ↓ ghcr.io/slade66/docker-fleet:latest
AWS EC2
  ↓ docker pull
停止旧容器
  ↓ docker stop / rm
启动新容器

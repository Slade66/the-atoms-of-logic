# 结构体的值 vs 指针


遇到「几百个字段的大结构体」「要改原对象」「要表示 nil」时,才用指针。

,Go 只对「没用到的局部变量」和「没用到的 import」报错,函数参数没用到不报错。
 
 
 每步都打 Error

失败:每个失败点都打 Error │ 带 uuid + 哪一步,排查一眼定位。你现在第 2、3 步失败没打,补上              │
  ├───────────────────────────┼───────────────────────────────────────────────────────────────────────────┤
  │ 成功:只打 1 条 Info       │ 别每步都 Info。中间「读规划成功/转换成功」是噪声,生产环境会刷屏           │
  ├───────────────────────────┼───────────────────────────────────────────────────────────────────────────┤
  │ 成功那条带「可观测指标」  │ uuid + 规模(设备/连线数)+ 字节数 + 耗时——导出可能慢,耗时是排查/告警的关键 │

▎ 错误每步记,成功记一条带耗时和规模的。 中间「第 N 步成功」别用 Info(那是 Debug 级别);耗时一定要带——导出可能慢,它是排查慢/告警的关键指标。访问日志中间件已经记了请求级信息,你这层只补「业务细节」,不重复。

 %.1f 是 Go 的格式化占位符(fmt 包的「动词」),专门用来打印小数。拆开看:

  ┌──────┬────────────────────────────┐
  │ 部分 │            意思            │
  ├──────┼────────────────────────────┤
  │ %    │ 占位符开始                 │
  ├──────┼────────────────────────────┤
  │ f    │ 按**浮点数(小数)**格式打印 │
  ├──────┼────────────────────────────┤
  │ .1   │ 精度:小数点后保留 1 位     │
  └──────┴────────────────────────────┘

  合起来 %.1f = 「打印一个小数,保留 1 位小数」。
  .2 保留 2 位，.0 一位都不保留

###

map[string]struct{}{} 是 Go 里**「集合(Set)」的惯用写法

只关心「有没有」,不关心值。

  拆开看

  map[string]struct{}  {}
  └──────┬──────────┘  └┬┘
       类型             空map字面量(初始化成空)

  - map[string]struct{} = 类型:键是 string,值是 struct{}(空结构体)
  - 末尾 {} = 创建一个空的这种 map

  为什么值用 struct{}

  struct{}(空结构体)占 0 字节。做集合时我们只用 key、根本不用 value,所以用最省内存的「空」当占位。

  对比:
  - map[string]bool 也能当集合,但每个 value 占 1 字节
  - map[string]struct{} 的 value 占 0 字节 ← 更省,Go 的惯用法

  怎么用(你代码里就是这么用的)

  set := map[string]struct{}{}     // 建空集合
  set["a"] = struct{}{}            // 加元素(struct{}{} = 空结构体的一个值)
  set["a"] = struct{}{}            // 再加 "a" —— 还是一份,自动去重

  _, ok := set["a"]                // 查在不在:ok=true
  _, ok = set["x"]                 // ok=false


  那个把人绕晕的 struct{}{}(两对括号)

  struct{}  {}
  └──┬──┘   └┬┘
   空结构体  这个类型的一个"值"(实例)
   的"类型"
  - struct{} = 空结构体类型
  - 后面 {} = 造一个该类型的值
  - 合起来 struct{}{} = 「造一个空结构体的值」,专门拿来塞进 set 当占位

  value 用占 0 字节的空结构体当占位

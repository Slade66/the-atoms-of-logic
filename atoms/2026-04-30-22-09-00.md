# Appsmith 如何根据主键或外键显示可读名称

### 一、问题背景

- **使用场景** 数据库里保存的通常是主键或外键，例如 `reviewer_no`，但页面上直接显示编号不够直观。

- **页面目标** 在 Appsmith 页面中，把数据库里的编号转换成用户能看懂的名称，例如把审核人编号显示成审核人姓名。

- **核心思路** 先查询一份编号和名称的映射表，再在控件的默认值里用 JS 表达式根据编号找到对应名称。

### 二、先查询编号和名称的映射数据

1. 在左侧菜单新建查询 `Queries`。

2. 将查询名设置为 `GetReviewers`。

3. SQL 查询审核人的编号和姓名。

    ```sql
    SELECT no, name FROM users;
    ```

4. 打开右上角齿轮 `Settings`。

5. 在 `Run behavior` 中设置为 `On page load`。

- **加载目的** 页面打开时先执行 `GetReviewers`，这样后续控件才能从 `GetReviewers.data` 中找到编号对应的名称。

### 三、用 Default value 转换显示名称

1. 找到需要显示审核人名称的页面控件。

2. 将控件的 `Default value` 设置为下面的 JS 表达式。

    ```js
    {{ GetReviewers.data.find(r => r.no == sourceData.reviewer_no)?.name }}
    ```

3. 保存配置后，页面会根据当前记录的 `reviewer_no` 找到对应的 `name`。

    - **GetReviewers** `GetReviewers` 是前面创建的查询，`GetReviewers.data` 是查询返回的数据列表。

    - **r** `r` 代表 `GetReviewers.data` 里的每一行记录。

    - **reviewer_no** `sourceData.reviewer_no` 是当前控件绑定数据里的外键编号。

    - **name** `?.name` 表示找到匹配记录后，取出这条记录中的姓名。

### 四、把控件类型改成文本显示

1. 找到这个页面控件的 `Field Type`。

2. 将 `Field Type` 改为 `Text Input`。

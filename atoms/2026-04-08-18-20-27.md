# 如何 Debug Go Test

### 使用 `t.Log` 和 -v 参数 

1. 在代码中用 `t.Log` 或 `t.Logf` 打印关键变量
2. 运行测试时带上 `-v` 参数：`go test -v .`（默认 `go test` 会隐藏输出，除非测试失败；加上这个参数能强制输出所有信息）

### VS Code 图形化 Debug 

1. 你会看到测试函数 TestXXX 上方有两行浅色的小字：run test | debug test。
2. 点击 debug test。
3. 在左侧行号处点击设置断点（红色圆点）。
4. 程序运行到断点处会停下，你可以在左侧“变量”栏里动态查看变量的值。

### 使用命令行调试器 Delve（dlv）

- 适用场景：如果你更喜欢命令行调试，或者是在没有 GUI 的服务器上。

1. 安装 Delve（如果还没装）：`go install github.com/go-delve/delve/cmd/dlv@latest`
2. 执行测试调试：`dlv test .`
3. 在进入的交互式命令行里：
    1. `break TestEnableCBT`：在测试函数入口设断点
    2. `continue` (或 `c`)：运行到断点
    3. `next` (或 `n`)：单步执行
    4. `print obj` (或 `p`)：查看某个对象的属性

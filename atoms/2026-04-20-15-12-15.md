# Linux 开机全流程：从按下电源到看到桌面

- **一句话记住整个流程：** BIOS 叫醒 MBR，MBR 指路给 Bootloader，Bootloader 搬来操作系统内核，内核启动 PID 1，PID 1 把你领到桌面。

### 阶段 1：BIOS（固件阶段）

- **谁在运行：** BIOS（或 UEFI），住在主板上一块叫 ROM 的小芯片里，**不在硬盘里**。

- **它在干啥：** 自检 → 初始化硬件 → 按启动顺序找引导设备 → 跳过去。

    ```
    按下电源
        │
        ▼
    CPU 通电，第一条指令自动指向 BIOS ROM 的入口
        │
        ▼
    ┌──────────────────────────────────┐
    │ ① POST 自检                      │
    │   - CPU 能不能跑？               │
    │   - 内存有多少？坏没坏？         │
    │   - 显卡/键盘/硬盘连上没？       │
    │   出问题就"滴滴滴"报警，停机     │
    └──────────────────────────────────┘
        │
        ▼
    ┌──────────────────────────────────┐
    │ ② 初始化硬件                     │
    │   - 启动显卡（屏幕才能显示）     │
    │   - 启动硬盘控制器（能读盘了）   │
    │   - 初始化 USB、键盘             │
    └──────────────────────────────────┘
        │
        ▼
    ┌──────────────────────────────────┐
    │ ③ 按"启动顺序"找引导设备         │
    │   你在 BIOS 设置过：             │
    │     1. SSD                       │
    │     2. HDD                       │
    │     3. USB                       │
    │     4. 网卡                      │
    │                                  │
    │   for 每个设备:                  │
    │     读 LBA 0 的 512 字节         │
    │     到内存 0x7C00                │
    │     检查末尾是 55 AA 吗？        │
    │       ├─ 否 → 试下一个           │
    │       └─ 是 → 就它了！           │
    └──────────────────────────────────┘
        │
        ▼
    jmp 0x7C00         ← BIOS 的最后一条指令，从此它退场
    ```

- **这一阶段硬盘里发生了啥：** BIOS 只读了硬盘的第 0 扇区（512 字节），其他扇区连看都没看。

### 阶段 2：MBR（跳板阶段）

- **谁在运行：** MBR 的 446 字节引导代码，此时已经被 BIOS 加载到内存 `0x7C00`。

- **它长啥样：** 硬盘的第 0 扇区（512 字节）。

    ```
    ┌────────────────────────────┬────────────────┬────────┐
    │  446 B：引导代码            │ 64 B：分区表   │ 2 B    │
    │  (MBR 的大脑)               │ (4 个分区)     │ 55 AA  │
    └────────────────────────────┴────────────────┴────────┘
              ↑                        ↑
       CPU 正在执行这里           446 字节的代码会读这里
    ```

- **它在干啥：** 自己搬家腾位置 → 扫分区表找活动分区 → 把活动分区第一个扇区读进来 → 跳过去。

    ```
    CPU 现在在 0x7C00 执行 MBR 代码
        │
        ▼
    ┌──────────────────────────────────┐
    │ ① 自己搬家                       │
    │   从 0x7C00 挪到 0x0600          │
    │   (因为 0x7C00 马上要给 VBR 用)  │
    └──────────────────────────────────┘
        │
        ▼
    ┌──────────────────────────────────┐
    │ ② 扫描分区表                     │
    │   看 4 个分区表项里谁的第 1 字节 │
    │   是 0x80（活动标志 ★）          │
    │                                  │
    │   ┌──────────┬────────┐          │
    │   │ 分区 1   │  0x00  │          │
    │   │ 分区 2   │ ★0x80  │ ← 找到   │
    │   │ 分区 3   │  0x00  │          │
    │   │ 分区 4   │  0x00  │          │
    │   └──────────┴────────┘          │
    └──────────────────────────────────┘
        │
        ▼
    ┌──────────────────────────────────┐
    │ ③ 读该分区的第一个扇区           │
    │   (叫 VBR，或者就是 GRUB 第一段) │
    │   读到内存 0x7C00                │
    └──────────────────────────────────┘
        │
        ▼
    jmp 0x7C00         ← MBR 的最后一条指令，从此它退场
    ```

- **这一阶段硬盘里发生了啥：** 又读了 1 个扇区（活动分区的第一个扇区），还是 512 字节。

### 阶段 3：Bootloader（搬运工阶段）

- **谁在运行：** Bootloader，比如 GRUB（Linux）或 Windows Boot Manager（Windows）。

- **为什么要分段接力：** Bootloader 本身太大（几 MB），一次装不进内存、也读不完，所以被拆成几小段接力。

    ```
    ┌────────────────────────────────────────────────────────────┐
    │ Stage 1  (住 MBR 里，446 字节)                             │
    │   ← 上一阶段跑完的那个                                     │
    │   唯一任务：读 stage 1.5                                   │
    └──────────────────────┬─────────────────────────────────────┘
                           ▼
    ┌────────────────────────────────────────────────────────────┐
    │ Stage 1.5  (住 MBR 后的保留区，约 30KB)                    │
    │   任务：加载文件系统驱动（ext4/ntfs/...）                  │
    │   ★ 从这一刻起，终于"认识文件"了 ★                         │
    │   然后去 /boot/grub/ 读 stage 2                            │
    └──────────────────────┬─────────────────────────────────────┘
                           ▼
    ┌────────────────────────────────────────────────────────────┐
    │ Stage 2  (住 /boot/grub/ 里，几 MB，是个真正的程序)        │
    │   ★ 这才是真正的 bootloader 主体 ★                         │
    │                                                            │
    │   ① 读配置文件 /boot/grub/grub.cfg                         │
    │   ② 画出菜单给你看：                                       │
    │        ┌─────────────────────────────┐                     │
    │        │ → Ubuntu 22.04              │ ← 你看到的第一个界面│
    │        │   Ubuntu (recovery mode)    │                     │
    │        │   Windows Boot Manager      │                     │
    │        └─────────────────────────────┘                     │
    │   ③ 读你选的那个内核文件到内存                             │
    │        /boot/vmlinuz-6.x.x  (压缩的 Linux 内核，几 MB)     │
    │        /boot/initrd.img-6.x.x  (初始内存盘，几十 MB)       │
    │   ④ 准备启动参数：                                         │
    │        "root=/dev/sda2 ro quiet splash ..."                │
    │   ⑤ 切 CPU 模式：16 位实模式 → 64 位保护模式               │
    │   ⑥ jmp 到内核入口                                         │
    └──────────────────────┬─────────────────────────────────────┘
                           ▼
                  Bootloader 退场
    ```

- **这一阶段硬盘里发生了啥：** 终于开始按文件路径读东西了——

    - `/boot/grub/grub.cfg`（配置）
    - `/boot/vmlinuz-xxx`（内核，几 MB）
    - `/boot/initrd.img-xxx`（initramfs，几十 MB）

### 阶段 4：操作系统内核启动

- **谁在运行：** Linux 内核（`vmlinuz`）或 Windows 内核（`ntoskrnl.exe`），现在已经在内存里跑起来了。

- **它在干啥（以 Linux 为例）：** 自解压 → 初始化子系统 → 挂 initramfs → 探测真实硬件 → 挂真正的根分区 → 启动 PID 1。

    ```
    内核入口被执行
        │
        ▼
    ┌──────────────────────────────────────┐
    │ ① 自解压                             │
    │   vmlinuz 是压缩的，自己解压自己     │
    └──────────────────────────────────────┘
        │
        ▼
    ┌──────────────────────────────────────┐
    │ ② 初始化内核子系统                   │
    │   - 内存管理（分页、虚拟内存）       │
    │   - 进程调度器                       │
    │   - 中断、时钟                       │
    │   - 系统调用接口                     │
    └──────────────────────────────────────┘
        │
        ▼
    ┌──────────────────────────────────────┐
    │ ③ 挂载 initramfs 为临时根文件系统    │
    │   initramfs 是 bootloader 读进来的   │
    │   一个小型 Linux 环境，里面装着：    │
    │     - 真实硬盘的驱动                 │
    │     - LVM / RAID / 加密盘工具        │
    │   目的：能"认识"真正的根分区         │
    └──────────────────────────────────────┘
        │
        ▼
    ┌──────────────────────────────────────┐
    │ ④ 探测真正的硬件                     │
    │   - 加载显卡驱动                     │
    │   - 加载网卡驱动                     │
    │   - 识别 USB 设备                    │
    │   屏幕上哗哗滚过的日志就是这个       │
    └──────────────────────────────────────┘
        │
        ▼
    ┌──────────────────────────────────────┐
    │ ⑤ 挂载真正的根分区                   │
    │   "root=/dev/sda2"（bootloader 传的）│
    │   把 /dev/sda2 挂到 /                │
    │   扔掉 initramfs                     │
    └──────────────────────────────────────┘
        │
        ▼
    ┌──────────────────────────────────────┐
    │ ⑥ 启动第一个用户进程：PID 1          │
    │   Linux：/sbin/init 或 /lib/systemd  │
    │   Windows：smss.exe                  │
    │   ★ 从这一刻起，进入"用户空间" ★     │
    └──────────────────────────────────────┘
    ```

- **这一阶段硬盘里发生了啥：** 真正的操作系统开始全面接管硬盘。

### 阶段 5：用户空间初始化

- **谁在运行：** PID 1（systemd / Windows 的 smss）。

- **它在干啥：** 读配置 → 并行拉起服务 → 启动图形界面 → 接收登录 → 桌面环境。

    ```
    PID 1 启动
        │
        ▼
    ┌──────────────────────────────────┐
    │ ① 读配置（Linux: /etc/systemd/） │
    │   决定要启动哪些"服务"           │
    └──────────────────────────────────┘
        │
        ▼
    ┌──────────────────────────────────┐
    │ ② 并行拉起服务                   │
    │   - 网络 (NetworkManager)        │
    │   - 日志 (journald / syslog)     │
    │   - SSH (sshd)                   │
    │   - 数据库、cron、防火墙 ...     │
    └──────────────────────────────────┘
        │
        ▼
    ┌──────────────────────────────────┐
    │ ③ 启动图形界面                   │
    │   - 显示服务器 (X11 / Wayland)   │
    │   - 显示管理器 (gdm / sddm)      │
    │   ★ 你看到登录界面了 ★           │
    └──────────────────────────────────┘
        │
        ▼
    ┌──────────────────────────────────┐
    │ ④ 你输入密码，启动桌面环境       │
    │   GNOME / KDE / Windows Explorer │
    └──────────────────────────────────┘
        │
        ▼
                 ✓ 开机完成
           你可以开始上网刷剧了
    ```

### 总结对比表：每一棒都在干啥

| 阶段 | 主角 | 住在哪 | 大小 | 会干啥 | 不会干啥 |
| --- | --- | --- | --- | --- | --- |
| 1 BIOS | BIOS | 主板 ROM 芯片 | 几 MB | 自检、读第 0 扇区 | 不认识文件系统、不认识 OS |
| 2 MBR | 446 字节跳板 | 硬盘 LBA 0 | 446 B | 找活动分区、读该分区第 1 扇区 | 不认识文件 |
| 3 Bootloader | GRUB stage2 | 硬盘 `/boot/grub/` | 几 MB | 读内核文件、显示菜单 | 不会管理进程 |
| 4 内核 | Linux / NT | 硬盘 `/boot/vmlinuz` | 几十 MB | 管硬件、管内存、管进程 | 不直接服务用户 |
| 5 PID 1 | systemd | 硬盘 `/sbin/` | 几 MB | 拉起所有服务 | 不做内核的事 |

### 开机流程的精简版

- **一句话总路径：**

    ```
    电源键 ─► BIOS ─► MBR ─► Bootloader ─► 内核 ─► systemd ─► 桌面
    ```

#### ① BIOS（主板 ROM 里，开机自动跑）

- 自检硬件（CPU、内存、硬盘在不在）。
- 按启动顺序找第一块带 `55 AA` 的硬盘。
- 读它的**第 0 扇区（512 字节）**到内存 `0x7C00`，跳过去。
- 退场。

#### ② MBR（硬盘 LBA 0，446 字节）

- 扫分区表，找活动分区（标志 `0x80`）。
- 读该分区的第一个扇区到内存，跳过去。
- 退场。

> 现代 Linux 下，这个"第一个扇区"其实是 GRUB stage 1.5 的开头。

#### ③ Bootloader（比如 GRUB，分三段接力）

- **三段接力概览：**

    ```
    stage 1   (446B, 在 MBR 里)
        └─► 只会读下一段
    stage 1.5 (~30KB, 在 MBR 后隙)
        └─► 学会认文件系统，能按文件名读了
    stage 2   (几 MB, 在 /boot/grub/)
        └─► 真正的 GRUB，画菜单、读内核
    ```

- **stage 2 干的事：**

    - 读 `/boot/grub/grub.cfg`，画菜单。
    - 你选了条目后：
        - 读 `/boot/vmlinuz`（压缩的内核）到内存。
        - 读 `/boot/initrd.img`（临时文件系统）到内存。
        - 设置内核参数（`root=UUID=... ro quiet ...`）。
    - CPU 从 16 位实模式切到 64 位保护模式。
    - 跳到内核入口，退场。

#### ④ 内核

- 自解压（`vmlinuz` 是压缩包，解压器自己解自己）。
- 初始化：内存管理、调度器、中断、驱动框架。
- 解压 initrd 挂为 `/`（临时根，在内存里）。
- 启动 PID 1 = initrd 里的 `/init`（临时的）。
- **`/init` 干的事：**
    - 加载真硬盘/文件系统驱动（ext4、nvme...）。
    - 按 UUID 找到真根分区。
    - 挂到 `/new_root`。
    - `switch_root`：根切到真硬盘，丢掉 initrd，PID 1 换成 `/sbin/init`（= systemd）。

#### ⑤ systemd（真正的 PID 1）

- 并行拉起所有服务（网络、日志、SSH...）。
- 启动图形登录 gdm → 登录界面。
- 输密码 → 桌面。

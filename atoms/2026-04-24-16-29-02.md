# IMC：抹平 CPU 差异

### 问题

- **指令集代差** 集群内主机 CPU 代际不同时，VM 在高代 CPU 上启动后使用了高级指令集（如 AVX-512），热迁移到低代 CPU 后因指令集不支持而抛出“非法指令”异常，导致蓝屏或内核恐慌。

### 原理

- **向下取平** IMC（Intelligent Machine Check / Compatibility）设定基准指令集（Baseline），所有主机对 VM 只展示基准线以内的功能，消除迁移时的“水土不服”。

- **拦截 CPUID** IMC 在 VM 启动前强行拦截 CPUID 指令的返回结果，把新 CPU 的高级特性“藏起来”，告诉 VM 全集群只支持最基础的指令集。

- **代价** 牺牲新 CPU 的部分性能潜力，换来跨主机自由迁移的安全性。

### 注意

- **先开启再开机** IMC 策略只能作用于关机状态下启动的 VM。已开机的 VM 已读取了真实 CPU 信息，再开启 IMC 不会生效。

- **Intel 与 AMD 不可混用** 两者的指令集架构完全不同，IMC 也无法兼容，热迁移必败。

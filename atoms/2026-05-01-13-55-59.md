# Appsmith 创建调用后端 API 的页面

### 一、明确页面要解决的问题

- **页面目标** 在 Appsmith 中创建一个“数据抓取页”，让用户填写书名或 ISBN 后点击“爬取图书”，页面负责校验输入、调用后端图书抓取接口、提示请求结果，并把成功返回的图书信息展示在表格中。

- **最低输入要求** 书名和 ISBN 至少填写一个；如果两个都为空，不应该请求后端，而是直接提示“请填写书名或 ISBN”。

### 二、理解后端接口的调用方式

- **接口地址** 后端图书抓取接口使用 `GET /api/books/lookup`。

- **完整地址** Appsmith 中请求的完整 URL 是：

    ```text
    http://47.128.236.133:16666/api/books/lookup
    ```

- **请求参数** 接口支持 `title` 和 `isbn` 两个查询参数。

    - `title` 表示书名
    - `isbn` 表示 ISBN
    - 两个参数至少传一个

- **按书名查询** 如果用户按书名查询，请求示例是：

    ```text
    http://47.128.236.133:16666/api/books/lookup?title=战争与和平
    ```

- **按 ISBN 查询** 如果用户按 ISBN 查询，请求示例是：

    ```text
    http://47.128.236.133:16666/api/books/lookup?isbn=9787020071203
    ```

### 三、确认接口返回的数据形态

- **成功结构** 接口成功时，`code` 为 `0`，`message` 为 `success`，真正的图书数据放在 `data` 字段里。

    ```json
    {
      "code": 0,
      "message": "success",
      "data": {
        "id": 5,
        "title": "战争与和平",
        "author": "托尔斯泰, 辽逸刘",
        "isbn": "9787020071203",
        "category": "其他",
        "description": "本书...",
        "coverUrl": "...",
        "totalQuantity": 0,
        "availableQuantity": 0,
        "status": "AVAILABLE",
        "createdAt": "2026-05-01 05:47:..."
      }
    }
    ```

- **失败结构** 接口失败时，`data` 可能是 `null`，失败原因通常放在 `message` 里。

    ```json
    {
      "code": 20000,
      "message": "Google Books 调用失败，HTTP=429",
      "data": null
    }
    ```

- **页面处理思路** 页面只需要判断 `LookupBook.data?.data` 是否存在；存在就包装成单行数组交给 Table，不存在就展示空数组。

### 四、新建 Appsmith 页面和输入控件

1. 在 Appsmith 左侧页面栏新增页面，命名为“数据抓取页”。

2. 拖入一个 Input 控件，用来输入书名，控件名称改为 `InputTitle`。

3. 再拖入一个 Input 控件，用来输入 ISBN，控件名称改为 `InputIsbn`。

4. 确认控件名称和后续 API 参数、JS 代码中的引用保持一致，否则按钮点击时会读不到输入值。

### 五、配置 LookupBook API Query

1. 在左侧点击 `Queries`，新建一个 API，命名为 `LookupBook`。

2. 请求方法选择 `GET`。

3. URL 填写后端完整地址：

    ```text
    http://47.128.236.133:16666/api/books/lookup
    ```

4. 在 API 的 Params 中添加参数 `title`，值填写：

    ```js
    {{InputTitle.text}}
    ```

5. 在 API 的 Params 中添加参数 `isbn`，值填写：

    ```js
    {{InputIsbn.text}}
    ```

### 六、用按钮触发查询前校验和请求

1. 拖入一个 Button 控件，按钮文字设置为“爬取图书”。

2. 在按钮的 `onClick` 中选择 `Execute a JS function`，让按钮点击时执行自定义 JS 方法。

3. 在 JS 页面中编写 `runFetch` 方法，用它完成输入校验、接口请求和结果提示。

    ```js
    export default {
      runFetch() {
        InputTitle.text || InputIsbn.text
          ? LookupBook.run(
              () => showAlert("图书查询完成", "success"),
              () => showAlert(LookupBook.data?.message || "图书查询失败", "error")
            )
          : showAlert("请填写书名或 ISBN", "warning");
      }
    };
    ```

4. 按钮的 `onClick` 最终绑定这个 JS 方法，例如调用 `JSObject1.runFetch()`；如果 JS 对象名称不是 `JSObject1`，就按实际名称替换。

### 七、用表格承接查询结果

1. 在页面下方拖入一个 Table 控件，用来展示接口返回的图书信息。

2. 将 Table 的 `Table data` 绑定为：

    ```js
    {{ LookupBook.data?.data ? [LookupBook.data.data] : [] }}
    ```

- **绑定逻辑** 因为接口成功时返回的是单个图书对象，而 Table 需要数组，所以要把 `LookupBook.data.data` 包装成 `[LookupBook.data.data]`。

- **空态处理** 当接口还没调用、调用失败，或者 `data` 为空时，Table 接收 `[]`，页面不会因为空数据报错。

# VRF（Virtual Routing and Forwarding，虚拟路由转发）

## 一、是什么

- **一句话总结：** 让一台路由器里同时跑多张相互独立、彼此互不可见的路由表，于是相同的 IP 段可以在同一台设备上共存而互不干扰。

- **本质：** 把“一台物理路由器”在逻辑上切成多台互不相识的虚拟路由器。

- **解决的问题：** IP 地址重叠（address overlap）与多租户隔离。

## 二、没有 VRF 的困境

- **场景：** 一台路由器接两个客户端，两边用了完全相同的网段。

    ```text
    PC-A1 (192.168.1.10) ---- g0/0 [ R1 ] g0/1 ---- PC-B1 (192.168.1.10)
    ```

- **配置直接报错：** 给第二个接口配相同网段时被拒。

    ```text
    R1(config)#interface g0/0
    R1(config-if)#ip address 192.168.1.1 255.255.255.0
    R1(config-if)#no shutdown
    R1(config-if)#exit
    R1(config)#interface g0/1
    R1(config-if)#ip address 192.168.1.1 255.255.255.0

    % 192.168.1.0 overlaps with GigabitEthernet0/0
    ```

- **报错根因：** 一台路由器只有一张全局路由表，表里不允许出现两条相同的 `192.168.1.0/24` 直连路由——转发时无法判断该把包送向 `g0/0` 还是 `g0/1`，所以根本配不上去。

- **换不同网段也做不到隔离：** 若 A 用 `.1.0/24`、B 用 `.2.0/24`，全局表里大家互相可见，PC-A1 默认就能 ping 通 PC-B1。

## 三、有了 VRF

- **第一步，建两个 VRF：**

    ```text
    R1(config)#ip vrf CUST_A
    R1(config-vrf)#exit
    R1(config)#ip vrf CUST_B
    R1(config-vrf)#exit
    ```

- **第二步，把接口划进各自的 VRF：** 划入后再配地址，两个接口都成功配上了相同的 `192.168.1.1/24`，没有任何报错。

    ```text
    R1(config)#interface g0/0
    R1(config-if)#ip vrf forwarding CUST_A
    R1(config-if)#ip address 192.168.1.1 255.255.255.0
    R1(config-if)#no shutdown
    R1(config-if)#exit
    R1(config)#interface g0/1
    R1(config-if)#ip vrf forwarding CUST_B
    R1(config-if)#ip address 192.168.1.1 255.255.255.0
    R1(config-if)#no shutdown
    ```

- **验证，相同网段分别待在各自的表里：**

    ```text
    R1#show ip vrf
      Name           Interfaces
      CUST_A         GigabitEthernet0/0
      CUST_B         GigabitEthernet0/1

    R1#show ip route vrf CUST_A
      C  192.168.1.0/24 is directly connected, GigabitEthernet0/0

    R1#show ip route vrf CUST_B
      C  192.168.1.0/24 is directly connected, GigabitEthernet0/1
    ```

- **结论：** 同样的 `192.168.1.0/24` 现在安静地待在两张互不干扰的表里；`show ip route`（全局表）里压根看不到这两条路由——它们只属于各自的 VRF。

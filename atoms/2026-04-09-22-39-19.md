# VMware 虚拟机 IP 的识别机制

- **硬件黑盒限制**：ESXi 处于虚拟化底层，默认无法穿透硬件抽象层去感知 Guest OS 的内部 IP。这种“可见性封锁”要求必须通过 VMware Tools 建立一条跨越边界的“通信专线”。

### 获取方法

- **上报链条**：虚拟机 IP 的识别路径为 Guest OS (网络配置) -> VMware Tools (搬运工) -> ESXi/vCenter (展示层)。运行在虚机内的 `vmtoolsd` 进程插件捕获 IP 后主动上报，并在管理界面的 Summary 页面展示。

### 工程意义：自动化迁移的基石

- 在 VMware → FusionCompute 的迁移工程中，此知识点直接决定了“自动化的程度”：

- **自动化校验**：若无法通过 API 动态检索虚机 IP，将导致迁移后的“存活率与连通性”无法进入无人化验证流程。

- **IP 映射与改写**：如果无法感知源端 IP，迁移引擎就无法自动完成目标端静态 IP 的配置下发。

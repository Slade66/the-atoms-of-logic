# VirtIO

- **VirtIO 是什么？**

    - 半虚拟化技术（即操作系统知道自己是虚拟机，并配合底层的超级通道传输数据）。
    
    - 

- **为什么非要用 Virtio？**

    - 因为 KVM 其实是可以模拟上古硬件的。如果你在华为 FusionCompute 创建虚拟机时，不把磁盘挂载模式选为 Virtio，而是强行在界面的“硬盘控制器”选项里，选为 IDE 或 SATA，所有现代操作系统都原生内置了通用的 IDE 驱动，这种“上古时代”的通用协议，所有的系统连急救包都不看就能认识。它当然能开机，此时你的 CentOS 完全不需要任何 Virtio 注入，无缝顺利开机。

    - 代价： 每一次读写，都要强制触发 CPU 的“陷入和模拟 (Trap and Emulate)”。在 IDE 模式下，磁盘 IOPS 会暴跌 70% 以上，网卡可能连 1 Gbps 都跑不满。对企业级业务来说，这种性能等同于自杀。华为不推荐，也不把这当做生产方案。

- **virtio 是什么？**

    - virtio：这是总框架。它是 KVM 虚拟化环境中最快的一条“高速公路”协议，全称“虚拟 I/O”。

    - virtio_ring：高速公路上的“传送带”，负责在虚拟机和华为宿主机物理层之间高效搬运数据的内存环机制。
    
    - virtio_pci：接口转换器。它让这条高速公路能伪装成普通的 PCI 接口插在主板上，让操作系统有地方可以对接。
    
    - virtio_blk（极其关键）：虚拟块存储驱动。有了它，系统才能认出华为 FC 挂载的硬盘是个啥玩意儿。这就是那把钥匙 B。
    
    - virtio_net：虚拟网卡驱动。有了它，系统进去后才能认出网卡，配置上真实的 IP 地址，否则网络就是瘫痪的。

- **virtio 驱动在哪下载？**

    - 对于 Linux，你根本不需要去任何地方下载它。它从头到尾都“沉睡”在你的系统里！
    
    - virtio 虽然在华为 FC 里大显神威，但它并不是华为自己开发的私有专利技术。它是全世界 KVM 虚拟化（开源界）的通用标准协议。
    
    - 由于它是全球开源标准，Linux 操作系统之父 Linus Torvalds 早就把 virtio 系列驱动的代码原生地集成到了 Linux 官方主分支内核里。
    
    - 所以，无论你装的是 CentOS、Ubuntu 还是 Debian，系统在安装时，这些驱动就已经以 .ko 模块文件的形式，静静地躺在你硬盘的 /lib/modules/内核版本/kernel/drivers/ 目录下了。
    
    - 对于 Windows 阵营： Windows 系统在出厂时，天生拒绝内置这些开源界的 virtio 驱动。此时，你就必须真正地去下载外置驱动了。Fedora 维护着 virtio-win.iso，在它的网站上下载它。

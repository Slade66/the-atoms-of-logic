# SSH 远程主机身份变更报错的原因与处理

### 问题背景

- **报错场景** 执行 `ssh-copy-id root@10.30.80.153` 时，SSH 提示远程主机身份发生变化，并拒绝继续连接。

    ```bash
    [root@longxi216 ~]# ssh-copy-id root@10.30.80.153
    /usr/bin/ssh-copy-id: INFO: Source of key(s) to be installed: "/root/.ssh/id_rsa.pub"
    /usr/bin/ssh-copy-id: INFO: attempting to log in with the new key(s), to filter out any that are already installed

    /usr/bin/ssh-copy-id: ERROR: @@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
    ERROR: @    WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!     @
    ERROR: @@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
    ERROR: IT IS POSSIBLE THAT SOMEONE IS DOING SOMETHING NASTY!
    ERROR: Someone could be eavesdropping on you right now (man-in-the-middle attack)!
    ERROR: It is also possible that a host key has just been changed.
    ERROR: The fingerprint for the ECDSA key sent by the remote host is
    ERROR: SHA256:7fD686YEUkXY915N1zIjDTjZRbHN59mzYxmof531/Ho.
    ERROR: Please contact your system administrator.
    ERROR: Add correct host key in /root/.ssh/known_hosts to get rid of this message.
    ERROR: Offending ECDSA key in /root/.ssh/known_hosts:17
    ERROR: ECDSA host key for 10.30.80.153 has changed and you have requested strict checking.
    ERROR: Host key verification failed.
    ```

### 根因分析

- **信任机制** SSH 第一次连接服务器时，客户端会把服务器公钥指纹保存到本地 `~/.ssh/known_hosts`，后续连接都用这条记录确认服务器身份。如果本地记录的指纹不等于服务器当前返回的指纹，SSH 会认为服务器身份异常，并拒绝连接。

- **本次冲突** `/root/.ssh/known_hosts` 第 17 行记录的 `10.30.80.153` 指纹，与该 IP 当前 SSH 服务返回的 ECDSA 公钥不匹配。

    - `Offending ECDSA key in /root/.ssh/known_hosts:17`
    - `ECDSA host key for 10.30.80.153 has changed`

- **中止链路** `known_hosts` 按连接历史追加写入，第 17 条说明这台机器之前连接过，并留下了旧记录；新指纹与旧记录冲突后，SSH strict checking 模式直接中止，不允许自动覆盖。

    1. 服务器端 SSH 主机密钥发生变更。
    2. 客户端本地 `known_hosts` 存有旧指纹。
    3. SSH 指纹比对失败。
    4. 为防止中间人攻击，连接被强制终止。

- **可能原因** 指纹变化通常来自以下情况。

    1. 目标机器重装系统后，SSH 主机密钥对重新生成，指纹必然改变。
    2. 原来的 `10.30.80.153` 下线，新机器复用了该 IP，但 SSH 主机密钥不同。
    3. 有人在网络链路上伪造该 IP 的 SSH 服务，即 MITM。


### 解决方案

- **推荐方案** 使用 `ssh-keygen -R` 删除旧的 host key 记录，然后重新连接并确认新指纹。

    1. 删除旧的 host key。

        ```bash
        ssh-keygen -R 10.30.80.153
        ```

        这会自动从 `known_hosts` 中移除该 IP 的旧记录。

    2. 重新连接。

        ```bash
        ssh-copy-id root@10.30.80.153
        ```

        首次连接时输入 `yes` 接受新指纹即可。

- **手动方案** 也可以直接编辑 `known_hosts`，删除第 17 行旧记录。

    ```bash
    vim /root/.ssh/known_hosts
    ```

    ```bash
    sed -i '17d' /root/.ssh/known_hosts
    ```

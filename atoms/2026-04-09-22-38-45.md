# ARP 协议 和 arp-scan 命令

### ARP 协议

- **ARP 的作用：** 把 IP 地址 → 转换成 MAC 地址

- **IP 和 MAC 有什么用？**
    
    - IP 用来“逻由”（三层）
    
    - MAC 用来“实际发包”（二层）

- **工作流程：**

    - 当主机 A 尝试访问局域网内的 192.168.1.10 时：

    1.  **查缓存**：主机 A 首先检索本地 ARP 缓存表，检查是否存在目标 IP 对应的 MAC 地址。
    
    2.  **发广播 (Request)**：若缓存未命中，主机 A 向全网段发送 ARP 请求广播（广播问）：“谁是 192.168.1.10？请回复 A”。
    
    3.  **回单播 (Reply)**：目标主机匹配到自己的 IP 后，向 A 发回 单播响应（单播答）：“我是 192.168.1.10，我的 MAC 是 `xx:xx`”。
    
    4.  **本地缓存**：主机 A 将结果存入缓存，随后开始正式的数据封装与发送。

### arp-scan 如何扫描存活设备？（技术原理）

1.  **全路段遍历**：它会自动获取当前网段范围（如 `192.168.1.0/24`），并遍历其中的所有可用 IP 地址。

2.  **并发请求**：它会向网段内所有主机并发发送 ARP 请求广播（Who is IP?）。

3.  **结果判定**：
    
    - **存活主机**：根据 TCP/IP 规范，必须回送 ARP 响应（I am IP, MAC is...）。
    
    - **离线主机**：无响应。

4.  **厂商识别**：它会将收集到的 MAC 地址前缀与内置的数据库比对，从而推断出设备厂商（如：Apple, VMware, Huawei）。

- **为什么它比 `ping` 更专业？**
    
    - **不可阻挡性**：许多设备会通过防火墙屏蔽 ICMP（导致 ping 不通），但任何接入二层网络的设备都无法禁掉 ARP，否则它将无法进行正常的内网通信。因此，`arp-scan` 的扫描结果比 ping 更真实、更具穿透力。

### arp-scan 命令的用法

- 主动发 ARP，扫描整个局域网的设备。

- **常用命令：** `arp-scan -l`（扫描本地网段所有在线设备）。

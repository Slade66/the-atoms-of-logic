# AI 算力平台的前置知识学习路线

- 按这个顺序学，每一层都是下一层的地基，不要跳层。

## 一、第 0 层 · 绝对前置

- **Linux 基础 + 命令行：** SSH、文件权限、进程、`systemd`、看日志（`journalctl` / `dmesg`）、网络排查（`ip` / `ss` / `ping`）。整个平台跑在 Linux 上，是地基的地基。

- **网络基础：** IP / 子网、DNS、端口、负载均衡、反向代理（Nginx）、HTTP / gRPC。多机训练的卡间通信、模型网关都要用到。

## 二、第 1 层 · 平台核心底座

- **容器 / Docker：** 镜像、容器、`Dockerfile`、镜像仓库、`containerd`。最该先吃透的一环——平台里所有东西（训练、推理、Volcano、HAMi）本质都是容器，不懂容器就完全无从理解 K8s。

- **Kubernetes（重中之重）：** Pod / Deployment / Service / Ingress、调度原理、`kubectl`、YAML、Namespace、RBAC、存储（PV / PVC / StorageClass）。整个项目的主战场，要花最多时间，先学会普通 K8s 应用编排再往下。

- **K8s 进阶（CRD + Operator + Device Plugin）：** 怎么扩展 K8s。Volcano 是一套 CRD + 调度器，GPU / NPU 纳管靠 device plugin。不懂这套机制就只能当用户，改不动核心。

## 三、第 2 层 · 算力与调度（核心竞争力）

- **GPU / NPU 基础：** 显存、算力、驱动、CUDA（NVIDIA）/ CANN（华为昇腾）、`nvidia-smi` / `npu-smi`、NVLink vs PCIe 拓扑、MIG / vGPU 概念。主搞华为，则昇腾 CANN + MindCluster + `npu-smi` 是重点。

- **Volcano + 批调度理论：** Gang 调度、FairShare、优先级抢占、队列 / 配额、拓扑感知。平台最难也最值钱的一层。

- **GPU 虚拟化 / 共享：** HAMi（NVIDIA 系）、昇腾的虚拟化（华为那套），实现一卡多用。

## 四、第 3 层 · 训练与推理两条业务线

- **分布式训练：** PyTorch DDP、AllReduce、数据 / 模型并行、检查点；华为侧的 MindSpore；显存优化 DeepSpeed。

- **大模型推理：** vLLM、推理服务化、KV Cache、量化等概念（会用即可，不必精通原理）。

## 五、第 4 层 · 平台化 / 可观测 / 安全（让它像个产品）

- **可观测：** Prometheus + Grafana + 各类 Exporter。

- **存储：** 对象存储（OBS / S3）、网络存储。

- **权限 / 多租户：** RBAC、OAuth2、LDAP。

- **后端开发能力：** Go（K8s / Volcano 生态都是 Go，改源码 / 写 Operator 必备），写 API、做网关。

## 六、学习顺序与避坑

- **主线顺序：** 严格按地基往上爬，每一步通了再迈下一步。

    1. Linux 基础打底
    2. 吃透容器 / Docker
    3. 学 Kubernetes 应用编排
    4. 学 CRD / Operator / Device Plugin
    5. 碰 GPU / NPU 纳管 + Volcano 调度

- **精力分配：** Docker + Kubernetes 这两步占 60% 的精力，是整条路线的重心，值得花最多时间。

- **核心门槛：** 只有 K8s 扩展机制（CRD / Operator / Device Plugin）通了，才有资格碰 GPU / NPU 纳管和 Volcano 调度这两个核心，训练、推理、监控、计费都是后面的事。

- **避坑提醒：** 别一上来就啃 Volcano 或昇腾底层，它们属于第 2 层，缺了第 1 层的 K8s 地基硬啃会直接劝退。

# Appsmith 前端插入数据时不要传数据库管理的时间字段

### 一、问题背景

- **触发场景** 在 Appsmith 里新增公告数据时，如果页面手动给数据库时间字段传值，可能会因为 JS 的时间格式不符合 MySQL 的 `DATETIME` 要求而报错。

- **解决思路** 这张公告表已经给时间字段配置了默认值，所以新增公告时不必从 Appsmith 手动传时间，直接让数据库自动生成即可。

### 二、确认公告表时间字段的默认行为

- **时间字段** 公告表里有 3 个和时间相关的字段：`published_at`、`created_at`、`updated_at`。

    ```sql
    published_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '发布时间',
    created_at   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间'
    ```

- **共同特点** 这 3 个字段都是 `DATETIME NOT NULL`，但它们同时也配置了默认值，所以插入数据时不一定要手动传时间。

- **发布时间** `published_at` 默认使用当前时间。

- **创建时间** `created_at` 默认使用当前时间。

- **更新时间** `updated_at` 默认使用当前时间，并且每次执行 `UPDATE` 时会自动刷新成更新时的时间。

### 三、插入公告时只写业务字段

1. 插入公告时，只需要写真正由页面提交的业务字段。

2. 例如只插入 `title`、`content`、`status`。

    ```sql
    INSERT INTO announcements (title, content, status)
    VALUES (...);
    ```

3. 不需要在 Appsmith 里额外拼 `published_at`、`created_at`、`updated_at`。

### 四、让数据库自动补齐时间字段

- **自动补值** 当插入 SQL 没有显式传入时间字段时，数据库会自动补上默认值。

    ```text
    published_at = 当前时间
    created_at = 当前时间
    updated_at = 当前时间
    ```

- **避免报错** 这样 Appsmith 不需要处理时间格式，也就不会因为前端传入的时间格式不符合数据库要求而报错。

- **更新行为** 后续如果更新这条公告，`updated_at` 会因为 `ON UPDATE CURRENT_TIMESTAMP` 自动变成更新时的时间。

### 五、前端不要传数据库托管的字段

- **通用原则** 前端页面只提交业务真正需要修改的字段；凡是已经由数据库默认值、自动更新时间或触发规则托管的字段，除非有明确业务原因，否则都不要手动传值。

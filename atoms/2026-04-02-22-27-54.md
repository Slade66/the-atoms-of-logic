# FusionCompute

- **FusionCompute 是什么？** FusionCompute 是一套云操作系统，专门负责将物理服务器的硬件资源（CPU、内存、存储、网络）池化，并根据需要分配给各个虚拟机。

- **FusionCompute 的架构：**

    - **VRM (Virtual Resource Management)：管理中枢**
    
        - 它负责整个集群的资产管理、资源调度、虚拟机生命周期管理以及对外提供 Restful API。

        - 你可以通过浏览器登录 VRM 的 Web 界面进行所有配置。

    - **CNA (Computing Node Agent)：执行节点**

        - 安装在每一台物理服务器上的软件。它基于 KVM 深度定制，直接负责虚拟机的运行和硬件指令的解析。

- **基于 KVM 深度定制：** 早期版本基于 Xen，现在都基于 KVM，因此它可以非常好地支持 qcow2 等开源镜像格式。

- **国产化适配：** 对国产 CPU（华为鲲鹏、海光等）和国产操作系统（麒麟、统信等）有极佳的原生支持。

- FusionCompute 支持导入 OVF 模板文件或 QCOW2、.raw，以及华为自研的 .zvhd2。

- **原生 REST：** 与 VMware 拥有官方 SDK（govmomi）不同，华为官方没有提供专门针对 Go 或 Python 的开源 SDK，FusionCompute 对外暴露的是一套纯粹的 RESTful API。因此，连接 FusionCompute 的标准做法就是直接使用 HTTP 客户端库构造 REST 请求。

- **Token 鉴权：**

    1. 登录 (POST /service/session)：向 VRM 发送你的用户名和密码。

    2. 获取令牌 (Token)：如果密码正确，VRM 会在 HTTP 响应头 (Response Headers) 中返回一个名为 X-Auth-Token 的字符串。

    3. 拿着令牌办事：在后续所有的操作（如查询集群、创建卷、上传镜像）中，把这个 Token 塞进请求头里即可。

- **硬盘的逻辑大小：** 你需要知道硬盘的 “分配容量（配置大小）” 是多少，在华为 FC 端创建目标卷时，目标卷的大小必须 ≥ 这个 原始 VMDK 的逻辑分配容量，否则数据灌不进去。

# FusionCompute 网络：DVS 与 Port Group

### 层级关系

- **链路** 站点（Site）→ 分布式交换机（DVS）→ 端口组（Port Group）→ 虚拟机网卡（vNIC）。
- **划分** 一个 DVS 下可有多个端口组，通常按业务（管理网、业务网、存储网）或 VLAN 划分。

### DVS：分布式虚拟交换机

- **是什么？** Distributed Virtual Switch，跨越多台物理主机的虚拟交换机，为 FusionCompute 提供统一网络。

- **核心能力** VM 跨主机迁移后仍处于同一虚拟网络，无需重新配置网络。

- **逻辑架构** 虚拟机通过端口组与 DVS 相连，DVS 通过上行链路与物理网络相连。

    ```
    VM1 (Host 1) ─┐
                  ├──→ DVS（跨多个主机）
    VM2 (Host 2) ─┤    │
                  │    └──→ 上行链路 → 物理网络
    VM3 (Host 3) ─┘
    ```

### Port Group：批量管理网络属性

- **定义** 一组具备相同网络属性（VLAN、MAC 绑定、DHCP 隔离等）的虚拟交换端口集合。

- **作用** 简化管理（批量配置而非逐个配置）、提供隔离（不同端口组可有不同 VLAN 和安全策略）、逻辑划分（按业务需求分组）。

- **工作机制** 创建端口组并定义属性，VM 加入后自动继承，无需逐个配置网卡。

### 端口类型：Access 与 Trunk

- **Access（通常为 0）** 一个口对应一个 VLAN，绝大多数业务 VM 都应选此类型。

- **Trunk（通常为 1）** 可同时通过多个 VLAN 的流量，给路由器、防火墙等需要自己处理 VLAN 标签的虚拟化设备使用。

### 类比

- **端口组 = 公寓楼层** 同一楼层的住户有相同的网络规则，楼层经理一次性配置规则，所有住户自动适用。

- **DVS = 整栋楼的网络系统** 跨楼层统一管理，住户换楼层（VM 迁移）后网络配置不变。

### VM 创建到联网的流程

1. 管理员在 VRM Web 界面创建虚拟机。
2. VRM 通过调度算法选择最合适的 CNA（如 Host 1）。
3. VRM 通知该 CNA 创建 VM。
4. CNA 通过 KVM Hypervisor 创建虚拟机。
5. VM 的虚拟网卡连接到端口组（如 Port Group 1）。
6. 端口组属于 DVS，DVS 通过上行链路连接到物理交换机。
7. VM 可与集群内其他主机通讯，也可与外部网络通讯。

# 盒式交换机的内部结构

## 一、以 CE5850-48T4S2Q-EI 为例

- **端口配置：** 共有 48 个 1000M 接入口、4 个 10G 上行口和 2 个 40G 上行口。

- **内部结构：** 主面板固定口归在子卡号 `0`，可选的扩展子卡归在子卡号 `1`。

    ```text
    一台盒子（功能全集成，没有独立主控板 / 交换网板）
    │
    ├─ 主面板固定口（子卡号 = 0）
    │    ├─ 48 个千兆电口   → GE0/0/1 ~ GE0/0/48
    │    ├─ 4 个 10G 上行口 → 10GE0/0/x
    │    └─ 2 个 40G 上行口 → 40GE0/0/x
    │
    └─ 扩展子卡槽（子卡号 = 1，如果设备带的话）
         └─ 插入子卡后       → XGE0/1/1、XGE0/1/2 …
    ```

## 二、端口三段命名

- **命名格式：** 端口名形如 `GE0/0/1`，按“第一段 / 第二段 / 第三段”拆解。

    1. 第一段（设备位）：含义随场景变化，最容易踩坑——盒式设备没有多个机框槽位可数，单机时固定为 `0`；堆叠时变成堆叠成员 ID，用来区分是堆叠里的哪台物理设备。

    2. 第二段（子卡号）：盒式设备真正有意义的扩展位——主面板固定口是 `0`，带上行扩展子卡时子卡上的口是 `1`（如 `XGE0/1/1`）。

    3. 第三段（端口序号）：板上物理口的编号。

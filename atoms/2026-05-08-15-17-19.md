# Docker 网络的 Bridge 模式

### 一、核心理解

- **基本作用** Docker 的 bridge 模式会在宿主机上创建一个虚拟网桥，容器通过这个网桥互相通信。

- **连接方式** 每个容器都会有一对 `veth pair`，一端在容器内叫 `eth0`，另一端在宿主机上挂到 Docker 网桥。

    ```text
    宿主机
    └── docker0 (网桥 172.17.0.1)
        ├── veth3a1b2c  <── peer ──>  容器 A 的 eth0 (172.17.0.2)
        └── veth9d4e5f  <── peer ──>  容器 B 的 eth0 (172.17.0.3)
    ```

### 二、常用命令

- **查看网络** 使用 `docker network ls` 查看 Docker 当前已有的网络。

    ```bash
    docker network ls
    ```

- **创建网络** 使用 `docker network create` 创建自定义 bridge 网络，实际使用时通常推荐使用自定义网络。

    ```bash
    docker network create my-network
    ```

- **查看详情** 使用 `docker network inspect` 查看某个 bridge 网络的详情，输出里的 `Containers` 字段会列出所有接入该网络的容器及其 IP。

    ```bash
    docker network inspect my-network
    ```

- **连接容器** 启动容器时通过 `--network` 指定自定义网络，让多个容器加入同一个 bridge 网络。

    ```bash
    docker run -d --name app1 --network my-network nginx
    docker run -d --name app2 --network my-network redis
    ```

### 三、推荐自定义 Bridge 的原因

- **内置 DNS** 用 `docker network create` 创建的自定义 bridge 网络，会内置 Docker DNS 服务器；容器之间不需要知道对方 IP，可以直接通过容器名或服务名互相访问。

- **默认限制** 默认 `bridge` 网络不提供同样的容器名 DNS 解析能力，通常只能通过 IP 通信。

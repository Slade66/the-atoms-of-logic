# docker update 命令

- **作用：** 修改一个已存在容器的运行时配置，原地生效，不重建容器。

- **能改（资源类配置 + 重启策略）：** 都是运行时可以热调的字段。

    - **重启策略：** `--restart`，四种取值如下表。

        | 策略 | 含义 |
        |------|------|
        | `no` | 永不自动重启，容器停了就一直躺着（默认值；事故的根源）。 |
        | `on-failure[:N]` | 只有非正常退出（exit code ≠ 0）才重启，`:N` 可限重试次数；正常退出或被手动停不管。 |
        | `always` | 不管因为什么停的都拉起来；dockerd 自身启动时（如开机）也会把它拉起来。 |
        | `unless-stopped` | 跟 `always` 几乎一样，唯一区别：若你手动 `docker stop` 过它，则之后 daemon 重启它也保持停着、不自动起。 |

    - **资源限额（常用就这几个）：**

        | 类别 | 参数 | 含义 |
        |------|------|------|
        | CPU | `--cpus` | 限制最多使用多少个 CPU 核，如 `--cpus 1.5`。 |
        | 内存 | `-m/--memory` | 内存硬上限，超过触发 OOM kill，如 `-m 512m`。 |
        | 内存 | `--memory-swap` | 内存 + swap 总上限；设成和 `--memory` 相等即禁用 swap。 |

- **改不了（得删了重建）：** 端口映射 `-p`、挂载 `-v`、环境变量 `-e`、镜像、网络模式、`--entrypoint` 等，只能 `docker rm` 后重新 `docker run`。

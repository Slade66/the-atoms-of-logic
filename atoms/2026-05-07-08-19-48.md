# Windows 驱动安装的本质

## 一、核心定义

- **安装本质** Windows 驱动安装分为两个阶段：先把经过验证的驱动文件放入受保护的驱动仓库，也就是 Stage；再由系统根据硬件 ID 自动匹配并加载进内核，也就是 Install。

## 二、关键角色

- **INF 文件** `.inf` 文件是驱动的“说明书”，是安装信息，声明驱动支持哪些硬件 ID、需要安装哪些文件、需要写入哪些注册表项。

- **SYS 文件** `.sys` 文件是驱动的真正实体，是运行在 Windows 内核态的代码。

- **CAT 文件** `.cat` 文件是数字签名目录，用来证明驱动包没有被篡改。

- **Driver Store** Driver Store 是 Windows 受信任的驱动仓库，普通程序不能随意修改。

- **PnP 管理器** PnP 管理器是 Windows 内核组件，负责设备发现、硬件 ID 读取和驱动匹配。

## 三、Stage：暂存入库

- **仓库位置** Windows 维护的 Driver Store 位于 `C:\Windows\System32\DriverStore\FileRepository\`。

- **入库动作** Stage 会把驱动包复制进 Driver Store，通常包括 `.inf`、`.sys` 和相关文件。

- **合法性检查** Stage 阶段会验证驱动包是否符合系统要求。

    - `.inf` 文件格式是否合法。
    - 驱动是否有数字签名。
    - 驱动文件是否完整。

- **入库结果** 驱动进入 Driver Store 后，只代表这份驱动被系统认可，还没有安装到任何具体设备上。

## 四、Install：安装绑定

- **触发条件** 当硬件设备接入时，PnP 管理器会读取设备上报的硬件 ID，例如 `USB\VID_1234&PID_5678`。

- **匹配过程** PnP 管理器会扫描 Driver Store 中的 `.inf` 文件，寻找声明支持该硬件 ID 的驱动。

- **绑定动作** 找到匹配驱动后，Windows 会从 Driver Store 取出驱动文件，把 `.sys` 文件加载进内核，并在注册表中记录绑定关系。

- **失败结果** 如果 Driver Store 中没有匹配驱动，系统会提示“未找到驱动”。

- **完整流程** 驱动从硬件接入到设备可用，大致按以下顺序完成。

    1. 硬件设备插入。
    2. PnP 管理器读取设备硬件 ID。
    3. PnP 管理器扫描 Driver Store 中的 `.inf` 文件。
    4. 如果没有找到匹配驱动，系统提示“未找到驱动”。
    5. 如果找到匹配驱动，系统从 Driver Store 取出驱动文件。
    6. `.sys` 文件加载进 Windows 内核。
    7. 注册表记录设备和驱动的绑定关系。
    8. 设备正常工作。

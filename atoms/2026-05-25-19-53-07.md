# nohup 后台运行

## 一、核心作用

- **后台持续运行** `nohup` 的作用是让命令在退出终端后继续运行；通过 SSH 连接服务器时，即使关闭终端，程序也会忽略挂断信号，服务器里的任务可以继续执行。

- **名称来源** `nohup` 来自 `no hang up`，意思是忽略终端断开时发送的 `SIGHUP` 信号。

## 二、基础命令

- **运行服务** 例如要运行一个 Go 后端服务，可以把程序放到后台执行，并把日志写入 `server.log`。

    ```bash
    nohup ./server > server.log 2>&1 &
    ```

- **命令拆解** 这条命令由程序启动、日志重定向、错误合并和后台运行几部分组成。

    ```text
    nohup        # 忽略终端退出信号
    ./server     # 要运行的程序
    > server.log # 标准输出写入 server.log
    2>&1         # 标准错误也写入标准输出，也就是 server.log
    &            # 放到后台运行
    ```

    - 命令最后的 `&` 表示把整个命令放到后台运行，让终端不用一直卡在当前程序上，可以继续输入其他命令。

- **启动结果** 敲完后台命令后，终端可能输出类似 `[1] 23456` 的内容，其中 `23456` 是该后台任务的进程 ID，也就是 `PID`。

    ```text
    [1] 23456
    ```

## 三、日志输出

- **指定日志** `> download.log` 表示重定向标准输出，程序原本打印到屏幕上的内容会被写入 `download.log`。

- **合并错误** `2>&1` 表示把标准错误合并到标准输出里；这样程序的普通日志和报错信息都会进入同一个日志文件。

- **文件描述符** Linux 里常见的文件描述符包括标准输入、标准输出和标准错误。

    ```text
    0  标准输入 stdin
    1  标准输出 stdout
    2  标准错误 stderr
    ```

- **默认日志** 如果不指定日志文件，直接运行 `nohup ./server &`，默认输出通常会写到当前目录下的 `nohup.out`。

    ```bash
    nohup ./server &
    ```

    ```text
    nohup.out
    ```

- **查看日志** 程序放到后台后，可以用 `tail -f` 实时查看日志。

    ```bash
    tail -f download.log
    ```

## 四、进程管理

- **查找进程** 要停止 `nohup` 启动的进程，可以先用 `ps aux | grep` 查找进程。

    ```bash
    ps aux | grep server
    ```

- **停止进程** 假设查到的进程号是 `12345`，可以用 `kill` 停止它。

    ```bash
    kill 12345
    ```

- **后台任务** `jobs` 可以查看当前 shell 里的后台任务。

    ```bash
    jobs
    ```

    - `jobs` 只能看到当前 shell 里的后台任务；关闭终端后再打开新终端，`jobs` 看不到原来的后台任务，但进程可能仍然在运行。

## 五、资源监控

- **查看整体资源** 可以用 `top` 查看系统进程和资源占用。

    ```bash
    top
    ```

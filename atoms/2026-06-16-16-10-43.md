# MySQL TEXT 字段、索引与底层存储原理

## 一、TEXT 家族

- **用途：** TEXT 是 MySQL 用来存储长文本字符串的数据类型。

- **按容量分四种：**

    | 类型 | 最大容量 | 适用场景 |
    | --- | --- | --- |
    | `TINYTEXT` | 255 字节 | 短标签、摘要 |
    | `TEXT` | 65,535 字节（约 64KB） | 文章正文、评论，最常用 |
    | `MEDIUMTEXT` | 16,777,215 字节（约 16MB） | 较大文档、JSON |
    | `LONGTEXT` | 4,294,967,295 字节（约 4GB） | 完整网页、日志 |

- **限制的是字节而非字符：** 用 `utf8mb4` 时，一个汉字或 emoji 可能占 4 字节，实际能存的字符数少于字节上限。

## 二、VARCHAR 还是 TEXT

- **核心区别在存储位置：** VARCHAR 存在行内（受 65,535 字节的行大小限制约束）；TEXT 内容通常存在行外（off-page），行内只保留一个指针。所以表里有很大字段时，用 TEXT 能减小行本身体积，对查询其他字段的性能有好处。

- **VARCHAR 与 TEXT 本质相同：** 较长的 VARCHAR 同样可能被挪到溢出页；TEXT 本质上就是默认走溢出存储的变长大字段。

- **短且可预期用 VARCHAR：** 长度可控且不太大（比如几百字符以内）时优先用 `VARCHAR`，方便建索引、设默认值。

- **长或不确定用 TEXT：** 内容确实可能很大、长度不可预测时才上 TEXT 系列，并按预估上限选合适的子类型。

## 三、TEXT 的索引

- **必须用前缀索引：** TEXT 可能存几 KB 甚至几 MB，而 InnoDB 的 B+ 树对每个索引项的长度有上限，塞不进整段超长文本，所以 MySQL 禁止完整列索引，强制指定前缀长度，例如 `INDEX idx_content (content(100))` 只取每个值的前 100 个字符建索引。

    ```text
    -- INDEX idx_content (content(10)) 下，索引只存前 10 个字符
    "MySQL 是一个关系"   → 第 1 行
    "MySQL 索引的底层"   → 第 2 行
    "MongoDB 是文档"     → 第 3 行
    ```

- **查询必须回表两步走：** 索引项是截断前缀而非完整值，不同完整值可能共享同一前缀（如 `MySQL 是一个关系型数据库` 与 `MySQL 是一个关系运算`），光靠索引无法判定真正命中，所以分两步——① 用前缀在 B+ 树里定位、筛出候选行的主键；② 回表读完整 `content` 做精确比对，排除前缀相同的误命中。

- **前缀长度是权衡：** 太短则区分度（选择性）低——若一堆值都以 `https://www.example.com/` 开头，取前 20 字符几乎筛不掉任何行；太长则索引体积大、占内存多，失去省空间的意义。实践中通过计算选择性来挑长度。

- **两个固有局限：** 索引里只有截断值，拿不到完整数据，所以用不了覆盖索引、必走回表；且本质是「从左往右匹配开头」，只能优化 `WHERE content LIKE 'xxx%'`，对 `LIKE '%关键词%'`（中间包含）无能为力。

## 四、TEXT 的底层存储原理

- **页是基本单位：** InnoDB 把数据组织成默认 16KB 的页，行数据都存在页里。InnoDB 要求每页至少放下两行，所以单行（不含溢出部分）大小上限约 8126 字节，差不多半页。

- **行格式决定存法：** InnoDB 早期是 `REDUNDANT`、`COMPACT`，5.7 起默认 `DYNAMIC`，另有带压缩的 `COMPRESSED`。区别在于如何处理放不下的大字段。

- **溢出页机制：** 当一行因大字段超出页容量时，InnoDB 把最长的变长列挪到行外溢出页，行内只留指针。具体挪法取决于行格式。

    1. **`COMPACT` / `REDUNDANT`：** 行内保留该字段前 768 字节作为前缀，再加 20 字节指针指向溢出页，剩余存溢出页。768 字节前缀会占用宝贵的行内空间。

    2. **`DYNAMIC` / `COMPRESSED`：** 行内只留 20 字节指针，内容全部进溢出页，不再保留 768 字节前缀。这是 DYNAMIC 成为默认格式的原因之一——更高效地利用行内空间。

    - **溢出页是链表：** 溢出页以单向链表串起，大字段很长时会跨越多个溢出页。

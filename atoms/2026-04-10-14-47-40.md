# Linux 共享库路径配置 - /etc/ld.so.conf.d/

- `/etc/ld.so.conf.d/` 是 Linux 系统中用于管理共享库（Shared Libraries，即 `.so` 文件）搜索路径的配置目录。

## 为什么需要它？

- 在 Linux 中，当程序启动需要调用库文件（如 `libvixDiskLib.so`）时，系统动态链接器需要知道其具体位置：

    - **传统路径**：系统默认只搜索标准目录，如 `/lib64` 或 `/usr/lib64`。

    - **自定义路径**：若将库安装在非标准目录（如 `/usr/lib/vmware-vix-disklib/lib64`），链接器默认无法找到。

    - **解决方案**：`/etc/ld.so.conf.d/` 允许你通过创建一个以 `.conf` 结尾的配置文件，将自定义路径告知系统。

---

## 配置步骤

1. **添加路径配置**：

   - 在目录下创建一个新文件（文件名建议具名且以 `.conf` 结尾）：
        
        ```bash
        sudo vim /etc/ld.so.conf.d/vmware-vix-disklib.conf
        ```
   
   - 文件内容仅需写入库目录的绝对路径：
        
        ```text
        /usr/lib/vmware-vix-disklib/lib64
        ```

2. **刷新系统缓存**：
   执行以下命令使配置立即生效：
   ```bash
   sudo ldconfig
   ```

---

## 核心机制

- **生效机制**：`ldconfig` 会读取 `/etc/ld.so.conf.d/` 下的所有配置，并将所有有效的库路径汇总到 `/etc/ld.so.cache` 二进制缓存文件中。

- **系统级优势**：这种配置方式对系统内所有用户和所有进程（包括后台服务）均永久生效，比临时设置 `LD_LIBRARY_PATH` 更稳固、更规范。

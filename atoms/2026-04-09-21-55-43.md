# 如何修改 CentOS 7 的软件源？

### 1. 背景：为什么旧源失效了？

CentOS 7 目前已停止维护（**EOL**），其官方 YUM 源已同步失效。

- **现状**：由于官方域名 `mirrorlist.centos.org` 已停止解析，系统默认的 YUM 配置会导致连接超时并报错。

- **对策**：所有下线的历史版本已被移至 **Vault**（归档）仓库。我们需要将请求地址手动修正到阿里云的 Vault 镜像站，以恢复系统的软件包下载能力。

### 2. 操作步骤

#### 第一步：备份现有的 Repo 文件

```bash
mkdir -p /etc/yum.repos.d/bak
mv /etc/yum.repos.d/CentOS-*.repo /etc/yum.repos.d/bak/
```

#### 第二步：写入新的阿里云 Vault 源

- 直接在终端执行以下命令，向 `/etc/yum.repos.d/` 目录下写入名为 `CentOS-Vault.repo` 的配置文件。

```bash
cat > /etc/yum.repos.d/CentOS-Vault.repo <<'EOF'
[base]
name=CentOS-7.9.2009 - Base - mirrors.aliyun.com
baseurl=https://mirrors.aliyun.com/centos-vault/7.9.2009/os/$basearch/
gpgcheck=0
enabled=1

[updates]
name=CentOS-7.9.2009 - Updates - mirrors.aliyun.com
baseurl=https://mirrors.aliyun.com/centos-vault/7.9.2009/updates/$basearch/
gpgcheck=0
enabled=1

[extras]
name=CentOS-7.9.2009 - Extras - mirrors.aliyun.com
baseurl=https://mirrors.aliyun.com/centos-vault/7.9.2009/extras/$basearch/
gpgcheck=0
enabled=1
EOF
```

#### 第三步：清理并重建 YUM 缓存

```bash
yum clean all
yum makecache
```

1. `yum clean all`：清理本地旧的缓存元数据、软件包和暂存文件。
    
    - **原理**：递归删除 `/var/cache/yum/` 目录下的所有本地索引与临时文件，强制 YUM 放弃旧数据。
    
    - **风险**：若不执行，系统可能会因残留旧源的路径信息而尝试连接已失效的域名。

2. `yum makecache`：从新配置的 Vault 源中拉取最新的 RPM 包索引，并在本地重新构建缓存。

    - **原理**：提前将远端仓库的 `repodata` 索引下载到本地并建立数据库映射，从而大幅提升后续查询与安装的速度。

### 3. 深度解析：YUM 配置文件（.repo）干了什么？

`.repo` 文件指定了系统该去哪里拉取 RPM 软件包。

- **配置项详解**：
    - `[base]`：仓库唯一 ID。
    - `name`：易读的仓库描述信息。
    - `baseurl`：RPM 包的真实下载地址根目录。
    - `enabled=1`：启用该仓库。
    - `gpgcheck=0`：是否校验 RPM 包的数字签名（此处为了恢复服务暂设为 0）。

- **Vault URL 路径分解**：
  以 `https://mirrors.aliyun.com/centos-vault/7.9.2009/os/$basearch/` 为例：
    - `mirrors.aliyun.com`：阿里云官方镜像站域名。
    - `centos-vault`：代表此路径指向的是历史归档仓库。
    - `7.9.2009`：CentOS 7 的最后一个稳定发行版本号。
    - `$basearch`：YUM 内部变量。执行时会自动替换为具体的系统架构（如 `x86_64` 或 `aarch64`）。

### 4. 补充：什么是 Vault？

- **定义**：CentOS 的“历史档案馆”，专门存放已停止维护（EOL）或过时的旧版本数据。

- **机制**：当某个版本停止维护，官方会将其从实时同步的主镜像站下线，转入 Vault 站点长期保存，确保老用户仍能获取软件。

- **特性**：内容只读且不再更新。在生产环境中，建议使用国内大厂提供的 `centos-vault` 镜像（如阿里云），速度比直接连接官方 Vault 更快。

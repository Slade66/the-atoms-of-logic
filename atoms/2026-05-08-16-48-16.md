# veth pair（虚拟以太网对）

### 一、核心理解

- **基本定义** `veth pair` 是 Linux 内核提供的一种虚拟网络设备，总是成对出现，可以理解成“一根虚拟网线，两端各有一个网口”。

### 二、核心特性

- **双端互通** 从一端发送的数据包，会立即从另一端收到，就像一根直连网线。

- **跨命名空间** 两端可以分别放在不同的 `Network Namespace` 中，这是它在容器网络里最重要的用途。

- **成对生命周期** 如果一端被删除，另一端也会自动消失。

### 三、典型使用场景

- **容器网络** Docker 或 Kubernetes 会把 `veth pair` 的一端放进容器的 `Network Namespace`，另一端留在宿主机，并连接到网桥，例如 `docker0` 或自定义 Docker 网桥。

- **命名空间互联** `veth pair` 专门用来打通不同 `Network Namespace` 之间的网络隔离，两个隔离的网络命名空间可以通过一对 `veth` 设备互相通信。

### 四、在 Docker 里的连接方式

- **Docker 连接模型** 在 Docker 里，宿主机侧的 `veth` 像插头一样接到 Docker 网桥上，容器侧的另一端进入容器内部并显示为 `eth0`；可以把这一对设备理解成一根跨越命名空间的虚拟网线。

    ```text
    宿主机这侧                          容器那侧
    veth7a7d122  <==== 虚拟网线 ====>  容器 172.18.0.2 里的 eth0
    vethda98f16  <==== 虚拟网线 ====>  容器 172.18.0.3 里的 eth0
    ```

    ```text
    [ br-1b748c70f17b 网桥（交换机）]
            |              |
       veth7a7d122    vethda98f16       <- 宿主机侧，插在交换机上
            |              |
      （穿越命名空间）  （穿越命名空间）
            |              |
      容器 X 的 eth0   容器 Y 的 eth0    <- 容器侧，容器以为自己有真网卡
    ```

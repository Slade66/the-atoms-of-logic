# BIOS 与 UEFI

- BIOS 与 UEFI 都是计算机开机后、操作系统启动前运行的固件（firmware）。它们的核心目标是：在你按下电源键后、操作系统还没跑起来之前，负责把硬件叫醒，然后去硬盘上把操作系统找出来加载进内存。UEFI 是 BIOS 的现代替代品。

- [BIOS（Basic Input/Output System）](2026-04-19-01-56-43.md)

- [UEFI（Unified Extensible Firmware Interface）](2026-04-19-01-57-36.md)

- **固件用于解决什么问题？** 一句话：开机那一刻，CPU 只会读内存里的指令，可操作系统躺在硬盘上、内存还是空的——得有人先把操作系统从硬盘搬到内存。这个"搬运工"就是固件（BIOS/UEFI）。

    - **CPU 是个"只会读指令的傻子"：** CPU 的工作方式极其机械——永远从内存某个地址读一条指令、执行、再读下一条。它不知道什么叫"操作系统"、"硬盘"、"文件"，只认内存地址和机器码指令。

    - **通电瞬间的"鸡生蛋"难题：** 三个硬约束互相矛盾——

        - **内存是空的：** RAM 是易失性的，断电清零，刚通电时全是垃圾数据。

        - **操作系统够不着：** 操作系统躺在硬盘上，但 CPU 不会访问硬盘，它只会访问内存。

        - **CPU 等不了：** 通电后立刻要执行指令，可内存是空的，执行啥？

        - **矛盾：** 操作系统在硬盘里，CPU 只能执行内存里的指令，而内存是空的——谁来把操作系统从硬盘搬到内存？

    - **解决方案：在"断电不丢"的地方放一段启动代码。**

        - **硬件层：** 主板上焊一块 ROM/Flash 芯片，断电数据不丢，里面预先烧录一段程序。

        - **CPU 约定：** CPU 设计时就硬性规定——通电后第一条指令去读一个固定的物理地址（x86 上是 `0xFFFFFFF0`）。

        - **地址路由：** 这个地址不指向 RAM，而是被主板电路路由到 ROM 芯片。这段 ROM 里的程序，就是 BIOS/UEFI。

    - **完整启动链条：**

        - 通电 → CPU 去 `0xFFFFFFF0` 取指令
        - → 地址被主板电路路由到 ROM 芯片
        - → 执行 ROM 里的代码（= BIOS/UEFI）
        - → 固件初始化内存、认硬盘
        - → 固件从硬盘读出操作系统引导程序到 RAM
        - → 固件跳转到 RAM 里那段代码
        - → 操作系统接管

    - **为什么要两阶段加载引导程序？** 操作系统文件可能几个 GB，BIOS 芯片才几 MB，根本塞不下。所以固件只做一件小事：从硬盘第 0 扇区（固定位置）读 512 字节到内存，然后跳过去执行。那 512 字节叫 MBR，是个"小引导程序"，它再去读更大的引导程序，引导程序再去读操作系统内核——层层接力。

    - **固件开机后具体做了哪些事？**

        - **自检（POST）：** 看看 CPU、内存、硬盘在不在，坏没坏。

        - **认外设：** 初始化硬盘、键盘、显卡等外设。

        - **找引导设备：** 按照你设置的顺序（硬盘 → U 盘 → 网络）去找"谁来接班"。

        - **交棒：** 找到一个能启动的硬盘后，把硬盘开头那段引导代码读进内存，跳过去执行，然后撒手不管。

- **BIOS 与 UEFI 对比：**

    - 
      | 维度 | BIOS (Legacy) | UEFI |
      | --- | --- | --- |
      | 位宽 | 16 位 | 32/64 位 |
      | 分区表 | MBR | GPT |
      | 磁盘上限 | 2TB | 基本无限制 |
      | 引导位置 | MBR 首扇区 | ESP 分区中的 .efi 文件 |
      | 安全启动 | 无 | Secure Boot |
      | 界面 | 纯文本 | 图形 + 鼠标 |

- **V2V 迁移固件一致性要求：** 源虚拟机的固件类型（BIOS vs UEFI）必须与目标平台保持一致。

    - **BIOS 虚拟机：** 磁盘使用 MBR，目标 VM 也需配置为 BIOS 引导。

    - **UEFI 虚拟机：** 磁盘使用 GPT 且有 ESP 分区，目标 VM 必须配置为 UEFI 引导，否则无法启动。

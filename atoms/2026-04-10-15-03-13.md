# VMware VDDK SDK 安装与验证

- VMware Virtual Disk Development Kit (VDDK) 是开发能够访问 VMware 虚拟机磁盘内容的应用程序的基础。本文详细记录了在 Linux（Anolis 8）环境下安装 VDDK 9.0.1 并通过 C 程序进行初始化验证的全过程。

---

## 1. 解压安装包

首先将 VDDK 的 tar 包解压到临时目录，以便检查其内部结构。

```bash
mkdir -p /tmp/vddk-extract
tar xzf /root/VMware-vix-disklib-9.0.1.0.24964809.x86_64.tar.gz -C /tmp/vddk-extract
```

**解压后的核心目录结构：**
- `include/`：包含 `vixDiskLib.h` 等开发必需的头文件。
- `lib64/`：包含 `libvixDiskLib.so` 及其依赖项（如 OpenSSL、cURL 等）。

---

## 2. 部署到标准路径

建议将 SDK 移动到 `/usr/lib` 下的固定目录，方便长期管理。

```bash
sudo cp -r /tmp/vddk-extract/vmware-vix-disklib-distrib /usr/lib/vmware-vix-disklib
```

**验证部署：**
```bash
ls /usr/lib/vmware-vix-disklib/
# 应输出: bin64  doc  FILES  include  lib32  lib64
```

---

## 3. 配置动态链接库路径

为了让系统在程序运行时能够找到 VDDK 的共享库，需要将其 `lib64` 路径注册到动态链接器。

```bash
# 1. 创建配置文件
echo '/usr/lib/vmware-vix-disklib/lib64' | sudo tee /etc/ld.so.conf.d/vmware-vix-disklib.conf

# 2. 刷新系统共享库缓存
sudo ldconfig

# 3. 验证库是否已被正确识别
ldconfig -p | grep vixDiskLib
```

**预期输出：**
```text
libvixDiskLib.so.9.0.1.0 (libc6,x86-64) => /usr/lib/vmware-vix-disklib/lib64/libvixDiskLib.so.9.0.1.0
libvixDiskLib.so (libc6,x86-64) => /usr/lib/vmware-vix-disklib/lib64/libvixDiskLib.so
```

---

### 4. 编写最小验证程序

通过一个极简的 C 程序调用初始化和清理接口，验证库的加载是否正常。

- **关键 API 说明：**
    - **`VixDiskLib_InitEx`**：初始化 VDDK 库。
    - **`VixDiskLib_Exit`**：释放资源。

- **示例代码：`test_vddk_init.c`**

    ```c
    #include <stdio.h>
    #include <vixDiskLib.h>

    // 日志回调：此处使用空函数以屏蔽大量内部调试日志
    static void Noop(const char *fmt, va_list args) { }

    int main(void) {
        printf("[1/3] 正在调用 VixDiskLib_InitEx(9, 0) ...\n");
        
        // 注意：VDDK 9.0.1 需显式传版本号
        VixError err = VixDiskLib_InitEx(9, 0, Noop, Noop, Noop,
                                        "/usr/lib/vmware-vix-disklib/lib64", NULL);
        
        if (err != VIX_OK) {
            printf("[-] 初始化失败, 错误码: %llu\n", (unsigned long long)err);
            return 1;
        }
        printf("[2/3] VixDiskLib_InitEx 成功!\n");

        VixDiskLib_Exit();
        printf("[3/3] VixDiskLib_Exit 完成，VDDK SDK 验证通过!\n");
        return 0;
    }
    ```

---

## 5. 编译与运行

### 编译命令
使用 `-Wl,-rpath` 参数可以减少对 `LD_LIBRARY_PATH` 的依赖。

```bash
gcc -Wall -o test_vddk_init test_vddk_init.c \
    -I/usr/lib/vmware-vix-disklib/include \
    -L/usr/lib/vmware-vix-disklib/lib64 \
    -lvixDiskLib \
    -Wl,-rpath,/usr/lib/vmware-vix-disklib/lib64
```

### 运行结果
```bash
./test_vddk_init
```
**成功提示：**
```text
[1/3] 正在调用 VixDiskLib_InitEx(9, 0) ...
[2/3] VixDiskLib_InitEx 成功!
[3/3] VixDiskLib_Exit 完成，VDDK SDK 验证通过!
```

---

## ⚠️ 踩坑记录：版本宏缺失问题

**现象**：在旧版示例中常看到使用 `VIXDISKLIB_VERSION_MAJOR` 等宏，但在 9.0.1 版本编译时会报错 `undeclared`。

**原因**：VDDK 9.0.1 的 `vixDiskLib.h` 已移除这些宏定义。

**对策**：根据头文件声明，直接在 `VixDiskLib_InitEx` 中传递具体的数值（如 `9, 0`）即可。

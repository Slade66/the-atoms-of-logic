# 网络 翻墙协议与节点搭建

### 🧠 翻墙流量的两段旅程

- 买好机器后，你要在 VPS 上装代理软件（也就是服务端）。这涉及你与服务器之间通信的加密伪装协议——流量从你到目标服务器，要过两道关卡，每道关卡的"警察"和破防手段完全不同。

- **第一段：你的电脑 ➡️ GFW ➡️ 你的美国 VPS**

    - **看门人：** GFW（长城防火墙），手段是深度包检测 (DPI) 与主动探测，专门抓"翻墙行为"。

    - **过关方式：** 把翻墙流量伪装成正常网站的流量，骗过 GFW——让他以为你只是在访问一个境外的非敏感 IP（非 Facebook、Pornhub），数据便能安全地、不受干扰地到达你的美国 VPS。

- **第二段：你的美国 VPS ➡️ 互联网 ➡️ Claude 服务器**

    - **看门人：** Claude 的风控系统，手段是 IP 欺诈库查询 (IPQualityScore)、DNS 泄露检测、WebRTC 探测、浏览器指纹比对，专门抓"非服务区用户"和"滥用机器人"。

    - **过关方式：** VPS 把你的数据解密后，以 VPS 自己的名义（美国住宅 IP）发给 Claude。Claude 查了一下这个 IP 的"户口本"，发现是"美国某家庭宽带"，于是放行。

### 🧠 旧协议为何必死：GFW 是怎么识破你的

- **旧方案的完整流程：** 在 Reality 协议诞生之前，搭节点必须经历：买个废弃域名 → 把域名解析到 VPS → 申请网站 TLS 证书 → 伪装成一个连你自己都不会去看的垃圾博客。

- **致命伤：** 这套老方法最大的致命伤在于——GFW 的 AI 审查员只需对你的服务器稍作主动探测，便会发现破绽："这个野生域名毫无知名度，TLS 证书是免费签发的，流量却出奇地大，绝对是翻墙节点，封！"

- **连锁反应与 Claude 的致命风控点：** IP 一旦被墙，你就连不上服务器，只能花钱找商家换新 IP——而这正是 Claude 的致命风控点所在：账号今天在洛杉矶的 IP 登录，明天换到纽约，后天又换了一个，频繁的跨地域 IP 跳变（根源是你的 IP 总被墙、被迫不断更换）会让 Claude 大概率给你封号。

### 🧠 VLESS + XTLS-Reality 的绝招：不建假网站，直接借用真权威域名

- **核心思想：** 不再自己搭建伪装网站，而是直接"借用"一个真实存在的、极具权威性的海外大厂网站的 TLS 身份来伪装流量。这一设计赋予了 Reality 协议极强的抗主动探测能力：当 GFW 对可疑 IP 发起主动扫描时，Reality 会完美模拟一个合法的 TLS 网站，向探测方返回与目标"被借用网站"一模一样的响应，极难被识破和区分。

- **实际效果：** Reality 极其稳定，能确保你的极品双 ISP 住宅 IP 长达几个月甚至几年都不被 GFW 墙掉。你不仅有了一个干净的美国 IP，而且能长久固定使用它——这在 Claude 眼里是最高级的"号主信任分"。

### 🧠 动手搭建：3X-UI 面板部署 Reality 节点

- **安装步骤：**

    1. 用 SSH 连接到服务器

    2. 执行 3x-ui 面板一键安装脚本

        1. `Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]: n`
        
        2. SSL/TLS 证书配置（强制性安全要求）

           - 当前的 3x-ui 在安装过程中已默认强制要求申请 SSL 证书，旨在通过 HTTPS 协议确保管理面板的通信安全。

            - 
                ```
                ═══════════════════════════════════════════
                    SSL Certificate Setup (MANDATORY)     
                ═══════════════════════════════════════════
                For security, SSL certificate is required for all panels.
                Let's Encrypt now supports both domains and IP addresses!

                Choose SSL certificate setup method:
                1. Let's Encrypt for Domain (90-day validity, auto-renews)
                2. Let's Encrypt for IP Address (6-day validity, auto-renews)
                3. Custom SSL Certificate (Path to existing files)
                Note: Options 1 & 2 require port 80 open. Option 3 requires manual paths.
                Choose an option (default 2 for IP):
                ```

            - **Let's Encrypt for Domain**：适用于已有域名，且已将该域名解析至服务器 IP 地址的场景。

            - **Let's Encrypt for IP Address**：适用于无域名，仅希望通过服务器 IP 地址（如 `https://1.2.3.4`）直接访问的场景。

            - **Custom SSL Certificate**：适用于已在第三方平台（如阿里云、腾讯云、Cloudflare 等）申请了证书（通常包含 .crt 与 .key 文件）并希望手动上传使用的场景。用户需手动提供这些证书文件在服务器上的绝对存放路径。

            - **为什么配置 SSL 时必须开放 80 端口？**
            
                - **所有权验证**：在申请 Let's Encrypt 证书时，系统需要验证你对该服务器的实际控制权。

                - **完整流程**：

                    1. 你在服务器上执行申请命令后，本地的 ACME 客户端（如 Certbot）会在服务器上生成一个一次性令牌（Token），并将其写入一个临时文件，存放于固定路径：`http://<你的域名或IP>/.well-known/acme-challenge/<token>`

                    2. 随后，Let's Encrypt 的验证服务器会主动通过 HTTP 协议访问上述路径，尝试读取该文件。
                    
                    3. 若文件可被成功访问，则证明你对该服务器拥有实际控制权，证书随即签发，临时文件也会被自动清理。

                - **验证机制**：选项 1 和 2 均采用 HTTP-01 质询验证。Let's Encrypt 的验证服务器会通过 HTTP 协议（固定走 80 端口）向你的服务器请求一个临时验证文件；若文件可被成功读取，则证明"该服务器确实归你管理"，证书随即签发。

                - **失败原因**：若防火墙或云服务商安全组封锁了 80 端口，导致验证请求无法到达服务器，整个质询流程将直接中断，SSL 证书也将无法自动签发。

    3. 通过浏览器登录可视化后台管理面板

    4. 创建 Reality 节点：在面板左侧导航栏点击「入站列表」，再点击「添加入站」按钮。在弹出的配置窗口中，按以下设置逐项填写：

        1. **备注**：自由填写，用于识别节点，例如 `reality-test`

        2. **协议**：下拉选择 `vless`

        3. **端口**：随意填写一个未被占用的端口号

        4. **网络 (Network)**：选择 `tcp`

        5. **安全性 (Security)**：下拉选择 `reality`

        6. **传输方式设置 (Reality Settings)**：

            - **uTLS**：选择 `chrome` 或 `firefox`，用于模拟对应浏览器的 TLS 指纹

            - **target**：填入一个未在国内被屏蔽、支持 TLS、且服务器 IP 不在大陆的网站域名。该字段通常会自动生成，无需手动填写。

            - **伪装域名 (Server Names/SNI)**：与上方 target 保持一致，例如 `www.yahoo.com`。该字段通常会自动生成，无需手动填写。

            - **私钥 (Private Key) / 公钥 (Public Key)**：点击旁边的「Get New Cert」按钮，系统将自动生成一对非对称密钥。

            - **Short IDs**：点击旁边的生成按钮，自动生成一串随机标识符。

        7. 确认各项配置无误后，点击「创建」保存。

- **导出连接：** 在本机打开客户端代理软件（推荐使用 Clash Verge Rev 或 v2rayN），将面板生成的节点链接导入即可。

### 🧠 什么是 Skill

Skill 是一个可复用的"知识包"，其本质是一个包含 `SKILL.md` 文件的文件夹。这个文件里写有当智能体处理特定类型任务时可以遵循的说明——你可以把它理解为一份"处理特定任务的 SOP"。当遇到某类工作时，智能体会翻阅对应的说明书，按照里面的方法操作，而不是临时凭感觉处理。

每个 Skill 通常包含：

- 针对某一类任务的处理步骤或方法说明
- 需要遵循的最佳实践和约定规范
- 可选的脚本和资源，供智能体在执行任务时使用

### 🧠 工作原理：渐进式展开

Skills 遵循"渐进式展开"的三段式模式：

**Discovery（发现）** — 对话开始时，智能体会看到一份可用技能的列表，包括每个技能的名称和描述。

**Activation（激活）** — 如果某个技能看起来与当前任务相关，智能体会读取该技能完整的 `SKILL.md` 内容。

**Execution（执行）** — 在处理任务时，智能体会按照技能中的说明进行操作。

你不需要显式告诉智能体使用某个技能——它会根据上下文自行判断。不过，如果你希望确保某个技能被使用，也可以在对话中直接提到技能名称。

### 🧠 Skills 存放位置

Antigravity 支持两种作用域的 Skills：

| Location | Scope |
|----------|-------|
| `<workspace-root>/.agent/skills/<skill-folder>/` | Workspace-specific（当前项目） |
| `~/.gemini/antigravity/skills/<skill-folder>/` | Global（所有工作区） |

### 🧠 创建一个 Skill

**第一步：** 在 `.agent/skills/` 下创建一个文件夹，文件夹名就是 Skill 名称。

**第二步：** 在文件夹中创建 `SKILL.md` 文件，写入说明内容。

```
.agent/skills/
└─── my-skill/
    └─── SKILL.md
```

**`SKILL.md` 模板：**

```markdown
---
name: my-skill
description: 在处理什么任务时使用这个 skill。
---

# My Skill

这里填写给智能体的详细说明。

## 什么时候使用这个技能

- 当……时使用
- 在……场景下会很有帮助

## 如何使用

这里写清楚分步骤的操作指引、需要遵循的约定，以及建议采用的模式或流程。
```

**Frontmatter 字段说明：**

| 字段 | 是否必填 | 说明 |
|------|----------|------|
| `name` | 否 | 技能的唯一标识符（全部小写，单词之间使用连字符）。如果未提供，默认使用文件夹名称。 |
| `description` | 是 | 清晰描述该技能的功能以及适用场景。这是智能体在判断是否使用该技能时看到的内容。 |

**可选的文件夹结构：**

`SKILL.md` 是唯一必需的文件，但你也可以包含额外的资源文件，智能体在执行任务时可以读取它们：

```
.agent/skills/my-skill/
├─── SKILL.md       # 主要说明（必需）
├─── scripts/       # 辅助脚本（可选）
├─── examples/      # 示例实现（可选）
└─── resources/     # 模板和其他资源文件（可选）
```

### 🧠 最佳实践

1. **保持技能聚焦** — 每个技能应专注于做好一件事。与其创建一个"什么都做"的技能，不如为不同类型的任务分别创建独立的技能。

2. **编写清晰的描述** — `description` 是智能体判断是否使用该技能的唯一依据。要具体说明技能能做什么，以及在什么场景下适用。描述越清晰，智能体越容易在合适的时候调用它。

3. **将脚本视为黑盒** — 如果技能中包含脚本，建议让智能体先通过 `--help` 查看使用方式，而不是直接阅读完整源码。这样可以把注意力集中在"如何使用工具"，而不是"工具内部是怎么实现的"，避免占用不必要的上下文。

4. **包含决策树** — 对于较复杂的技能，可以添加一个"决策流程"或"选择指南"部分，帮助智能体根据不同情况选择合适的处理方式。这就像给出一张分岔路口的地图，让智能体在不同场景下知道该往哪条路走。

# Visual Studio Code Agent Skills

### 🧠 Agent Skills 是什么

- **能力定义：** Agent Skills 是包含指令、脚本和资源的文件夹，GitHub Copilot 可以根据需要加载这些文件夹来执行特定任务。

- **可移植性：** 技能具有可移植性，可在任何支持技能的 AI 上运行。

### 🧠 Agent Skills 的主要优势

- **使 Agent 更加专业化：** 针对特定领域的任务定制功能，将专业知识和最佳实践内置其中，使 AI 助手能够自动理解你的专业需求并提供精准的帮助。

- **减少重复：** 一次创建后即可在所有对话中自动使用，无需在每次对话中重复提供相同的上下文信息。

- **组合功能：** 将多个技能组合成工作流，一次调用完成复杂任务。

- **高效加载：** 采用按需加载机制，仅在实际需要时将相关内容引入上下文，避免一次性加载所有信息造成的资源浪费，既提升响应速度，又降低 token 消耗成本。

### 🧠 Skills 与 Custom Instructions 的区别

- **定位差异：** 虽然 Agent Skills 和自定义指令都能帮助定制 Agent 的行为，但与用于定义规范的“自定义指令”不同，技能可以包含脚本、示例和其他资源，用于实现特定功能和工作流程。

- **对比表：**

  | 特性 | Agent Skills | 自定义指令 |
  |------|--------------|-----------|
  | 用途 | 教授专业能力和工作流程 | 定义标准和指南 |
  | 内容 | 指令、脚本、示例和资源 | 仅指令 |
  | 作用范围 | 特定任务，按需加载 | 始终应用（或通过 glob 模式根据文件类型应用规则） |
  | 标准 | 开放标准 (agentskills.io) | VS Code 专用 |

- **使用 Agent Skills 的场景：**

  - **跨工具复用能力：** 创建可在不同 AI 工具中使用的可复用能力，基于开放标准构建的技能可以跨平台使用，无论是在 VS Code、命令行工具还是其他支持该标准的 AI 代理中，都能保持一致的功能体验。

  - **承载资源而非纯规则：** 在指令之外包含脚本、示例或其他资源，不仅可以提供文本指令，还能打包实际的代码脚本、配置文件、测试用例等实体资源，让 AI 助手能够直接执行或参考这些内容。

  - **社区共享：** 与更广泛的 AI 社区共享能力，通过标准化的格式，可以轻松地将你创建的技能分享给其他开发者，或者使用社区贡献的技能来扩展自己的工作流程。

- **使用自定义指令的场景：**

  - **项目编码标准：** 定义项目特定的编码标准，为当前项目设置统一的代码风格要求，例如缩进规则、命名约定、注释格式等，确保团队成员生成的代码保持一致性。

  - **技术约定统一：** 设置语言或框架约定，指定使用特定版本的语言特性、首选的库或框架、API 调用方式等技术选型，让 AI 助手生成符合项目技术栈的代码。

### 🧠 创建 Skill 的方式

- **入口命令：** 在 Chat 面板的聊天框里输入 `/skills` 打开配置 Skill 的菜单。

- **存储规则：** Skills 存储在包含 `SKILL.md` 文件的目录中，该文件定义了技能的行为。

- **技能位置：**

  | 技能类型 | 位置 |
  |---------|------|
  | 项目技能，存储在你的仓库中 | `.github/skills/`, `.claude/skills/`, `.agents/skills/` |
  | 个人技能，存储在你的用户配置文件中 | `~/.copilot/skills/`, `~/.claude/skills/`, `~/.agents/skills/` |

- **位置扩展：** 你可以通过 `chat.agentSkillsLocations` 设置来配置 VS Code 搜索技能的其他位置。

- **创建步骤：**

  1. **创建目录：** 在你的工作区中创建 `.github/skills` 目录。

  2. **创建子目录：** 为你的技能创建一个子目录，每个技能都应该有自己的目录（例如，`.github/skills/code-review`）。

  3. **编写主文件：** 在技能目录中创建一个 `SKILL.md` 文件，结构如下。

     ```markdown
     ---
     name: skill-name
     description: 描述该技能的功能以及何时使用它
     ---

     # 技能指令

     在这里编写详细的指令和示例...
     ```

  4. **可选扩展：** 可选：向技能目录添加脚本、示例或其他资源。

- **目录示例：** 例如，一个用于代码审查的技能目录结构可能如下。

    - 
        ```text
        .github/skills/code-review/
        ├── SKILL.md                    # 主技能文件，包含代码审查的核心指令和流程
        ├── review-checklist.md         # 详细的审查清单，涵盖安全性、性能、可维护性等方面
        ├── examples/
        │   ├── good-practices.md       # 优秀代码示例和最佳实践
        │   ├── common-issues.md        # 常见问题和反模式
        │   └── review-comments.md      # 审查评论模板和建议用语
        └── scripts/
            └── analyze-complexity.js   # 自动分析代码复杂度的脚本
        ```

    - **结构价值：** 这样的结构让 AI 助手能够根据 `SKILL.md` 中的指令执行系统化的代码审查，参考 `review-checklist.md` 确保不遗漏关键检查点，从 `examples/` 中学习如何识别问题和提供建设性反馈，并调用 `scripts/` 中的工具进行自动化分析。

### 🧠 使用 AI 生成 Skill

- **生成命令：** 你可以使用 AI 根据能力描述来生成技能，在聊天框中输入 `/create-skill` 并描述你想要的技能（例如，“一个用于运行和调试集成测试的技能”）。

- **生成流程：** AI 代理会询问澄清性问题，然后生成包含目录结构、指令和 frontmatter 的 `SKILL.md` 文件。

- **对话提炼：** 你还可以从正在进行的对话中提取可复用的技能，例如在经过多轮对话调试了一个复杂问题后，你可以说“根据我们刚才的调试过程创建一个技能”，将这个多步骤流程捕获为可复用的技能。

### 🧠 SKILL.md 文件格式

- **文件定义：** `SKILL.md` 文件是一个带有 YAML frontmatter 的 Markdown 文件，用于定义技能的元数据和行为。

- **头部要求：** 头部（必需）使用 YAML frontmatter 格式，包含以下字段。

  | 字段 | 是否必需 | 描述 |
  |------|---------|------|
  | `name` | 是 | 技能的唯一标识符。必须小写，使用连字符分隔单词（例如，`webapp-testing`）。必须与父目录名称匹配。最多 64 个字符。 |
  | `description` | 是 | 描述技能的功能以及何时使用它。要具体说明功能和使用场景，以帮助 Copilot 决定何时加载该技能。最多 1024 个字符。 |
  | `argument-hint` | 否 | 当技能作为斜杠命令调用时，在聊天输入框中显示的提示文本，帮助用户了解需要提供哪些额外信息。 |
  | `user-invocable` | 否 | 控制技能是否作为斜杠命令出现在聊天菜单中。默认为 true。设置为 false 可以在 `/` 菜单中隐藏该技能，但仍允许代理自动加载它。 |
  | `disable-model-invocation` | 否 | 控制代理是否可以根据相关性自动加载技能。默认为 false。设置为 true 则要求仅通过 `/` 斜杠命令手动调用。 |

- **正文要求：** 技能正文包含 Copilot 使用该技能时应遵循的指令、指南和示例，编写清晰、具体的指令，描述技能帮助完成什么任务、何时使用该技能、要遵循的分步流程、预期输入和输出的示例，以及对任何包含的脚本或资源的引用。

- **相对路径引用：** 你可以使用相对路径引用技能目录中的文件，例如要引用技能目录中的脚本，使用 `[test script](./test-template.js)`。

### 🧠 将 Skill 作为斜杠命令使用

- **命令入口：** 在聊天中，Skill 会和 Prompt 文件一起作为斜杠命令可用，在聊天输入框中输入 `/`，即可查看可用的 Skill 和 Prompt 列表，并选择一个 Skill 来调用。

- **补充上下文：** 你可以在斜杠命令后补充额外上下文，例如：`/webapp-testing for the login page`，或 `/github-actions-debugging PR #42`。

- **默认行为：** 默认情况下，所有 Skill 都会出现在 `/` 菜单中，你可以通过 frontmatter 中的 `user-invocable` 和 `disable-model-invocation` 属性来控制每个 Skill 的访问方式。

- **配置矩阵：**

  | 配置 | 斜杠命令可见 | Copilot 自动加载 | 适用场景 |
  |------|--------------|------------------|----------|
  | 默认（两个属性都省略） | 是 | 是 | 通用 Skill |
  | `user-invocable: false` | 否 | 是 | 背景知识类 Skill，由模型在相关时自动加载 |
  | `disable-model-invocation: true` | 是 | 否 | 仅希望按需手动调用的 Skill |
  | 两者同时设置 | 否 | 否 | 禁用的 Skill |

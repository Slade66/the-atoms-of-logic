# MCP 解决了「工具不可描述」的问题

### 没有 MCP 之前

一个天气工具长这样：
```json
{
  "action": "getWeather",
  "payload": {
    "city": "Beijing"
  }
}
```
LLM 怎么知道：
这个工具是干嘛的？
参数有哪些？
city 是必须的吗？

这会导致工具对 LLM 来说就是黑盒，LLM 只能猜着用。

### 有了 MCP 之后

MCP 让工具“自描述”：工具会带“结构化描述”，能被 LLM 理解是什么、怎么用。LLM 不再“猜接口”

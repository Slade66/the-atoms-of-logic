## 卸载机制与原理解析

### 🧠 三位一体的工程化清理操作

当执行 `npm uninstall` 时，npm 会在后台自动且同步地完成以下三个维度的清理：

1. **清理物理实体：** 它会前往项目根目录下的 `node_modules/` 文件夹，物理删除掉 `<包名>` 的所有代码文件。如果它还引入了其他依赖，且没被项目中的其他包使用，npm 也会聪明地把这些“孤儿依赖”一并清理掉，防止项目变成垃圾场。
2. **清理声明：** 它会自动打开 `package.json` 文件，找到该包所在的依赖字段（如 dependencies），将那行代码彻底抹除。这确保了你的项目配置文件与实际安装的包保持精准同步。
3. **更新锁文件：** 它最后会去编辑 `package-lock.json`，把锁文件里关于该包的完整依赖树和哈希指纹彻底擦除，保持项目依赖状态的 100% 同步和安全。

### 🧠 强制清理本地缓存

**缓存冗余风险：** 尽管 npm 已经把包从项目里删了，但它通常会把这个包的安装文件（Tarball）缓存在系统全局，以备将来快速重装。

**彻底释放空间：** 如果你想彻底释放磁盘空间，或者怀疑某次安装因为缓存损坏而失败，可以执行强制清理：
```bash
npm cache clean --force
```

## 当前项目命令速查

### 🧠 卸载生产依赖 (dependencies)

**核心依赖卸载：** 从项目 `dependencies` 中彻底移除包和它的关联文件。这通常意味着该包（如 `axios`、`react`）不再是线上运行时或最终构建产物中不可或缺的一部分。
```bash
npm uninstall <包名>
```

### 🧠 卸载开发依赖 (devDependencies)

**开发依赖卸载：** 专门卸载仅在本地开发、测试或构建时使用的辅助包（如 `eslint`、`webpack`、`jest`），它们原本存在于 `devDependencies` 字段中。

```bash
npm uninstall -D <包名>
# 或全写
npm uninstall --save-dev <包名>
```

## 全局环境命令速查

### 🧠 卸载全局安装包

**系统级抹除：** 将曾经通过 `-g` 安装在操作系统全局环境下的 CLI 工具（如 `create-react-app`、`pm2`、`nodemon` 等）进行彻底抹除。

```bash
npm uninstall -g <包名>
```

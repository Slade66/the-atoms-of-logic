# MCP 解决了「工具参数检查」问题

工具说它要某个参数，但是 LLM 可能乱生成，没办法判断是否正确。

MCP 让参数有类型，让系统能“检查对不对”

MCP 让工具调用从“可能对”变成“可以被验证是否正确”。

before
“系统”通常不知道参数规范（因为没有 schema），只能直接把数据丢给工具。

after
系统可以在调用前就判断：
缺字段 ❌
类型错误 ❌
不符合 → 直接拒绝调用，而不是等工具内部崩。

这里的系统指的是负责把 LLM 输出变成工具调用的那个程序，它负责“接收 LLM 输出 → 检查参数决定要不要调用工具 -> 调用工具”

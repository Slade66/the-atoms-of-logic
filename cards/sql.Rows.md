# `sql.Rows`：表示多行查询结果的集合

## 类型签名

- `type Rows struct`

- **所属包：** `database/sql`

### `sql.Rows` 有什么特点

- 它通常来自 `Query` 或 `QueryContext`，适合处理“会返回多行数据”的查询结果。

- `sql.Rows` 背后占着数据库连接、网络资源和结果集游标，所以用完后必须及时 `defer rows.Close()`。

### 使用 `sql.Rows` 时要注意

- 如果忘记关闭 `Rows`，连接就不能及时归还给连接池。请求一多，连接池可能被耗尽，最后导致查询阻塞、报错，甚至把服务拖垮。

- 遍历结果集时，一般是先 `for rows.Next() { ... }`，循环结束后再检查一次 `rows.Err()`，这样才能拿到迭代过程中产生的错误。

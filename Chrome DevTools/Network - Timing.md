# Network - Timing

- **作用：** 用来分析一次 HTTP 请求的各个阶段耗时。分析慢在请求的哪个阶段（网络慢还是后端慢）？

- 面板按请求生命周期把耗时拆成几段：
    
    - **Resource Scheduling（资源调度）**
        
        - **Queueing（排队）：** 请求在浏览器里“排队等待发送”的时间。
        
            - 浏览器并发连接数限制（同域名最多 6 个连接），其它请求占用了连接。
    
    - **Connection Start（连接建立）** 请求准备建立连接但被“卡住”的时间。
    
    - **Request/Response（请求与响应）**
        
        - **Request sent（请求发送）：** 浏览器把请求数据发送给服务器的时间。
        
            - **作用：** 判断是不是网络慢、服务端读取慢？

            - 可能原因：请求体很大（比如上传文件）、网络带宽很慢、请求被限速。
        
        - **Waiting for server response（等待服务器响应）：** 服务器处理请求并返回第一个字节的时间。
            - 包括：后端处理逻辑、DB 查询、RPC 调用。
            
            - **作用：** 判断是不是后端慢。

        - **Content Download（下载响应）：** 浏览器下载响应数据的时间。


###  ESXi “Sidecar Proxy” 挂载截取 (可实现度：中 | 价值：极致的读写速度)


如果你觉得走 vCenter 的 HTTPS API 下载太慢（通常会被 VMware 的管理网卡限速），我们可以玩底层挂载。

极客玩法：

你的 Go 程序调用 vCenter API，在源端 VMware 集群里，自动克隆并启动一个极小的 Alpine Linux 虚拟机（我们称之为 Proxy 代理机，只有 50MB 大小）。

调用 VMware Hot-Add（热拔插）API，将客户要迁移的虚拟机的 VMDK 磁盘，直接挂载到这个 Proxy 代理机上（变成 Proxy 机的 /dev/sdb）。

因为 Proxy 机和目标磁盘在**同一个底层物理存储（SAN/vSAN）**上，Proxy 机读取该磁盘的速度是底层的光纤速度（可达 2GB/s 以上），完全绕过网络限速！

在 Proxy 内部直接通过内网高速流推送到华为 FusionCompute。

极客收益：速度比传统 API 提取快 3-5 倍，且对 vCenter 管理节点的压力几乎为零。
